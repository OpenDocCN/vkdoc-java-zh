# 附录 A：正则表达式参考

本附录为日常 Java 正则表达式需求提供了全面的快速参考。附录中的内容以纯正则表达式模式呈现，而非 Java 字符串转义后的形式。例如，在指代数字时，此处使用 ***\d***，而非 ***\\d***。

表 A-5：边界匹配器

| **正则表达式** | **描述** | **备注** |
| --- | --- | --- |
| **^** | 行首字符 | 这是一个不可见字符。 |
| **$** | 行尾字符 | 这是一个不可见字符。 |
| **\b** | 单词边界 | 这是单词边界的位置。其使用需要谨慎，因为 ***\b*** 不匹配字符，而是匹配一个位置。因此，字符串 *anna marrie* 不匹配正则表达式 ***anna\bmarie***。然而，它确实匹配 ***anna\b\smarrie***。这是因为在 *anna* 中最后一个 *a* 之后的位置存在一个单词边界，并且恰好是空格字符，因此需要 ***\s*** 来匹配它，然后 *marie* 必须紧随其后。由于 ***\b*** 匹配一个位置，因此为其添加贪婪量词是没有意义的。所以，***\b+***、***\b\b\b\b\b*** 和 ***\b*** 匹配的内容完全相同。更复杂的是，在字符类中，***\b*** 表示退格字符。这在语法上是合法的（尽管有点别扭），因为单词边界在字符类中没有位置。因此，***[\b]*** 描述了一个退格字符，因为它被 *[* 和 *]* 包围。 |
| **\B** | 非单词边界 | 这是单词边界的相反项，如上所述。 |
| **\A** | 输入的开头 | ***\A*** 匹配输入的开头，但它不仅仅是 ***^*** 模式的同义词。当你在编译模式时使用 Pattern.MULTILINE 标志时，这种区别就变得清晰了。***\A*** 匹配输入的开头，即文件的最开头。相比之下，当 Pattern.MULTILINE 标志激活时，***^*** 匹配每一行的开头。 |
| **\Z** | 输入的结尾，不包括最后的 ***$***（如果有） | ***\Z*** 匹配输入的结尾，但它不仅仅是 ***$*** 字符的同义词。当你在编译模式时使用 Pattern.MULTILINE 标志时，这种区别就变得清晰了。***\Z*** 匹配输入的结尾，即文件的最末尾。相比之下，当 Pattern.MULTILINE 标志激活时，***$*** 匹配每一行的结尾。 |
| **\G** | 上一次匹配的结尾 |   |
| **\z** | 输入的结尾 | 其行为与使用大写字母 *Z* 的 ***\Z*** 完全相同，区别在于它还会捕获结束的 ***$*** 字符。 |

表 A-7：懒惰量词

| **正则表达式** | **描述** | **说明** |
| --- | --- | --- |
| **X??** | *X*，出现零次或一次 | 此模式与 ***?*** 模式非常相似，区别在于它更倾向于不匹配任何内容。当与 `Matcher.matches()` 方法一起使用时，***??*** 的功能与 ***?*** 模式完全相同。然而，当与 `Matcher.find()` 一起使用时，行为则不同。例如，模式 ***x??*** 应用于字符串 *xx* 时，实际上不会找到 *x*，但仍认为匹配成功。这是因为我们要求它懒惰匹配，而它能做的最懒惰的事情就是匹配零次 *x*。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |
| **X*?** | *X*，出现零次或多次 | 此模式与 ******* 模式非常相似，区别在于它更倾向于尽可能少地匹配。当与 `Matcher.matches()` 方法一起使用时，****?*** 的功能与 ******* 模式完全相同。然而，当与 `Matcher.find()` 一起使用时，行为则不同。例如，模式 ***x*?*** 应用于字符串 *xx* 时，实际上不会找到 *x*，但仍认为匹配成功。这是因为我们要求它懒惰匹配，而它能做的最懒惰的事情就是匹配零次 *x*。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |
| **X+?** | *X*，出现一次或多次 | 此模式与 ***+*** 模式非常相似，区别在于它更倾向于尽可能少地匹配。当与 `Matcher.matches()` 方法一起使用时，***+?*** 的功能与 ***+*** 模式完全相同。然而，当与 `Matcher.find()` 一起使用时，行为则不同。例如，模式 ***x+?*** 应用于字符串 *xx* 时，实际上会找到一个 *x*，但仍认为匹配成功。这是因为我们要求它懒惰匹配，而它能做的最懒惰的事情就是匹配一次 *x*。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |
| **X{n}?** | *X*，恰好出现 *n* 次 | 此模式与 ***X{n}*** 模式完全相同。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |
| **X(n,}?** | *X*，至少出现 *n* 次 | 此模式与 ***X{n,}*** 模式非常相似，区别在于它更倾向于尽可能少地匹配。当与 `Matcher.matches()` 方法一起使用时，***X{n,}?*** 的功能与 ***X{n,}*** 模式完全相同。然而，当与 `Matcher.find()` 一起使用时，行为则不同。例如，模式 ***X{3,}?*** 应用于字符串 *xxxxx* 时，实际上只会找到 *xxx*，但仍认为匹配成功。相比之下，仅使用 ***X{3,5}*** 则会找到 *xxxxx*。这是因为我们要求它懒惰匹配，而它能做的最懒惰的事情就是匹配三次 *x*。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |
| **X{n,m}?** | *X*，至少出现 *n* 次，但不超过 *m* 次 | 此模式与 ***X{n,m}*** 模式非常相似，区别在于它更倾向于尽可能少地匹配。当与 `Matcher.matches()` 方法一起使用时，***X{n,m}?*** 的功能与 ***X{n,m}*** 模式完全相同。然而，当与 `Matcher.find()` 一起使用时，行为则不同。例如，模式 ***X{3,5}?*** 应用于字符串 *xxxxx* 时，实际上会找到 *xxx*，但仍认为匹配成功。相比之下，仅使用 ***X{3,5}*** 则会找到 *xxxxx*。这是因为我们要求它懒惰匹配，而它能做的最懒惰的事情就是匹配三次 *x*。请注意，如果有六个 *x* 字符，例如 *xxxxxx*，该模式会匹配两次：一次匹配前三个 *xxx* 字符，然后再次匹配另外三个。同样，这是因为三是最低要求。这适用于紧邻其前的单个字符、组（如果组紧邻其前）或字符类（如果字符类紧邻其前）。 |

表 A-9：逻辑运算符

| **正则表达式** | **描述** | **备注** |
| --- | --- | --- |
| **XY** | *X* 后跟 *Y* | 这是字符之间的默认关系假设。请注意，空格是该语法的有效组成部分。因此，***A*** ***B*** 表示字符 *A*，后跟一个空格，再后跟字符 *B*。 |
| **X&#124;Y** | *X* 或 *Y* | ***AB&#124;CD*** 将匹配 *AB* 或 *CD*。类似地，模式 ***hello sir&#124;madam*** 将匹配 *hello sir* 或匹配 *madam*。具体来说，它不会匹配 *hello madam*。这是由于前面讨论过的“与”模式的性质所致。当正则表达式引擎看到 *hello sir* 时，它假定你的意思是 *hello*，后跟一个空格，再后跟 *sir* 应被视为一个单一的逻辑单元。这些都被“与”运算组合在一起。然后引擎看到“或”模式，因此它假定逻辑替代项是 *madam*。如果你实际上想要接受 *hello sir* 或 *hello madam*，则必须使用分组——因此，模式为 ***hello (sir&#124;madam)***。或者更好的是，你可以使用非捕获组 ***hello (?:sir&#124;madam)***。 |
| **(X)** | *X*，捕获组 | 捕获组是一个逻辑单元，在概念上类似于你在代数中熟悉的逻辑单元。因此，***(\d\d\d\d)*** 是一个定义了四位数字的捕获组。捕获组可以在表达式的后面部分通过反向引用来引用，稍后将对此进行说明。它们从左到右计数，并且可以嵌套。因此，***h(ello (world))*** 有三个捕获组。捕获组 0 是整个表达式，它匹配字符串 *hello world*。捕获组 1 是 *ello world*，因为你从左到右计数，第一个组从 *hello* 中字母 *e* 之前的 *(* 开始。组 2 是 *world*，因为第二个组从 *world* 中字母 *w* 之前的 *(* 开始。 |
| **\n** | 匹配的第 *n* 个捕获组 | 在此上下文中，我指的不是换行符，即使 ***\n*** 看起来像换行符符号。此处的 ***n*** 指的是一个数字。正则表达式引擎允许你访问由组的前面部分捕获的信息，即使在搜索执行过程中也是如此。例如，如果你想查找重复的单词，你只需要模式 ***(\w+)\W\1***，它的意思是：“查找一组单词字符，后跟一个非单词字符，再后跟组 1 中捕获的完全相同的单词字符。”如果你尝试引用一个不存在的组，则会抛出 `PatternSyntaxException`。如果你碰巧有 13 个捕获组，那么 ***\13*** 将意味着你想要第十三个捕获组。如果你没有 13 个组，那么相同的表达式 ***\13*** 将意味着第一个捕获组，后跟数字 *3*。 |

表 A-12：环视

| **正则表达式** | **描述** | **备注** |
| --- | --- | --- |
| **(?=X)** | *X*，使用零宽度正向先行断言 | 此模式向右查看候选字符串中剩余待解析的内容，以找到表达式 *X* 存在的第一个位置。例如，如果你想从文本文件中提取所有内联注释，可以尝试模式 ***(?=//).*$*** 并提取组 0。此模式是非捕获的。 |
| **(?!X)** | *X*，使用零宽度负向先行断言 | 此模式向右查看候选字符串中剩余待解析的内容，以找到表达式 *X* 不存在的第一个位置。例如，如果你想跳过某些内容前面的前导空格，可以使用 ***(?!\s).****。此模式是非捕获的。 |
| **(?<=X)** | *X*，使用零宽度正向后行断言 | 此模式向左查看候选字符串中剩余待解析的内容，以找到表达式 *X* 存在的第一个位置。例如，如果你想从文本文件中提取所有内联注释，可以尝试模式 ***(?=//).*$***。此模式是非捕获的。 |
| **(?<!X)** | *X*，使用零宽度负向后行断言 | 此模式向左查看候选字符串中剩余待解析的内容，以找到表达式 *X* 不出现的位置。例如，如果你想从文本文件中提取内联 Java 注释之前的所有文本，可以尝试模式 ***.*(?<=//)***。此模式是非捕获的。 |
| **(?>*X*)** | *X*，作为一个独立的非捕获组 | 此模式拒绝释放匹配的内容，无论这对模式其余部分的匹配能力有何影响。因此，虽然模式 \***w+\d*** 匹配字符串 *java2*，但模式 ***(?>\w+)\d*** 不匹配，因为 ***(?>\w+)*** 消耗了 *java* 和 *2*，并且拒绝释放 *2* 以便 ***\d*** 能够匹配。 |
