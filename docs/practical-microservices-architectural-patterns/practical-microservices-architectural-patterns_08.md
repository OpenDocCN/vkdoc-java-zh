# 消息关联与微服务

在微服务架构中，企业应用程序的不同部分被划分为多个微服务，但它们仍然需要相互通信。为了获得微服务的真正好处，微服务间的调用必须通过消息传递基础设施进行。这意味着采用微服务架构的企业应用程序的生产生态系统将由许多微服务以及这些微服务的许多实例组成，它们通过一个或多个消息传递基础设施相互连接。当你在这些实例之间采用异步通信风格时，主要难题是如何有效解决它们之间的消息路由问题。本章前面的部分让你对连接微服务（包括消息关联）所涉及的内部复杂性有了一个概览。如果开发人员必须自己完成所有这些连接工作，代码将会变得过于臃肿。你需要一种更好的方法来处理这个问题。我将在第 12 章介绍 Axon 框架（它将为你隐式地完成这项工作）时，介绍处理此问题的方法。
