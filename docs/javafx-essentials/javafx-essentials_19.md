# JavaFX 历史

你可能认为 JavaFX 是一项相当新的技术，但实际上并非如此。JavaFX 已经存在很长时间了；非正式地说，自 2005 年起。自从 Sun Microsystems 收购了 *SeeBeyond* 公司以来，就存在一种名为 **F3**（**形**式**追**随**功**能）的图形丰富脚本语言，由工程师 Chris Oliver 创建。

在 2007 年的 JavaOne 大会上，Sun Microsystems 正式将 JavaFX 作为该语言的名称，取代了 F3。在 2007 年至 2010 年期间，Oracle 收购了许多大公司，如 BEA Systems、JD Edwards、Siebel Systems 等。我当时在 Oracle 工作，负责将不同的客户支持渠道整合到当时被称为 *MetaLink* 的 Oracle 支持网站。

2009 年 4 月 20 日，Oracle 公司宣布收购 Sun Microsystems，使 Oracle 成为 JavaFX 的新管理者。

在 2010 年的 JavaOne 大会上，Oracle 宣布了 JavaFX 路线图，其中包括逐步淘汰 JavaFX 1.3 脚本语言，并基于 Java 平台将 JavaFX 平台重建为基于 Java 的 API。正如承诺的那样，JavaFX 2.0 SDK 于 2011 年 10 月在 JavaOne 大会上发布。

除了发布 JavaFX 2.0，Oracle 还通过宣布承诺采取措施使 JavaFX 开源，将平台提升到了新水平，从而允许 Java 强大且多才多艺的社区帮助推动平台向前发展。使 JavaFX 开源增加了其采用率，加快了错误修复的周转时间，并产生了新的增强功能。

在 JavaFX 2.1 和 2.2 版本之间，新功能的数量迅速增长。JavaFX 2.1 是 Mac OS 上 Java SDK 的官方版本。JavaFX 2.2 是 Linux 操作系统上 Java SDK 的官方版本。

并没有 JavaFX 3.x 这样的版本，但 Java 开发领域的重大变化发生在 Java SE 8 发布之时，该版本于 2014 年 3 月 18 日宣布。Java SE 8 有许多新的 API 和语言增强功能，包括 **Lambdas**、Stream API、Nashorn JavaScript 引擎，以及 *JavaFX API*，这些都被整合到标准 JDK 捆绑包中，JavaFX 版本变为 8，作为 JavaFX 2.0 的直接继承者。

要查看 Java SE 8 中的所有新功能，请访问 [`www.oracle.com/technetwork/java/javase/8-whats-new-2157071.html`](http://www.oracle.com/technetwork/java/javase/8-whats-new-2157071.html)。

## JavaFX 8 何时可用？

答案是*现在*。如前所述，Java SE 8 于 2014 年 3 月 18 日发布。对于使用 Java 构建客户端应用程序的开发人员来说，JavaFX 富互联网应用框架现在支持 Java 8。

大多数 Java 企业版供应商也支持 Java 8。你是否立即迁移到 Java SE 8 取决于你正在从事的项目类型。

### 注意

事实上，根据 Oracle JDK 支持路线图，2015 年 4 月之后，Oracle 将不再在其公共下载站点上发布 Java SE 7 的进一步更新。

JavaFX API 作为 **Java SE 运行时环境**（**JRE**）和 JDK 的完全集成功能提供。JDK 适用于所有主要桌面平台（*Windows*、*Mac OS X*、*Solaris* 和 *Linux*），因此 JavaFX 也将在所有主要桌面平台上运行。

关于 JavaFX 8，它支持以下 API：

*   3D 图形
*   富文本支持
*   打印 API
