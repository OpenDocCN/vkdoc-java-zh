# 第 4 章 为 Android 开发 JavaFX 应用程序

毫无疑问，我们每天都能看到非 PC 客户端的增长。几乎每个人都至少拥有一部手机或平板电脑，可能来自不同的厂商，但几乎都搭载了 Android 或 iOS 系统——毕竟，在 2014 年售出的智能手机操作系统中，这两者占据了 96%的份额。

如今，智能手机和平板电脑极为流行，并且这些数字每年都在增长。因此，开发者应该考虑掌握为这个巨大市场开发应用程序所需的技能。

正如我们在第 3 章 *开发 JavaFX 桌面和 Web 应用程序*中所看到的，JavaFX 8 已经能够为 Web 和桌面提供丰富的客户端应用程序。但是，如果你像我一样编写过 JavaFX 应用程序，你肯定希望它能尽可能多地运行在各种设备上，以遵循*一次编写，随处运行*的真正精神。借此机会，我想告诉你：是的，我们可以在移动设备上运行 JavaFX。

基于 Oracle 公司对 JavaFX 的官方支持，JavaFX 社区中的许多人正在努力将 JavaFX 移植到尽可能多的设备和平台（桌面、移动和嵌入式）以及不同的操作系统上，并且使用相同的代码库。

他们已经成功创建了 SDK，使我们能够将 JavaFX 应用程序作为原生应用开发，以单个捆绑包（*JVM 加应用程序*）的形式在 Android 或 iOS 设备上运行，无需像桌面或 Web 应用那样额外安装任何软件。

本章将为你提供关于这些 SDK 的必备实践知识，使你能够为 Android 创建、打包和部署原生应用程序。

以下是本章将掌握的一些技能：

*   安装和配置 Android 环境工具及软件
*   准备和创建 JavaFX 8 移动项目结构
*   创建一个 Android JavaFX 8 应用程序
*   JavaFX 8 与 Android 底层 API 的互操作性
*   在移动设备上打包和部署应用程序
*   为最终提交到 Google Play 商店对应用程序进行签名
