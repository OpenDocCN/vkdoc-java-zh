# 1. 什么是 Java EE？

Java 企业版（Java EE），原名 J2EE，其核心是一组*抽象的、标准化的规范*，这些规范规定了软件开发中常见挑战的解决方案。

注意定义中*抽象*这个词很重要。这意味着 Java EE 只是一组接口和契约，为开发者提供了一个面向公众的 API。

这些抽象规范也被称为是*标准化的*。这是什么意思？这意味着整个 Java EE API 集合都是根据该 API 主题领域的专家制定的明确定义标准发布的^(¹)。

标准化还意味着每个 Java EE API 都经过了 Java 社区流程的 Java 规范请求项目的严格流程。这一流程的结果是一套经过行业实践检验、被认为会长期存在的 API。

但是，请记住我之前说过 Java EE 是抽象的，对吧？嗯，尽管您通常会针对 `javax.*` 包中的 Java EE API 进行编码，但您并不能直接在 Java EE 上运行您的应用程序。

## 如何运行 Java EE 应用程序？

要运行 Java EE 应用程序，您需要一个 Java EE 规范的具体实现。请记住，它们是抽象的。Java EE 具体实现的官方名称是*应用服务器*。我敢打赌您以前听过这个说法，对吧？

应用服务器本质上是 Java EE 抽象规范的具体实现。这意味着您可以在应用服务器上使用 Java EE API 实现来运行您的应用程序代码。

市面上有许多应用服务器，包括 Payara Server、Apache TomEE、JBoss Wildfly 和 IBM OpenLiberty 等。
