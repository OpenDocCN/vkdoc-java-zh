# 2. 什么是 Java 规范请求（JSR）？

Java 规范请求的核心，是由个人或组织向 Java 社区进程（JCP）提交的一份正式的、开放的标准文档提案^(²)。它包含了对 Java 技术平台提出的变更、新增或改进建议。

从这个定义中可以提炼出许多关键点。首先，JSR 是一份正式文档。这意味着 JSR 或向 Java 技术体系提交的添加请求必须采用某种预定义的格式。该格式由 JCP 定义。

其次，JSR 是一份开放标准文档。这同样意味着 JSR 是一份符合特定既定规则和条例的文档，这些规则涉及文档的分发与贡献。这也意味着任何有兴趣评估 JSR 的人都可以轻松访问其中包含的内容。

从 Java 规范请求的定义可以得出，JSR 可以由个人或组织提出。JCP 的任何成员都可以提出 JSR。JCP 的成员资格向公众开放；个人也可以免费加入。因此，这意味着如果你不是该组织的成员，就无法向 JCP 提出请求。

最后，JSR 是一份提议对 Java 技术平台进行变更、新增或改进的文档。每个 JSR 都以某种方式包含了对 Java 技术栈的新特性、错误修复或通用改进。

Java EE 上可用的每个主要 API 实际上都是一个经过 JCP 批准流程的 JSR 规范。所有 JSR 都必须经过一个流程才能获得 JCP 的批准。

一旦 JSR 获得 JCP 批准，它就成为 Java 技术栈的一部分，并且可以安全地用于生产环境。JSR 流程确保只有经过充分测试的技术才能成为 Java 技术栈的一部分，从而防止因追逐潮流技术而导致不必要的臃肿。

JSR 流程还确保 API 经过精心设计，以保持向后兼容性。如果说 Java 有一项闻名遐迩的特性，那便是向后兼容性，而 JSR 流程确保了这一关键的 Java 特性得以维持。

由于 JSR 只是一个抽象规范，它需要某种形式的实现才能发挥作用。这就是参考实现（在第 3 章中介绍）概念的由来。
