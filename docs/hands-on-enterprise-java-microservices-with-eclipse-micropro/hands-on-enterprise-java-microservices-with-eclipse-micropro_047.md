# 理解 Eclipse MicroProfile 容错机制

容错机制提供了一系列工具，通过增强代码的韧性来防止其失效。这些工具大多源于优秀的开发实践（例如重试或降级）或知名的开发模式（例如断路器或隔板）。

容错机制基于 CDI，更准确地说，是基于 CDI 拦截器实现。它还依赖 MicroProfile Config 规范，以便为容错策略提供外部配置。

该规范的主要思想是将业务逻辑与容错样板代码解耦。为实现这一目标，规范定义了拦截器绑定注解，用于在方法执行或类上应用容错策略（在这种情况下，该类的所有方法都应用相同的策略）。

容错规范中包含的策略如下：

*   **超时（Timeout）**：通过 `@Timeout` 注解应用。它为当前操作添加一个超时时间。
*   **重试（Retry）**：通过 `@Retry` 注解应用。它为当前操作添加重试行为，并允许对其进行配置。
*   **降级（Fallback）**：通过 `@Fallback` 注解应用。它定义了当前操作失败时要执行的代码。
*   **隔板（Bulkhead）**：通过 `@Bulkhead` 注解应用。它将当前操作中的故障隔离开来，以保护其他操作的执行。
*   **断路器（Circuit Breaker）**：通过 `@CircuitBreaker` 注解应用。它提供自动快速失败执行，以防止系统过载。
*   **异步（Asynchronous）**：通过 `@Asynchronous` 注解应用。它使当前操作变为异步（即代码将被异步调用）。

应用这些策略中的一个或多个，只需在你希望启用这些策略的方法（或类）上添加所需的注解即可。因此，使用容错机制相当简单。但这种简单性并不妨碍其灵活性，这要归功于每种策略提供的所有可配置参数。

目前，以下厂商提供了容错规范的实现：

*   Red Hat 在 Thorntail 和 Quarkus 中
*   IBM 在 Open Liberty 中
*   Payara 在 Payara Server 中
*   Apache Safeguard 用于 Hammock 和 TomEE
*   KumuluzEE 用于 KumuluzEE 框架

所有这些实现都支持容错机制，因此也支持下一节中描述的相同功能集。
