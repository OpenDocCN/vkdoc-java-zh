# 裸机、虚拟机与容器

决定是在裸机、虚拟机还是容器上运行基于 MicroProfile 的微服务或应用，取决于你应用的具体需求。事实上，底层云计算资源类型的确定完全取决于你的应用需求，而不是用于其开发的框架，即 Eclipse MicroProfile。例如，如果你的应用或微服务需要实时或近实时的响应时间，那么你很可能倾向于选择裸机或容器（在裸机上运行）部署。这个决定与你用来编写业务逻辑的框架无关，无论它是 Eclipse MicroProfile 还是其他框架。

由于 Eclipse MicroProfile 支持微服务和基于 HTTP REST 的、与语言无关的通信，你的微服务间通信不会受到微服务运行的底层计算类型的影响；例如，你可以让一个运行在虚拟机上的微服务通过 REST 与另一个运行在裸机上的微服务进行通信。但是，如果你的应用由一部分运行在本地的微服务和另一部分运行在云上的微服务组成（也称为混合云应用），该怎么办？你需要考虑哪些因素？
