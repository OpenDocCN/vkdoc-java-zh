# MicroProfile Metrics

MicroProfile Metrics 公开了运行中服务器的指标数据（通常称为**遥测**），例如 CPU 和内存使用情况以及线程数。这些数据通常被输入到图表系统中，以可视化指标随时间的变化或用于容量规划目的；当然，当数值超出预定义的阈值范围时，它们也用于通知 DevOps 人员。

Java 虚拟机长期以来一直通过 MBean 和 MBeanServer 提供一种公开数据的方式。Java SE 6 为所有虚拟机引入了一种（基于 RMI 的）远程协议，定义了如何从远程进程访问 MBean Server。处理此协议很困难，并且不适合当今基于 HTTP 的交互。

另一个痛点是，许多全球现有的服务器以不同的名称公开不同的属性。因此，设置对不同类型服务器的监控并不容易。

MicroProfile 创建了一个监控规范，通过一个基于 HTTP 的 API（允许监控代理访问）和一个 Java API（允许在一组服务器和 JVM 指标之上导出特定于应用程序的指标）来解决这两个问题。

MicroProfile Metrics 正在开发规范的 2.x 版本，该版本对 1.x 有一些重大更改。以下部分讨论 1.x——2.0 中的更改在 *MP-Metrics 2.0 中的新特性* 部分中讨论。

该规范为指标定义了三个范围：

*   **基础**：这些是每个合规供应商都必须支持的指标，主要是 JVM 统计数据。
*   **供应商**：可选的供应商特定指标，不可移植。
*   **应用程序**：来自已部署应用程序的可选指标。Java API 将在*提供特定于应用程序的指标*部分中展示。

MicroProfile Metrics 解决的经典 JMX 方法的另一个问题是缺乏关于指标语义的元数据信息。
