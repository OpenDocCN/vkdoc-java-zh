# MicroProfile 未来发展

如第 2 章*治理与贡献*所述，提交给 Eclipse MicroProfile 项目的新想法首先会在 MicroProfile 沙箱中尝试，遵循实现优先的创新方法。沙箱演练为实施者和社区提供了讨论、分析和评估这个新想法如何融入 MicroProfile 项目的机会。如果在沙箱演练结束时，社区认为这个新想法值得添加到项目中，则会为其创建一个特定的 MicroProfile 子项目。该子项目必须至少发布一个版本，才能被考虑添加到 MicroProfile 平台版本中。从高层级来看，这就是新想法和未来发展在 MicroProfile 项目下遵循的流程。

在接下来的几节中，我们将讨论两类项目：一类是目前属于 MicroProfile 子项目但位于 MicroProfile 平台版本之外的项目（可以将其视为已从 MicroProfile 沙箱毕业的项目），另一类是仍处于 MicroProfile 沙箱中的项目。最后，我们将讨论 Eclipse MicroProfile 与 Jakarta EE 之间的当前关系，以及它们的路线图是否可能交汇。
