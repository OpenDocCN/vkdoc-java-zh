# 企业级 Java 微服务

应用开发不再仅仅是在你喜欢的操作系统上使用单一的高级编程语言。如今，程序员需要学习大量的语言、规范、框架、专有和开源软件及工具、底层部署基础设施以及开发方法，才能开发现代应用。IT 组织的开发已变得多语言化，即根据特定项目的需求使用多种编程语言。在这个云、容器、微服务、响应式编程、12 要素应用、无服务器、MINI 服务、多语言环境等并存的时代，开发者现在可以选择适合其任务的工具，从而变得更高效、更有生产力。

随着 Java EE 近期以新名称 Jakarta EE 迁移到 Eclipse 基金会，MicroProfile 将在企业级 Java 的未来中扮演非常重要的角色，因为它与 Jakarta EE 具有协同作用，并有可能以多种方式影响它。

云和移动技术的出现，以及开放和**物联网**（**IoT**）的加速发展，催生了数字经济。虽然这开辟了新的市场，但也给企业及其 IT 组织带来了新的要求，它们现在不仅要支持和维护传统工作负载，还要以更快的速度交付新应用。

为了应对这些新需求，许多技术、语言、架构和框架在企业中变得流行起来。其中之一就是微服务，特别是企业级 Java 微服务，因为 Java 仍然是 IT 公司中最流行的语言之一。但什么是企业级 Java 微服务呢？

企业级 Java 微服务具有以下特征：

*   使用 Java 语言编写。
*   可以使用任何 Java 框架。
*   可以使用任何 Java API。
*   必须达到企业级标准，即可靠、可用、可扩展、安全、健壮且高性能。
*   必须满足 [`martinfowler.com/microservices/`](https://martinfowler.com/microservices/) 中列出的微服务架构特征，具体如下：
    *   通过服务实现组件化
    *   围绕业务能力组织
    *   产品而非项目
    *   智能端点与哑管道
    *   去中心化治理
    *   去中心化数据管理
    *   基础设施自动化
    *   为故障而设计
    *   演进式设计
