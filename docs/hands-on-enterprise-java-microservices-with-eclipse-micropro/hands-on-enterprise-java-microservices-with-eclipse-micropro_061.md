# 理解健康检查以及 MicroProfile 如何处理它们

在云原生架构中，健康检查用于确定计算节点是否存活并准备好执行工作。就绪状态描述了容器启动或滚动更新（即重新部署）时的状态。在此期间，云平台需要确保在该实例准备好执行工作之前，没有网络流量被路由到该实例。

另一方面，存活状态描述了正在运行的容器的状态；也就是说，它是否仍然能够响应请求？如果存活状态或就绪状态被视为无效，计算节点将被丢弃（终止或关闭），并最终被另一个健康的实例替换。

健康检查是与云平台的编排框架和调度程序的基本契约。检查过程由应用程序开发人员提供，平台使用这些过程来持续确保你的应用程序或服务的可用性。

MicroProfile 健康检查 1.0（MP-HC）支持一个可用于存活检查或就绪检查的单一健康检查端点。MicroProfile 健康检查 2.0 计划增加对多个端点的支持，以允许应用程序定义存活探针和就绪探针。

MP-HC 规范详细说明了两个元素：一个协议及其响应有线格式部分，以及一个用于定义响应内容的 Java API。

MP-HC 功能的架构被建模为一个应用程序，该应用程序由零个或多个健康检查过程组成，这些过程通过逻辑上的 `AND` 连接起来，以推导出整体健康检查状态。一个过程代表一个应用程序定义的对所需条件的检查，它具有名称、状态以及可选的检查数据。
