# 默认 ConfigSource

默认情况下，MicroProfile Config 实现必须提供三个配置源：

*   Java 虚拟机中的系统属性（序数为 `400`）
*   环境变量（序数为 `300`）
*   存储在 `META-INF/microprofile-config.properties` 中的属性（序数为 `100`）

配置源的序数值决定了配置源的优先级。特别是，如果一个属性同时在系统属性和环境变量中定义，则该值将从系统属性中获取（其序数高于环境变量）。

对属性名称没有限制。但是，某些操作系统可能会对环境变量的名称施加一些限制（例如，大多数 Unix shell 不允许使用 `"."`）。如果您有一个可能通过环境变量配置的属性，则必须相应地命名您的属性。

例如，属性名称 `my_url` 可以被环境变量使用，而 `my.url` 则不能。

**MicroProfile Config 1.3 中的新特性** MicroProfile Config 1.3 引入了从配置属性名称到环境变量的映射规则。此规则为每个属性名称搜索三种环境变量变体：

*   精确匹配
*   将所有非字母数字字符替换为 `_`
*   将所有非字母数字字符替换为 `_` 并使用大写

这意味着，在 Java 应用程序中，我们可以有一个名为 `app.auth.url` 的属性，并使用 `APP_AUTH_URL` 环境变量来配置它。
