# 响应式编程与未来发展

事件驱动架构已经存在了很长时间，异步方法调用、消息驱动 Bean、事件控制逻辑等结构是开发人员熟悉的。然而，随着云资源的普及和按需可扩展性的采用，组织对能够利用无服务器和函数即服务类型环境的响应式编程方法重新产生了兴趣。Eclipse MicroProfile 还包括与响应式编程相关的规范，这些规范目前位于 Eclipse MicroProfile 伞形/平台版本之外。

除此之外，Eclipse MicroProfile 沙盒中还有一些项目，社区正在讨论、实施和评估这些项目，以决定是否应将它们提升为正式的 MicroProfile 项目。本章将帮助您了解当前与响应式编程相关的 MicroProfile 规范，并让您一窥伞形/平台版本之外以及 MicroProfile 沙盒中的项目已经启动和即将到来的内容。本章将涵盖以下主题：

*   响应式消息传递概述
*   Eclipse MicroProfile 中响应式消息传递架构的解释
*   与响应式编程相关的 Eclipse MicroProfile 规范描述
*   如何使用 Eclipse MicroProfile 响应式消息规范的示例
*   伞形或平台版本之外的 MicroProfile 项目/规范概述
*   Eclipse MicroProfile 沙盒内项目的描述
*   对 Eclipse MicroProfile 与 Jakarta EE 当前关系的见解及其可能的未来分析
