# 健康检查协议与线路格式

MP-HC 规范定义了支持对逻辑上的 `/health` REST 端点发起 HTTP GET 请求的要求，该端点可能返回以下任一状态码来表示端点状态：

*   `200`：端点正常运行且健康。
*   `500`：端点因未知错误而不健康。
*   `503`：端点已宕机，无法响应请求。

请注意，许多云环境仅将请求返回码视为成功或失败，因此可能无法区分 `500` 和 `503` 状态码。

`/health` 请求的负载必须是一个 JSON 对象，该对象需符合下面给出的模式（有关 JSON 模式语法的更多信息，请参见 [`jsonschema.net/#/`](http://jsonschema.net/#/)）。
