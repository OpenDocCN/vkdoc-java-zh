# 多速 IT

尽可能快地实现和交付应用并非新要求。事实上，自第一台计算机发明以来，提高效率一直是计算机科学家们思考的问题。高级编程语言、封装、可重用性、继承、事件驱动设计、SOA、微服务、机器学习和人工智能，这些都是为了应对更快完成任务这一挑战而提出的概念。随着每一波新技术的浪潮，变速箱都为软件开发和交付方式的演进增加了一个新的速度要求。数字经济又为变速箱增加了一个高速档位。

企业需要适应数字经济的新需求。它们不仅要创建、运行和支持传统类型的应用，还要创建符合数字经济新需求的应用。它们必须同时支持瀑布式和 DevOps 流程、混合云基础设施以及 SOA 和微服务架构。

这给 IT 组织带来了许多挑战，因为其流程、人员和技术都一直面向传统类型应用的实现、交付和维护。许多组织已经踏上或正在开始其数字化转型之旅，以应对数字经济的挑战。这一旅程包括在应用的开发、交付、集成和维护方面，对技术、框架、语言和流程进行变革。

无论你称之为双模 IT（[`www.gartner.com/it-glossary/bimodal`](https://www.gartner.com/it-glossary/bimodal)）还是业务技术战略（[`go.forrester.com/wp-content/uploads/Forrester-False-Promise-of-Bimodal-IT.pdf`](https://go.forrester.com/wp-content/uploads/Forrester-False-Promise-of-Bimodal-IT.pdf)），事实是，IT 需要比以往任何时候都更快地满足业务需求，无论是现有应用还是新应用。这意味着 IT 还需要加快现有应用的维护和交付，同时专门为新应用采用敏捷方法论。然而，这并不排除在处理现有应用与新应用时，仍然需要使用不同的开发流程、发布周期和支持时间线，这实际上就是 IT 的多速特性。
