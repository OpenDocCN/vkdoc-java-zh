# 那十二要素应用呢？

与微服务和微服务架构的定义一样，十二要素应用并未规定底层技术，例如编程语言、数据库、缓存等，或用于实现它们的框架。十二要素应用是一种用于实现应用的方法论。这十二个要素如下：

*   一份在版本控制中追踪的基准代码，多次部署
*   显式声明并隔离依赖关系
*   在环境中存储配置
*   将后端服务视为附加资源
*   严格分离构建和运行阶段
*   将一个或多个无状态进程的形式执行应用
*   通过端口绑定导出服务
*   通过进程模型进行横向扩展
*   通过快速启动和优雅关闭最大化健壮性
*   尽可能保持开发、预发布和生产环境相似
*   将日志视为事件流
*   将管理/管理任务作为一次性进程运行

使用此方法论实现应用有助于我们实现以下目标：

*   最小化新加入项目的开发人员的时间和成本
*   提供执行环境之间的可移植性

*   轻松将应用部署到云平台
*   最小化开发与生产环境之间的差异
*   无需更改即可进行扩展

您可以在 [`12factor.net`](https://12factor.net) 阅读关于这十二个要素的全部内容。

十二要素应用是开发人员在设计和实现微服务及应用时可以遵循的一种方法论，与用于实现它们的编程语言或框架无关。开发人员可以使用 Eclipse MicroProfile 作为框架，通过十二要素应用来实现微服务。十二要素应用和 Eclipse MicroProfile 并非互斥，而是相辅相成。
