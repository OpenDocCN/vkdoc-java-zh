# 第 3 章

1.  配置属性的默认来源是环境变量、Java 系统属性和 `META-INF`/`microprofile-config.properties` 文件。
2.  您可以提供一个自定义的 `ConfigSource` 实现，将属性名称映射到自定义源中的值。
3.  字符串并非唯一支持的类型，因为 MP-Config 通过可插拔的 SPI 支持类型转换，并默认提供几种默认转换。
4.  如果您已经指定了 `defaultValue`，或者将属性注入为 `Optional<?>` 值，则无需为注入的属性提供值。
5.  可以使用自定义的 `Converter<?>` 实现来处理复杂的属性类型，该实现接收一个字符串并返回复杂类型。
