# 重试

容错中的重试功能允许你在首次请求失败时，重试向另一个服务发送请求（更多信息请参见第 3 章，*MicroProfile 配置与容错*）。现在，假设你有以下代码：

```
@Retry (maxRetries = 3)
void aMethod() {
    callBackend();
}
```

尽管这告诉 Istio 重试 5 次，但在 `aMethod` 最终以错误结束之前，你可能会得到 15 次重试（Istio 会在你的代码中的 3 次重试中，每次重试 5 次，然后才返回错误）。你可以考虑在代码中关闭重试，因为 Istio 中的重试次数可以在运行时动态更改，无需重启 Pod。
