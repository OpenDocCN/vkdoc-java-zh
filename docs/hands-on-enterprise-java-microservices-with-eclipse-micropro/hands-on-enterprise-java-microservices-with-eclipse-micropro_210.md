# 总结

在本章中，你学习了 Eclipse MicroProfile 如何促进云原生应用开发，无论其是混合云还是多云应用。我们还讨论了微服务、云原生开发和容器原生开发之间的关系，以及运行在容器上的微服务如何成为云原生开发的理想选择。你还了解了 Eclipse MicroProfile 与 12 要素应用以及无服务器和 FaaS 环境的关系。

我们涵盖了引导你进入云原生应用之旅的八个步骤，以及 Eclipse MicroProfile 如何在每个步骤中提供帮助。此外，我们还讨论了在跨云应用中使用 Eclipse MicroProfile 时需要考虑的事项，何时在裸机、虚拟机与容器上运行 Eclipse MicroProfile，在混合云应用中运行 Eclipse MicroProfile 微服务时需要考虑的事项，在多云部署中运行 Eclipse MicroProfile OpenTracing 时需要注意的挑战，以及最后在服务网格中使用 Eclipse MicroProfile 时需要考虑的事项。

在本书中，我们介绍了 MicroProfile（一个 Java 微服务规范）的起源及其诞生的历史背景。我们介绍了这个开源项目，包括其使命、治理、优势、如何为其做出贡献，以及其子项目的生命周期。然后，我们深入探讨了构成伞式/平台发布的每个 MicroProfile API/子项目，以及伞式发布之外的子项目。

我们还介绍了市场上 MicroProfile 的当前实现，包括 Quarkus，并向你展示了如何“quark”一个由 MicroProfile Starter 生成的 MicroProfile 项目。我们讨论了 Conference Application，这是一个社区示例项目，展示了 MicroProfile 在不同供应商实现之间的互操作性。我们还提供了贯穿全文的代码示例以供参考和讨论，并提供了一个包含源代码的完整工作项目，该项目实现了所有 MicroProfile API，你可以自由访问、下载、编译并在开发工作中重用，从而快速上手 MicroProfile。随后，我们讨论了 MicroProfile 提供的响应式功能及其未来发展，例如正在开发中的子项目和 MicroProfile 沙箱中的项目，以及其未来与 Jakarta EE 可能的关系。

最后，我们回顾了基于 MicroProfile 的应用和微服务如何非常适合在容器、云和无服务器/FaaS 部署中实现解决方案。无论你是 Java 微服务的新手还是经验丰富的 Java 开发者，你都可以利用从本书中获得的所有知识，开始开发基于这一新颖且由社区驱动的规范的微服务，以创建可观察、可扩展、安全且高可用的混合云和多云应用，从而更快地为你的业务创造价值。
