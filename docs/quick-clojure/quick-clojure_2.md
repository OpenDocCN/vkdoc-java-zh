# 2. 数据结构和语法

Clojure 提供了一种基于选定数据集合的语言 API。这意味着某些编程功能的语法与底层数据结构相匹配。

例如，以下是 Clojure 提供的主要数据结构：

*   列表：`(1 2 3)`
*   向量：`[1 2 3]`
*   映射：`{:foo "bar"}`
*   集合：`#{1 2 3}`

注意

还有其他数据结构、术语和相关概念，以及一份很好的参考指南，可以在官方 Clojure 文档中找到。¹ 然而，就我们本章要涵盖的内容而言，它有时可能有点“底层”，因为它还讨论了 Clojure 使用的一些底层 Java 接口。

Clojure 的数据结构是只读的（也称为不可变的），这意味着当你对数据结构中存储的值进行更改时，会返回该数据结构的一个新版本。我们稍后会回到这个概念。

所有数据结构都可以混合和嵌套。参见清单 2-1。

```
[1 2 [3 {:name "Mark"}]]
清单 2-1.
嵌套数据结构示例
```

注意

你会看到各个元素之间没有分隔符。如果你愿意，可以添加它们，但它们是可选的，并且地道的 Clojure 代码通常会省略它们（例如，`[1 2 3]` 可以写成 `[1, 2, 3]`）。

你也可以将所有这类数据结构称为序列。序列是 Clojure 定义的一个接口，它允许支持该接口的多种数据结构共享对一组内置函数的访问。我们将在后面的章节中更详细地讨论序列这个主题。

Clojure 编程语言中有许多函数实际上被设计为可以处理多种类型的数据结构。这在许多其他语言中并不常见，因此它为我们代码提供了极大的灵活性和适应性。

现在，让我们更深入地逐一审视这些数据结构，看看我们可以使用 Clojure 核心库中的哪些函数来操作它们。我们从列表数据结构开始，因为它涉及的内容最为复杂。



## 列表

为了理解列表数据结构在 Clojure 这类语言中的重要性，我们需要知道 Clojure 是 Lisp 的一种方言，而 Lisp 代表“LISt Processing（列表处理）”。Lisp 既是一种编程语言（最初设计于 1958 年），也是一个公认的编程语言集合的结构基础（Common Lisp、Scheme 和 Clojure 就是基于 Lisp 相同语法构建的几个流行语言示例）。

简单来说，当一种语言实现了带括号的前缀表示法语法时，它就可以被视为 Lisp 的一种方言。这种表示法的独特之处在于，括号用于对逻辑进行分组，并且在括号内，运算符位于其操作数的左侧。

在实际应用中，当我们看 Clojure 这样的语言时，前缀表示法意味着在括号内，首先指定的是要执行的函数，后面跟着该函数接受的任何参数。如果我们观察列表数据结构，会发现其语法与前缀表示法相同。这就是为什么列表数据结构在所有 Lisp 语言中都很普遍，因为它是原始语言设计的基本组成部分。

列表使用括号作为其边界分隔符，因此一个空列表看起来像 `()`，而一个包含三个元素的列表可能看起来像 `("a" "b" "c")`。

基于 Lisp 的语言（如 Clojure）的强大之处在于，其语言 API 使用了与其操作的基础数据结构相同的语法。这与其他语言不同，其他语言的 API 设计通常是在其操作的基础数据结构之上添加大量语法糖。

API 语法与基础数据结构之间的这种相似性，使得 Clojure 编程语言能够重写自身，并实现其最初未设计的功能（我们稍后在学习 Clojure 的宏功能时会看到如何实现这一点）。但本质上，这就是“代码即数据”这一同像性概念的含义，正如第 1 章开头所提到的。

既然你已经认识到基于 Lisp 的语言的编程接口如何使用与基础数据结构相同的语法，你可能已经注意到 Clojure 执行函数的语法恰好也是一个列表数据结构。例如，`(+ 1 2)` 是一个产生 `1 + 2` 结果的函数调用，但 `(+ 1 2)` 本身也是一个列表数据结构。

当 Clojure 评估文本 `(+ 1 2)` 时，它会将各个部分进行词法分析并评估。在这个例子中，它被提供了一个列表数据结构，当提供这种特定数据结构时，Clojure 语言解释器期望该结构中的第一个元素是一个函数，其余元素是传递给该函数的参数。

因此，在我们的示例中，我们提供了 `+` 函数和两个参数。第一个参数是值 `1`，第二个参数是值 `2`。我们稍后会学习更多关于函数的知识，但现在你应该知道调用函数的语法结构如代码清单 2-2 所示。

```
(fn arg1 arg2 arg3...)
代码清单 2-2.
调用函数的语法结构
```

注意

使用 REPL 时，你可能没有意识到，但 `clojure.core` 命名空间（以及其中包含的所有函数）已经为你预加载了。这就是 `+` 函数的来源。我们将在后面的章节中介绍命名空间。

在 Clojure 中，列表被实现为链表，² 这意味着如果你想遍历其内容，你必须从列表的开头开始，然后逐个移动到下一个，直到到达列表的末尾。这就是链表数据结构工作的本质。

列表在将新元素添加到其开头时非常高效，但在将新元素添加到其末尾时则不太擅长。这是因为如果列表非常长，那么逐个遍历列表中的每个元素以找到末尾可能会花费一些时间。

现在，在大多数使用 Clojure 编程的情况下，你不必担心某些数据结构是如何“在底层”实现的。Clojure 会很乐意为你隐藏这些细节，并提供抽象，使得以最合适和最高效的方式处理数据结构变得非常容易。

话虽如此，我个人认为了解数据结构的底层实现很重要，因为它能让我们做出决策，从而优化代码性能（当我们真正需要时）。这也意味着我们不会因为试图理解为什么某个函数没有提供我们期望的结果而挠头。

如果你需要向链表数据结构添加一个值，那么你有几个选择。你可以通过使用 `cons` 函数来明确表达你对列表底层实现的理解（即，你现在知道列表数据结构是作为链表实现的），该函数会将你的值插入到列表的开头，如代码清单 2-3 所示。

```
(cons 4 '(1 2 3))
;; (4 1 2 3)
代码清单 2-3.
使用 cons 明确进行数据结构操作
```

或者，你可以使用像 `conj` 函数这样的抽象，它会根据底层数据结构选择正确的方法来插入新值。例如，如果你的数据结构是一个列表（作为链表实现），那么 `conj` 会将值放在列表的开头，如代码清单 2-4 所示。

```
(conj '(1 2 3) 4)
;; (4 1 2 3)
;; 与更明确的 cons 相同
代码清单 2-4.
在列表上使用 conj 抽象函数
```

但是，如果你的数据结构是向量集合（我们将在下一节中介绍向量），那么 `conj` 函数会知道将值插入到集合的末尾（这在处理向量时性能更高），如代码清单 2-5 所示。

```
(conj [1 2 3] 4)
;; [1 2 3 4]
代码清单 2-5.
在向量上使用 conj 抽象函数
```

有些读者可能想知道列表数据结构前面的单引号（即 `'(1 2 3)`）是什么意思；我们将在讨论 Clojure 的宏时更详细地讨论这一点，但它实际上是为了防止 Clojure 将列表作为函数进行评估，而是让 Clojure 将其视为字面列表数据结构。

否则，如果没有单引号，正如我们之前提到的，Clojure 会尝试将列表作为函数执行，就好像第一个参数是一个函数一样；因此，它会尝试定位并执行一个标识符为 `1` 的函数，并将值 `2` 和 `3` 作为参数传递给该函数。但不存在名为 `1` 的函数，因此会发生错误。

列表数据结构是唯一需要担心引用的结构。或者，你可以使用 `list` 函数动态创建列表。例如，`(cons 4 (list 1 2 3))`。

还有其他抽象函数（如 `conj`），例如 `peek` 和 `pop`，它们有助于确定应用于数据结构的正确“过程”。参见代码清单 2-6。

```
(peek [1 2 3])  ;; 3
(peek '(1 2 3)) ;; 1
(pop [1 2 3])  ;; [1 2]
(pop '(1 2 3)) ;; (2 3)
代码清单 2-6.
peek 和 pop 抽象函数
```

注意 `peek` 和 `pop` 方法如何根据所提供的数据结构以不同方式处理事情。它知道列表（即 `'(1 2 3)`）是一个链表，因此从数据结构的开头处理；而它知道向量（即 `[1 2 3]`）具有“索引访问”，因此从数据结构的末尾处理性能更高。



## 向量

你当前使用的编程语言很可能包含一种类似于向量的数据结构。在大多数其他语言中，它被称为数组（尽管在语义和实现上存在细微差异）。向量允许你通过索引访问数据结构中的任意元素。

举个例子，假设我们有一个包含五个元素的向量，想要获取第四个元素。请参见清单 2-7。

```
(get [1 2 3 4 5] 3)
;; 4
清单 2-7.
演示向量数据结构的索引访问
```

如你所见，向量是零索引的，因此请求 `3` 会返回第四个元素。你也可以使用 `nth` 函数代替 `get`。区别在于后者（`get`）适用于所有类型的集合（包括映射和集合），而 `nth` 仅适用于向量和列表。

你可以使用 `assoc` 函数（“associate”的缩写）来修改向量。其工作方式是：提供要修改的向量索引，然后提供新值（参见清单 2-8）。

```
(assoc [1 2 3 4 5] 5 6)
;; [1 2 3 4 5 6]
清单 2-8.
将值关联到向量中
```

在上面的例子中，我们表示“将值 6 放入索引 5”。请记住向量是零索引的，因此我们的新值 `6` 实际上会被添加到向量的末尾（因为当前不存在第五个索引）。

注意

如果你想在不指定索引的情况下向集合中添加值，请使用前面演示过的 `conj`。

添加值固然不错，但如果你想删除值呢？一种方法是使用 `pop` 函数，它会返回向量的副本，但会移除最后一个元素。例如，请参见清单 2-9。

```
(pop [1 2 3 4 5])
;; [1 2 3 4]
清单 2-9.
返回移除了最后一个元素的向量副本
```

那么，如果你想从向量中间删除元素呢？你应该意识到，在使用基于 Lisp 的语言（如 Clojure）时，这需要相当复杂的步骤。

你还应该意识到，如果你计划从向量中间删除元素，向量可能不是最佳的数据结构。对于需要以复杂方式操作数据的情况，你可能更希望使用哈希映射（详情请参见后面的“映射”部分）。

在探讨如何从向量中间删除元素之前，我们先从类似但更容易实现的操作开始。假设你想对数据进行采样。你可以通过获取向量的一个“切片”来实现。例如，如果向量是 `[1 2 3 4 5]`，并且你想要一个连续的样本，比如 `[2 3]`，你可以使用 `subvec` 函数，如清单 2-10 所示。

```
(subvec [1 2 3 4 5] 1 3)
清单 2-10.
使用 subvec 获取数据切片
```

在清单 2-10 中，代码描述为“从索引 1（在给定向量中对应值 `2`）开始，给我一份从该处到索引 3 的副本”，因此我们得到了那个特定范围 `[2 3]`。

注意

稍后我们会看到，还有更多用于处理序列/集合的函数，并且由于 Clojure 实现了序列抽象，我们可以在不同的数据结构上利用这些函数。

回到之前关于如何从向量中间删除元素的问题，清单 2-11 中的代码片段演示了如何使用 `subvec` 从集合 `[1 2 3 4 5]` 中删除数字 `3`，最终得到向量 `[1 2 4 5]`。

现在，重要的是要意识到我将要演示一个相当不优雅的解决方案。但一旦你更熟悉 Clojure 及其可用函数后，你会发现这很容易被优化。为了可读性，我还会将这个不优雅的解决方案扩展到多行。

```
(vec
(concat
(subvec [1 2 3 4 5] 0 2)
(subvec [1 2 3 4 5] 3)))
清单 2-11.
使用 subvec 获取数据切片
```

其工作原理是：我们使用 `subvec` 从向量的两端各取一个切片。因此 `(subvec [1 2 3 4 5] 0 2)` 返回 `[1 2]`，而 `(subvec [1 2 3 4 5] 3)` 返回 `[4 5]`。然后我们使用 `concat` 函数将这两个向量连接起来。最后，由于 `concat` 返回的是列表数据结构，我们使用 `vec` 函数将返回的列表转换回向量。

`concat` 函数返回列表而非向量这一事实其实不是什么大问题，因为 Clojure 能够将集合作为抽象来处理。但在某些情况下，确保返回的数据结构是特定格式可能很有用，这样你可以利用特定的性能优势。

对于这个基本示例，我无需担心这一点。我在示例中仅将结果转换回向量类型，是为了避免结果类型与输入类型不同时造成混淆。

我们将在后面的章节中讨论为什么会出现从向量到列表的转换，但目前只需知道我们不必过于担心集合类型之间的转换，是因为一个称为“序列抽象”的概念。

一个更灵活且可复用的解决方案是创建一个函数来封装其复杂的实现逻辑。下面的示例代码（如清单 2-12 所示）是一个更灵活的解决方案，其逻辑被封装在我们自己定义的函数中，我们将该函数命名为 `drop-nth`。

我们函数的签名将是 `(drop-nth <n> <collection>)`，一个示例调用是 `(drop-nth 3 [:a :b :c :d])`。预期结果是向量中的第三个元素（`:c`）被移除。

在我演示自定义 `drop-nth` 函数的实现之前，我建议如果你对编程和/或 Clojure 还不熟悉，或许可以先跳到下一节：“映射”。

提供以下示例是为了演示如何解决之前需要从向量中间删除元素的问题。但该解决方案确实利用了 Clojure 语言的几个不同的高级特性。我建议你继续阅读，并在有机会了解更多关于该解决方案所利用的 Clojure 特性后，再回来查看这个示例。

```
(defn drop-nth [n coll]
(->> coll
(map vector (iterate inc 1))
(remove #(zero? (mod (first %) n)))
(map second)))
(drop-nth 3 [:a :b :c :d])
;; (:a :b :d)
清单 2-12.
从向量中间移除元素
```

那么，我将尝试分解这个示例，但再次强调，目前不必过于担心真正理解这些活动部件，因为我们将在后面的章节中学习每个特性。

首先要提到的是 `->>` 的使用。这看起来像一个函数，但严格来说它实际上是一个宏。这个宏的名字是“线程最后宏”。虽然名字中包含“线程”一词，但这个宏与并发或多线程代码无关（并发和多线程代码涉及多个任务重叠以及同时执行多个任务，我们将在后面的章节中详细讨论）。

宏 `->>` 将值（本例中为 `coll`）作为最后一个参数传递给调用中列出的每个形式。

注意

在 Clojure 中，“形式”基本上就是一个“表达式”，它是可以被求值以产生值的有效代码。


好的，作为高级文档工程师和翻译员，我将严格遵循您提供的注意事项和示例格式，将给定的英文文本翻译成中文。


因此，集合 `[:a :b :c :d]` 作为参数/符号 `coll` 被传入 `drop-nth` 函数，同时也作为最后一个参数被传入 `(map vector (iterate inc 1))`。这意味着一旦 Clojure 编译了这个形式/表达式，它最终看起来会像 `(map vector (iterate inc 1) coll)`。

前一个 `map` 形式的结果本身会作为最后一个参数被传入 `(remove #(zero? (mod (first %) n)))`。

好的，最后一个形式看起来相当复杂。我们可以看到我们正在调用一个名为 `remove` 的函数，但它后面作为参数传入的所有东西是什么呢？嗯，那是一个“匿名”函数。

当我们定义 `drop-nth` 函数时，我们给它取名为 `drop-nth`。但函数并不需要名字才能执行，因此为了方便起见，我们没有定义一个在程序生命周期内存在的函数，而是定义了一个内联/匿名函数，用于快速处理一些数据，并将该结果作为第一个参数传递给 `remove` 函数。

所以，回顾那个形式，我们可以看到它实际上只是在调用 `remove` 函数，并向它传递两个参数：第一个是一个匿名函数 `#(zero? (mod (first %) n))`，第二个是前一个 `map` 形式的结果（由于 `->>` 宏，你不会显式地看到它作为第二个参数被传递）。

最后，前一个 `remove` 形式的结果作为最后一个参数被传入 `(map second)`。如果我们没有使用线程末位宏，那么我们将被迫编写如清单 2-13 所示的一组（更复杂的）嵌套形式（注意函数的顺序现在是相反的，并且是“由外向内”编写的）。

```
(defn drop-nth [n coll]
(map second
(remove #(zero? (mod (first %) n))
(map vector (iterate inc 1) coll))))
(drop-nth 3 [:a :b :c :d])
;; (:a :b :d)
清单 2-13.
未使用线程末位宏的解决方案
```

现在，我们的最终目标是将所有这些复杂逻辑封装在一个名为 `drop-nth` 的用户自定义函数中，这意味着无论我们在哪里需要从集合中移除一个元素，它都可以被重用。

上述代码工作的基本原理是，你提供的集合被转换成一个（稍微）类似于哈希映射的结构（我们将在下一节“映射”中讨论）。

例如，集合 `[:a :b :c :d]` 会被转换成类似 `([1 :a] [2 :b] [3 :c] [4 :d])` 的形式。你可以在 REPL 中自己尝试一下（见清单 2-14）。

```
(map vector (iterate inc 1) [:a :b :c :d])
;; ([1 :a] [2 :b] [3 :c] [4 :d])
清单 2-14.
将集合转换为带有数字索引的集合
```

我将在下一章解释 `map` 函数是如何工作的，但现在只需知道它会遍历给定的集合，并将集合中的每个元素传递给一个可以修改其值的函数就足够了。这为我们提供了一个访问集合中特定元素的数字钩子。

下一行代码 `(remove #(zero? (mod (first %) n)))` 是对 `remove` 函数的调用。我们现在知道我们传递的是一个匿名函数的结果。这个匿名函数正在执行谓词函数 `zero?`。

注意

“谓词”听起来比实际复杂。谓词就是一个返回真或假的函数。

传递给 `zero?` 的参数是来自集合 `([1 :a] [2 :b] [3 :c] [4 :d])` 的一个元素。`remove` 函数针对提供的集合中的每个元素执行 `zero?`。你可以通过将前面的代码片段与前面片段的输出结合起来自己尝试一下，如清单 2-15 所示。

```

'([1 :a] [2 :b] [3 :c] [4 :d]))
;; ([1 :a] [2 :b] [4 :d])
清单 2-15.

```

在清单 2-15 中，你会注意到我将 `drop-nth` 实现中的 `n` 替换成了值 `3`。我这样做只是为了演示这个提取出的代码片段是如何工作的。我还需要在代码片段中引用列表 `'(...)`；否则，会导致错误。

```

使用原始形式从集合中移除元素
```

好的，我们有点偏离了重点。回到我们之前说的，第一个元素是 `[1 :a]`，在这种情况下，匿名函数可以通过 `%` 符号访问它。它检索数字 `1` 并将其除以 `n`（在前面的例子中，这个数字是 `3`，它最初是作为参数传递给 `drop-nth` 的）。它进行这个计算是为了验证这些数字是否能被整除；如果能，那么结果将是零，并且谓词（即 `zero?` 函数）将返回 `true`，这意味着该元素应该从集合中移除。

集合中唯一能被整除并得到模数为零的元素是 `[3 :c]`。这意味着 `remove` 函数的整体结果是 `([1 :a] [2 :b] [4 :d])`。

现在，我们已经成功移除了所需的元素。但我们仍然需要将我们的数据结构转换回其原始形式。为此，我们转到函数的最后一行 `(map second)`，它映射结果集合（该集合仍然是一个子向量集合——`([1 :a] [2 :b] [4 :d])`），并通过专门检索“第二个”元素，从每个嵌套集合中移除第一个元素，从而返回一个修改后的版本。

因此，第一个嵌套集合是 `[1 :a]`，这意味着 `(map second)` 将移除 1 并保留 `:a`。整个函数的结果是 `(:a :b :d)`，这正是我们最初想要的——能够从集合中移除一个既不是开头也不是结尾位置的特定元素。

现在，当前 `drop-nth` 实现的一个问题是，它实际上会移除每一个第 n 个元素。所以，如果你执行 `(drop-nth 3 [:a :b :c :d :e :f :g :h :i :j])`，那么结果将是 `(:a :b :d :e :g :h :j)`。注意它不仅仅移除了第三个元素，而是移除了每一个第三个元素。这可能不是你希望发生的情况。

解决方案是，记住问题是从集合中间移除单个值，将 `#(zero? (mod (first %) n))` 替换为 `#(= (first %) n)`，这将导致所有其他迭代失败，因为只有 `3` 会“等于”第三个元素（注意我们将 `zero?` 谓词替换为了 `=` 函数）。

在我们继续之前，除了使用 `remove`，我们还可以使用另一个名为 `filter` 的函数（我将在下一章演示 `filter`），但我认为从语义上讲 `remove` 更有意义。这两个函数之间的唯一区别在于谓词的使用。

`filter` 函数会在其谓词返回 `true` 时保留元素（该函数的语义是“过滤出我想要的元素”）。`remove` 函数会在其谓词返回 `false` 时保留元素（该函数的语义是“移除通过测试的元素”）。

### 关键字

有些读者可能想知道键前面的冒号是什么意思（例如，我们在 `:foo` 和 `:baz` 中看到的）。冒号表示该键实际上是一个关键字。如果你在 Ruby 中使用过符号，那么这其实是相同的东西。如果没有，那么要知道它的使用主要是出于性能原因。

详细来说，如果你使用字符串创建映射键（例如，`{"my-key" "my key value"}`），那么每次你使用字符串引用该键时，都会在内存中重新创建它，而通过关键字引用键则只会创建一次。

为了更好地理解这一点，想象我们有一个如清单 2-22 所示的向量，其中每个元素的值都相同。

```
["hi" "hi" "hi"]
;; 在内存中创建了三次字符串 “hi”
[:hi :hi :hi]
;; 值 hi 在内存中只创建一次
清单 2-22.
演示关键字的性能
```

因此，我们可以看到指定一次关键字，并在代码中找到该关键字的地方重用底层内存分配所带来的性能优势。这就是为什么你会看到大多数时候使用关键字而不是字符串（除非你有非常特定的字符串需求）。

注意

你可以使用 `keyword` 函数将字符串转换为关键字：`(keyword "my-string")`，它返回 `:my-string`。反向操作也可以使用 `name` 函数完成：`(name :my-string)`，它返回 `"my-string"`。

关键字的使用还有一个重要的补充——如果你将它们用作映射内部的键，那么该关键字的作用就像一个 `get` 函数。

因此，例如，假设你有一个映射 `{:foo "bar" :baz "qux"}`，并且你想要从 `:baz` 键获取值。你可以使用清单 2-23 中所示的任一代码片段来获取该值。

```
(get {:foo "bar" :baz "qux"} :baz)
;; "qux"
(:baz {:foo "bar" :baz "qux"})
;; "qux"
清单 2-23.
关键字作为函数使用

在清单 2-23 中，我们像之前一样使用了熟悉的 `get` 函数来检索指定键的值，但我们也可以直接在期望函数的位置使用关键字，它会返回该键的值。如果你不确定你的映射是否包含某个键，那么你可以使用 `contains?` 函数，如清单 2-24 所示。

```
(contains? {:foo "bar" :baz "qux"} :foo)
;; true
清单 2-24.
演示 contains? 函数
```

```

注意

使用 `def` 函数是为了声明一个变量。我们稍后会再讨论变量，简而言之，它们定义了内存中一个可以存储数据的位置；然后我们可以通过变量名来引用该数据。

在前面的例子中，你会注意到我重复输入了键 `:name` 和 `:age`（实际上，我想要记录的属性会多得多）。这既浪费时间，又增加了因拼写错误导致恼人 bug 的可能性，更不用说在其中一个映射中添加新键却忘记添加到其他映射中这种不一致的情况了。

## 变量与符号

好了，这是在继续之前我想介绍的最后一个基本语法部分。在本节中，我不仅想演示变量是如何工作的（你从其他编程语言中应该已经很熟悉这个概念了），还想演示符号如何与变量关联，以及 `def` 函数和 `defn` 宏的使用。

让我们从理解什么是符号开始。符号，就其最简单的形式而言，是对某个其他值的引用。它是一种标识，就像你的名字；它标识了你，尽管你随着时间的推移而变化（你变老了，对吧？），但你的身份保持不变。

另一方面，变量是一个可变的存储位置。创建变量时，它会在当前命名空间内创建（这可以防止与其他命名空间冲突）。要在 Clojure 中创建变量，你需要使用 `def` 函数，以便将值赋给一个符号。参见列表 2-35。

```
(def foo "hello")
列表 2-35.
使用 def 将值赋给符号/变量
```

在列表 2-35 中，我调用了 `def` 函数，并将符号 `foo` 和字符串 `"hello"` 作为参数传递。`def` 函数将符号 `foo` 与字符串 `"hello"` 关联起来，这样当我在 REPL（在当前命名空间内）中输入符号 `foo` 时，它就知道去哪里查找该值，并返回与该符号关联的值（在本例中，该值为 `"hello"`）。

注意

在内部，Clojure 管理着一个全局的命名空间及其关联变量的映射。

变量在其他命名空间中不可用，除非它们被“内部化”到那些命名空间中。内部化是一种花哨的说法，意思是“在当前命名空间中查找变量 x；如果不存在则创建它”。但你也可以使用 `ns` 宏的 `:refer` 特性从另一个命名空间“内部化”变量。我将在后面的章节中，当我们更详细地研究命名空间时，演示其工作原理。

### 临时变量

可以通过 `let` 形式创建临时变量。`let` 的工作方式是使用绑定。绑定是一种花哨的说法，意思是“将值分配给符号”，并且该符号仅在 `let` 块执行期间存在，之后就会消失。请参见清单 2-40。

```
(let [xyz "Hi!"]
(prn xyz))
;; nil
(prn xyz)
;; CompilerException java.lang.RuntimeException:
;; Unable to resolve symbol: xyz in this context
清单 2-40.
let 绑定示例
```

在清单 2-40 中，我们可以看到 `let` 绑定作为副作用打印了“Hi!”，但最终返回了 `nil`。然后，如果我们尝试从 `let` 绑定外部再次打印符号 `xyz`，我们会看到它引发了一个异常。

注意

Clojure 还提供了一个称为瞬态⁴的特性，这是一种创建数据结构非持久化副本的方法。你可以修改这个临时结构，然后调用 `persistent!` 来返回一个持久化/不可变的版本。当在临时结构性能更优的合适场景中使用时，这可以带来一些良好的性能提升。

我们稍后会在讨论解构时再回到 `let` 块，解构是一种从复杂对象中提取值的非常强大的特性。

注意

通过 `let` 创建的局部变量并非真正的变量，因为它们的值一旦设置就无法更改。

### 动态变量

创建变量时，它们会被绑定到当前命名空间，并被标记为“静态”。但变量可以（在临时时间段内）被动态更改。类似于 `let` 允许你定义局部变量，`binding` 宏允许当前线程在其块内执行期间操作变量的值，如清单 2-41 所示。

```
(def ^:dynamic my-name "Mark")
(prn my-name) ;; "Mark"
(binding [my-name "Bob"]
(prn my-name)) ;; "Bob"
(prn my-name) ;; "Mark"
清单 2-41.
动态变量绑定示例
```

从前面的示例可以看出，我们定义变量 `my-name` 的方式与之前定义变量的方式相同；唯一的区别是我们现在添加了元数据 `^:dynamic`。现在，当使用 `binding` 形式时，我们可以临时动态更改变量的值。你可以看到一旦块执行完毕，它就会恢复原状。

这对于测试目的特别有用。请记住：函数也可以分配给变量，因此在测试环境中，你可以动态地替换函数实现。此后，Clojure 还专门为此目的实现了另外两个函数：`with-redefs` ⁵ 和 `with-redefs-fn`⁶。

你也可以使用此功能来提供一种称为面向方面编程⁷的编程风格。例如，你可以操作一个日志函数，使其在标准日志信息周围添加上下文特定的细节。
