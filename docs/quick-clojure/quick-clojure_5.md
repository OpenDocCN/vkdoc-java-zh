# 6. 解构

在本章中，我们将介绍 Clojure 中的解构概念。

具体来说，Clojure 支持所谓的抽象结构绑定……嗯？是的，让我们稍微分解一下。

对人类而言，这意味着 Clojure 为我们提供了从数据结构中提取数据的能力，而无需实际遍历数据结构本身。这个概念通常被称为解构。

虽然这一章相当简短，但我强烈建议你在完成本节后花一些时间在 REPL 中编码，并试验我们向你展示的语法，因为这将帮助你更快地巩固对这些概念的理解。

为了理解解构的工作原理，我们将从一个简单的示例（清单 6-1）开始，然后继续探讨更复杂的变体，这些变体展示了通过利用解构可以实现优雅而简洁的代码。

注意

解构可以应用于 `let` 绑定、函数参数列表以及扩展为 `let` 或函数的宏。但我将主要使用 `let` 形式，因为它是在 REPL 中演示示例的最简单形式。

```
(let [[x y] [:a :b]]
(prn y x))
;; :b :a
清单 6-1.
使用向量的简单解构示例
```

查看清单 6-1，我们可以看到我们使用 `let` 绑定定义了局部变量。当我们这样做时，我们还会注意到必须将我们的符号（`x` 和 `y`）包装在与传入数据结构匹配的数据结构中。

如果你还记得，`let` 绑定通常看起来像 `(let [x 1] (prn x))`，其中 `x` 被赋值为 `1`。再看一下清单 6-1。传入的数据结构 `[:a :b]` 是一个向量，因此符号 `x` 和 `y` 被放置在一个向量内以确保结构匹配。然后值 `:a` 和 `:b` 被分配给 `x` 和 `y`，供我们使用。

```
(let [{a :a b :b} {:a "A" :b "B"}]
(prn a b))
;; "A" "B"
清单 6-2.
使用映射的简单解构示例
```

查看清单 6-2，我们可以看到我们的示例现在使用了映射数据结构，而不是像清单 6-1 中的向量。因此，你应该注意到 `let` 形式内部的语法现在是一个映射，以匹配传入的数据结构。

另一个不同之处在于，既然我们处理的是映射数据结构，我们需要将特定的映射键分配给我们的符号 `a` 和 `b`。在这个例子中，我们提取了 `:a` 键的值并将其分配给符号 `a`，同样地，我们提取了 `:b` 键的值并将其分配给符号 `b`。

Clojure 提供的一个不错的功能是通过抽象使我们的代码更清晰、更明确。看一下清单 6-3，你会看到一个实际应用的例子。

```
(let [{:keys [a b]} {:a "A" :b "B"}]
(prn a b))
;; "A" "B"
清单 6-3.
使用 :keys 特性的解构示例
```

回顾清单 6-3，我们可以看到结构现在略有不同。我们定义了一个 `:keys` 字段，并为其分配了一个符号向量 `[a b]`。这些符号代表传入数据结构中同名的键，并将持有它们对应的值。

现在让我们进一步扩展这个示例，演示如何访问整个数据结构以及特定的提取数据。

```
(let [{:keys [a c] :as complete} {:a "A" :b "B" :c "C" :d "D"}]
(prn a c complete))
;; "A" "C" {:c "C", :b "B", :d "D", :a "A"}
清单 6-4.
使用 :keys 和 :as 特性的解构示例
```



通过查看清单 6-4，我们可以开始看到一组更强大的特性正展现在我们面前。当这段代码被编译时，Clojure 会识别出 `:as` 字段表示我们希望将整个传入的数据结构存储到紧随其后的符号中。

这意味着符号 `complete` 现在是对整个传入数据结构的引用，而符号 `a` 和 `c` 仅是对提取出的同名键的引用。你会注意到我们不必提取所有键，只需提取我们主要关心的那些键。

在解构数据结构时，另一个值得审视的项目是 `:or` 字段，它为我们提供了为传入数据结构中缺失的任何键实现默认值的功能。清单 6-5 演示了这一点。

```
(def a-map {:a 1 :c 3})
(let [{:keys [a b c]
:as original-data
:or {a 11 b 22 c 33}} a-map]
[a b c original-data])
;; [1 22 3 {:c 3, :a 1}]
清单 6-5.
使用 :or 特性的解构示例
```

你可以在清单 6-5 中看到，我实际上将语法分散到了几行中，并且为了可读性，还将传入的数据结构存储到了一个单独的变量中；否则，这个示例可能会开始变得有些杂乱。

我们可以看到我们已经看过的类似项目——`:keys` 和 `:as` 字段——但现在增加了 `:or`，它定义了一个映射，为我们想要绑定符号的所有键设置了默认值。

例如，我们可以看到传入的映射数据结构没有 `:b` 键，因此我们的 `b` 符号（原本会尝试引用一个缺失的键）现在将引用 `:or` 映射中由 `b` 符号提供的值。

此外，我们可以使用 `&` 后跟一个符号，这将表示我们希望这个符号以列表形式持有对所有剩余值的引用。在清单 6-6 中，你可以看到我们在 `&` 字符后指定了符号 `z`，因此我们可以看到它持有值 `(:c :d)`。

使用 `&` 意味着我们能够使这个 `let` 块变成可变参数的（即能够接受大量额外的项目）。

注意

`&` 仅适用于向量数据结构。

```
(let [[x y & z] [:a :b :c :d]]
(prn x y z))
;; :a :b (:c :d)
清单 6-6.
使用 & 实现可变参数行为的解构示例

```

最后一个我们应该探讨的项目是如何处理从嵌套数据结构中提取信息。让我们同时使用映射和向量来演示这一点。参见清单 6-7。

```
(let [[[a b][c d]] [[:a :b][:c :d]]]
(prn a b c d))
;; :a :b :c :d
(let [{{:keys [foo baz]} :stuff} {:stuff {:foo "bar" :baz "qux"}}]
(prn foo baz))
;; "bar" "qux"
清单 6-7.
嵌套解构示例
```

回顾清单 6-7，从第一个示例开始，我们可以看到我们实际上仍在做与之前相同的事情。我们理解了传入数据结构的嵌套性质，因此在尝试从中解构数据时，我们模仿了这种结构。

这意味着如果我们有一个嵌套向量 `[[:a :b][:c :d]]`，那么我们希望将其解构为一个同样嵌套的数据结构：`[[a b][c d]]`。

同样值得一提的是，清单 6-7 中的代码可能看起来相当难以阅读，但在实践中，你的代码可能不会像这样，因为你将处理来自应用程序其他区域的数据。这意味着你很可能会有对数据的变量引用，而不是内联数据（就像我们在此处的示例中使用的那样）。

对于清单 6-7 中的第二个示例，即嵌套映射数据结构，我们可以看到我们想要获取分配给 `:stuff` 键的数据结构。因此，为了实现这一点，我们在解构时引用了 `:stuff` 键，然后使用熟悉的 `:keys` 字段从中解构其值。

注意

如果你的映射数据结构键是符号或字符串而不是关键字，那么你可以使用 `:strs` 和 `:syms` 来代替 `:keys`，这将按预期工作。

至此，我们所有的示例都是使用 `let` 绑定形式设置的，但正如本章开头提到的，我们也可以将解构用于函数参数。

解构函数参数列表的一个有趣方面是，你可以设置可选的关键字参数。查看清单 6-8 以了解其工作原理。

```
(defn foo [a b & {:keys [c d]}]
(println a b c d))
(foo "A" "B")
;; A B nil nil
(foo "A" "B" :c "C")
;; A B C nil
(foo "A" "B" :c "C" :d "D")
;; A B C D
清单 6-8.
使用解构实现可选函数参数
```

从清单 6-8 中我们可以看到，我们再次使用了 `&` 来表示一个可变参数的函数参数列表，但现在我们还使用了 `:keys` 行为来允许我们处理映射键的提取。

## 总结

在本章中，我们学习了解构数据结构这一重要且有用的概念。在处理复杂集合时，这有助于实现非常简洁优雅的代码。

我们看到了如何对映射和向量进行基本解构的示例，以及如何处理嵌套的等价结构，并利用 `:keys`、`:as` 和 `:or` 字段来进一步扩展可用的行为。

在下一章中，我们将研究另一个有用的概念，称为模式匹配。模式匹配提供的是基于预定义模式触发特定功能的能力。这些模式应用于某种形式的输入，并且（很像解构）也提供了一些非常漂亮和优雅的代码解决方案。
