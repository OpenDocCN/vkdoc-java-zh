# 8. 硬件

> *Les bons outils font les bons ouvriers.*

> *工欲善其事，必先利其器。*
> 
> ——法国谚语

物联网和边缘计算解决方案的生命周期以年为单位，有时甚至长达数十年。无论是微控制器、网关还是边缘节点，支撑它们的硬件设备都需遵循比标准 IT 设备更高的标准。它们很可能面临恶劣的环境条件，例如低温或高温、灰尘、电磁干扰、潮湿、辐射和振动。在大多数情况下，某种形式的加固是必需的。此外，这类设备通常安装在难以触及的位置。因此，它们需要具备弹性且能效高，尤其是在依靠电池供电运行时。

如果工欲善其事，必先利其器，那么好的硬件才能成就好的软件。鉴于以上种种，为特定用例选择合适的硬件至关重要。本章旨在为您提供做出该决策的指导原则，并调研市场上可用的选择。

## 选择标准

没有两个物联网部署是完全相同的。因此，试图制定一份硬性的硬件选择标准清单是愚蠢的。在本节中，我挑选了最通用的标准，作为您评估过程的起点。当然，您必须根据自身的优先级为每个标准赋予适当的权重。不过，我建议不要过分看重价格。要使您的解决方案在长期内具有成本效益，您需要一定的可靠性和耐用性，而选择廉价低质的硬件会适得其反。

话虽如此，现在让我们来看看这些选择标准。

### 功耗

在大多数物联网解决方案中，受限设备被部署在现场，用于传输传感器采集的数据，并通过执行器与物理世界交互。在许多行业和用例中，这些设备由电池供电。然而，在过去几年中出现了一些替代方案。以太网供电（POE）基础设施和标准的成熟，使得通过网络电缆供电成为可能，适用于数字建筑和智能工厂等解决方案。在另一个极端，能量收集技术可以为低功耗电子设备提供非常微小的电量。例如，一个能量收集墙壁开关可以发送短距离无线电信号，以打开附近的照明设备或装置。

如您所见，功耗是众多物联网部署场景中的一个重要考量因素。当然，在确定特定组件的效率时，硬件只是其中的一部分。还有许多其他因素需要考虑，例如所用传感器的类型、采样率、网络技术和协议，以及最终的软件本身。然而，硬件在决定组件的功耗方面起着主导作用。这包括所使用的微控制器（MCU）或处理器、随机存取存储器的类型、持久化存储的类型，甚至用于连接外设的接口。此外，功率更高的组件可能会产生大量热量，需要冷却解决方案，从而增加整体功耗需求。在性能与能效之间取得恰当的平衡至关重要。

### 生命周期

鉴于物联网和边缘解决方案的生命周期可能很长，拥有稳定的硬件组件供应链至关重要。为新的硬件平台或新传感器重写软件，对您或最终用户而言价值甚微。幸运的是，有几种可能的方法可以解决这个问题。

第一种可能的方法是坚持使用那些有长期产品支持记录的可靠供应商。然而，产品线和战略的变化在企业界很常见。今天可靠的供应商，随着时间的推移可能会变得不可靠。如果您的策略依赖于特定的组件或板卡，那么通过具有法律约束力的协议来确保供应至关重要。

另一种选择是自行制造组件。传统上，这是拥有特定制造能力的大型组织才能走的道路。当然，微控制器和片上系统（SOC）是复杂的集成电路，涉及大量的知识产权。因此，市场上有几家参与者建立了非常盈利的业务，专注于知识产权许可。Arm 无疑是这一趋势的典型例子。[RISC-V 指令集](https://riscv.org/technical/specifications/)以及基于它的开源处理器 IP（例如[OpenHW Group](https://www.openhwgroup.org/)的[CORE-V](https://github.com/openhwgroup/core-v-cores)系列）的出现，使得各种规模的组织都有可能采用这一策略。

各种软件策略也有助于缓解与硬件平台变更相关的问题。利用可移植的编程语言是一种方法，尽管您应确保所使用的任何依赖项都是可移植的。另一种可能性是在您的技术栈中使用硬件抽象层。[Eclipse MRAA](https://github.com/eclipse/mraa)和[Eclipse UPM](https://github.com/eclipse/upm)项目就是很好的例子；MRAA 在多个 MCU 和板卡上提供统一的 API，而 UPM 则为各种传感器和执行器提供相同的功能。

从长远来看，您可能会依赖一种混合方法，结合可信赖的供应商、开放硬件和硬件抽象。

### 用例需求

物联网受限设备和边缘计算节点部署在企业数据中心之外。它们通常位于难以触及的位置，那里连接可能不稳定，物理条件恶劣。一个很好的例子是部署在深空探测器中的抗辐射处理器。虽然它们的规格与地球上部署的尖端嵌入式解决方案相比相形见绌，但它们却非常适合其预期用途^(³⁶)。换句话说，环境决定了硬件需要满足的大部分非功能性需求。

部署地点特定网络技术和电源的可用性将影响硬件选择过程。然而，用例的功能需求同样重要。对于关键任务、实时应用的解决方案，通常需要保证内存和网络访问的延迟。在设备需要处理大量传感器输入并做出影响人身安全的决策的应用中，内存保护和确定性中断处理也非常有用。

选择硬件平台时需要考虑的一个非常重要的方面是设备连接需求。您需要将所选传感器和执行器连接到受限设备或网关。因此，您选择的硬件平台必须提供适当类型的 I/O 端口。在某些情况下，外部端口的存在可以延长设备的使用寿命。例如，将基于 USB 的 AI 加速器插入网关比完全更换设备更具成本效益。

当然，在满足需求与其他考量之间取得恰当的平衡通常很困难。像流行的树莓派单板计算机这样的大众市场解决方案成本效益高且用途广泛，但并不特别适合在恶劣环境中部署。另一方面，加固组件成本更高，且供应通常更有限。



### 安全性

部署在现场的受限设备、网关和边缘节点容易受到一系列攻击，这些攻击不会影响云或传统 IT 基础设施。它们的物理位置使其面临被物理破坏甚至被盗的风险。这可能会造成严重后果。被盗设备可用于冒充合法设备，并向攻击者暴露服务器端后端的弱点。设备上存储的数据也可能被用来泄露商业机密甚至客户身份，从而带来严重的潜在影响。你需要为将要部署的设备制定全面的安全策略。

在硬件层面，此类策略首先要在能够承载可信平台模块（TPM）的设备中利用该模块。TPM 是一种专用微控制器，旨在提供安全相关功能。它们可以生成加密密钥以及一个几乎无法伪造的哈希密钥，该密钥可总结用于远程认证的设备的硬件和软件配置。此外，它们可以安全地存储加密密钥；这些密钥可用于磁盘加密或身份验证等目的。TPM、加密密钥和数字证书是可用于在设备中实现*信任根*的构建模块。信任根是一条信任链，确保你的设备仅启动合法代码。

存储加密，也称为静态数据加密，是任何部署在野外环境中的设备的关键功能。虽然软件解决方案可以满足这一需求，但也可以通过使用自加密驱动器将这一关注点卸载到硬件上。当然，这种解决方案会增加设备的成本和功耗。FIPS 140-2、Opal 2.0 和基于硬件的 AES 256 位加密是常见的可用方案。FIPS 驱动器具有防篡改特性，通常用于传输受控非机密信息的国防和政府应用。

物理安全也是防止设备被盗或受损的重要考量。具体来说，这意味着你至少应该能够在设备外壳上加锁。在大多数情况下，将设备固定在防篡改外壳中是个好主意。专用外壳可以抵御一系列环境因素。如果这适用于你的用例，你应该寻找符合 NEMA 或 IP 等级的外壳。来自美国电气制造商协会的 NEMA 等级系统根据外壳对固体颗粒侵入、液体侵入和腐蚀的防护能力进行评级。NEMA 等级主要在美国使用。相反，IP 等级系统在全球范围内使用，但仅评估液体侵入和固体颗粒侵入防护。NEMA 和 IP 等级在某种程度上可以相互比较，但由于两个系统考虑的因素集不同，无法直接比较。

在大多数情况下，最好移除或阻塞任何未实际使用的端口、接口或引脚。攻击者可能利用这些中的任何一个来破坏设备。

### 成本

我讨论的每一个标准都将直接影响你设备的成本。反过来，成本无疑将驱动你的许多硬件决策。找到正确的平衡点并不容易，因此你需要为项目恰当地收集功能性和非功能性需求。此外，在撰写本文时，市场仍受到 COVID-19 疫情后果的严重影响。所谓的“芯片短缺”^(³⁷) 使得许多组件难以找到，导致价格大幅上涨。这种暂时的稀缺性导致许多组织精简其供应链，并在某些情况下，在没有特定功能的情况下发货产品，或使用更昂贵的组件。

如果你打算为自己或客户大规模部署设备，你应该探索批量采购的成本节约。这意味着你将通过标准化使用更少的组件来降低成本。然而，对于特定应用，某些组件可能并非最优选择。

如果你仍处于原型设计或概念验证阶段，缓解硬件组件不可用的一种方法是使用仿真工具。此类工具使你能够模拟特定的开发板，在某些情况下，还能模拟传感器、执行器和外设。在 2022 年 3 月发布的一篇博文中，我在 Eclipse 基金会的前任 Benjamin Cabé 指出了该领域的三个有趣选项：[Renode](https://renode.io/)、[Wokwi](https://wokwi.com/) 和 [TinkerCAD](https://www.tinkercad.com/)。^(³⁸)

## 微控制器

微控制器（MCU）是大多数受限设备的核心。它们是实现在单个集成电路（IC）上的小型计算机，通常包含处理器核心、随机存取存储器、定时器、计数器和可编程输入/输出外设。MCU 旨在支持小型控制应用，并以可承受的价格提供低功耗。通常，它们提供大小从几百千字节到几兆字节的闪存或 EEPROM 存储。随机存取存储器很小，通常在 16 到 512 千字节之间。在微控制器中看到 8 位、16 位和 32 位 CPU 核心并不罕见。

注意

你可以将微控制器视为一种专门的、低功耗的片上系统（SOC）。在平板电脑、智能手机甚至使用 Apple Silicon 的 Mac 电脑中发现的 SOC 集成了更高级的功能，例如蜂窝调制解调器、图形处理单元（GPU），甚至人工智能加速器。

目前市场上可用的大多数微控制器都依赖于使用某种 RISC 架构的处理器核心，其中 Arm 和 AVR 是最广泛的选择。然而，RISC-V 正在开始取得进展，甚至还有可用的开源核心和 MCU 设计，例如 [OpenHW CORE-V MCU](https://www.openhwgroup.org/core-v-devkits/)。

微控制器通常实现冯·诺依曼架构或哈佛架构。术语*冯·诺依曼架构*指的是一种存储程序计算机，其中指令获取和数据操作不能同时发生，因为它们涉及共享总线。冯·诺依曼架构的主要替代方案是*哈佛架构*。哈佛架构计算机是存储程序系统，拥有专用的地址和数据总线用于内存访问，以及另一组用于指令。冯·诺依曼架构机器的设计通常比哈佛架构机器更简单。目前大多数高性能桌面级和数据中心级处理器都融合了这两种架构的特点。



### Arm：Cortex-M

Arm 架构无疑是当今微控制器领域的主导者。其原因在于 Arm 对功耗的高度重视。首个 Arm 处理器于 1985 年交付，其设计目标是在 1 瓦功率下运行。该芯片平均典型功耗低于 100 毫瓦，同时性能却是当时流行的 Intel 80286 或 Motorola 68020 处理器的十倍。^(³⁹)

如今，Arm 提供两个非常适合受限设备的特定设计系列：Cortex-M 和 Cortex-R。然而，您需要牢记，仅提及系列和型号并不足以确定微控制器的全部能力。这是因为 Arm 通常只提供处理器内核知识产权（IP），而其他能力则取决于实际的芯片制造商。

注意

ARM 是一家芯片设计公司，而非芯片制造商。它将其处理器 IP 授权给其他公司。它提供的大多数处理器内核模型都有多个可选功能。因此，仅了解 MCU 中的特定 Arm 内核（例如 Cortex-M4）是不够的。在为项目选择硬件之前，请务必仔细阅读规格说明书。

详细涵盖 Cortex-M 系列中众多的选项和变体将使本书过于厚重。下面我将列出每个内核的主要特性。

*   **M0：** 此内核针对小芯片尺寸进行了优化。常见于最经济实惠的 MCU 中。
*   **M0+：** 此内核是 M0 的超集，允许使用相同的工具链。流水线缩短了一级，从而降低了功耗。某些 Cortex-M3 和 Cortex-M4 功能可作为选项提供。
*   **M1：** 此内核针对集成到现场可编程门阵列（FPGA）进行了优化。
*   **M3：** 此内核为其三级流水线引入了分支推测功能，从而提高了性能。它用于微控制器以及作为更大芯片中的辅助内核。
*   **M4：** 此内核在 Cortex-M3 的基础上增加了 DSP（数字信号处理）指令，并可配备可选的浮点运算单元（FPU）。包含 FPU 的内核也称为 Cortex-M4F。
*   **M7：** 此内核具有带分支预测的六级超标量流水线。它可配备能够执行单精度和可选双精度运算的浮点运算单元。包含 FPU 的内核被称为 Cortex-M7F。M7 的能效几乎是 M4 的两倍。
*   **M23：** 此内核类似于 Cortex-M0+，并针对小芯片尺寸进行了优化。它增加了整数除法指令和 TrustZone 安全特性。它实现了 2015 年宣布的较新 Armv8-M 架构。
*   **M33：** 此内核是 M4 和 M23 的融合体。它也提供 TrustZone 指令。
*   **M35P：** 此内核通过添加可配置的奇偶校验和纠错码（ECC）内存特性来改进 M33。它还配备了新的指令缓存并实现了防篡改概念。
*   **M55：** 此内核基于较新的 Armv8.1-M 架构。它可选地提供 TrustZone 指令、DSP 支持、协处理器支持、ECC 以及许多其他特性。它是一个比其任何前代产品都更具模块化和可配置性的内核。

重要的是要记住，Cortex-M 内核不一定实现其目标 Arm 架构版本的所有指令。此外，Arm 提供的众多可选功能也会影响可用的指令。通常，为 Cortex-M0、M0+ 和 M1 编译的二进制文件可以在 Cortex-M3、M4 和 M7 内核上无需修改地运行。针对 Cortex-M3 的二进制文件可以在 M4、M7、M33 和 M35P 上直接运行。

[2021 年版 Eclipse IoT 与边缘开发者调查](https://outreach.eclipse.foundation/iot-edge-developer-2021)发现，35% 的受访者选择了基于 Cortex-M3 和 Cortex-M4 内核的 MCU。30% 的受访者选择了采用 Cortex-M7 内核的芯片，25% 的受访者使用了配备 M0/M0+ 内核的微控制器。该问题允许受访者最多选择三个选项。



### AVR：Arduino 的核心

Arm 从事的是授权业务。由于许多架构特性是可选的，你需要仔细阅读从特定供应商处采购的特定 MCU 的文档。而 Microchip 的 AVR 架构则要直接得多，因为该公司直接销售实际的 MCU。

注意

AVR 长期以来一直被称为 Atmel AVR。Microchip 于 2016 年收购了 Atmel，并逐步停止使用该品牌。

AVR 微控制器架构于 1997 年推出。到 2003 年，Atmel 已出货 5 亿颗芯片。2005 年流行的 Arduino 平台的推出，极大地加速了 AVR 的市场普及。AVR 微控制器是改进型哈佛架构机器。程序和数据存储在不同的内存系统中，并呈现于不同的地址空间。然而，该架构提供了特殊的指令，使其能够从程序存储器中读取数据。

AVR MCU 使用 8 位 RISC 处理器内核。通过引入新型号，其指令集随着时间的推移不断扩展。AVR 系列通常最高可运行在 20 MHz 时钟频率下，尽管特定型号可以达到 32 MHz。当然，更高的时钟频率会导致更高的功耗。AVR 系列中的新型号配备了片上振荡器，从而无需外部时钟或谐振器电路。

从存储角度来看，AVR MCU 可提供高达 256 KB 的内部可自编程指令闪存（XMEGA 上为 384 KB）和高达 4 KB 的内部数据 EEPROM。随机存取存储器最高可达 16 KB（XMEGA 上为 32 KB）。特定型号还可提供 64 KB 的外部小端数据空间。表 8-2 概述了 AVR 系列中主要系列的特性。

| XMEGA | 16–256 KB | 32 | 1–32 KB | 512–2048 字节 |

每个系列都包含多个具有不同功能集和特性的产品。以下列表突出了每个系列的差异化因素：

*   **tinyAVR：** ATtiny 系列针对小封装尺寸进行了优化，仅提供有限的外设。
*   **megaAVR：** ATmega 系列提供广泛的 I/O 引脚以及更多的内存和存储空间。
*   **AVR Dx：** AVR Dx 系列包含多个子系列，专注于模拟信号调理、功能安全和人机交互。
*   **XMEGA：** ATxmega 系列是 AVR 系列中最强大的。这些 MCU 支持加密功能，并提供模数转换器。

AVR MCU 最著名的用户是同名公司广受欢迎的 Arduino 板系列。目前和过去的大多数 Arduino 都采用 ATmega 系列的 MCU。例如，Arduino Uno 配备的是 ATmega328P，而 Uno WiFi Rev 2 则采用 ATmega4809。后者最高可运行在 20 MHz，并配备 48 KB 闪存、6 KB RAM 和 256 字节 EEPROM。

### RISC-V：一个严肃的挑战者

RISC-V 架构由加州大学伯克利分校的研究人员于 2010 年提出，是一个开放标准。其指令集架构（ISA）的文档以及一些核心设计均在开源许可下发布。RISC-V 基金会成立于 2015 年，负责管理 ISA 的演进；该基金会于 2019 年迁至瑞士，以解决对美国贸易法规的担忧，并于 2020 年 3 月更名为 [RISC-V International](https://riscv.org/exchanges/cores-socs/)。

RISC-V International 允许在硬件和软件中不受限制地使用该 ISA。这一立场导致了其被广泛采用。截至 2022 年 3 月，该组织网站上列出了 [111 个不同的内核](https://riscv.org/exchanges/cores-socs/)、31 个 SOC 平台和 12 个 SOC。RISC-V 的实际市场足迹可能要高得多，因为组织不一定会公开其对该技术的使用情况。值得一提的是，只有 RISC-V International 的成员才拥有批准变更的投票权，并且可以使用商标兼容性标志。

RISC-V ISA 的一个重要特性是其模块化。基础整数指令集有 32 位和 64 位两种格式（RV32I 和 RV64I）。还有针对 32 位嵌入式（16 个寄存器，RV32E）甚至 128 位版本（RV128I）的草案。所有其他类型的指令都在 ISA 标准扩展中进行了说明。这些扩展通常使用单个字母作为名称，尽管最近的扩展名称以 Z 开头，后跟字母名称。标准扩展的广泛使用示例包括整数乘法和除法（M）、原子指令（A）、单精度浮点（F）、双精度浮点（D）以及控制和状态寄存器（Zicsr）。标准扩展被设计为与所有标准基础指令集兼容，并且彼此之间不会冲突。RISC-V 内核的设计者还可以创建自己的非标准指令集扩展。关于用户定义扩展的行业合作倡议也已开始出现。一个很好的例子是 [OpenHW CORE-V eXtension I/​F](https://docs.openhwgroup.org/projects/openhw-group-core-v-xif/en/core-v-xif_v0.2.0/intro.html)，它旨在简化附加到或内置于 RISC-V 内核中的自定义指令和自定义加速器的实现。RISC-V 规范规定，此类扩展的名称必须以“X”开头（用于用户级指令）和“SX”开头（用于监管级指令）。

鉴于此，全面审视整个 RISC-V 格局将是困难的。这是因为实现者可以从头开始设计他们的芯片，采用众多可用的开源内核之一，甚至许可开源或商业内核并添加专有功能。一方面，这使得生态系统多样化，拥有可用于集成到受限设备、网关、边缘节点以及日益增多的桌面级和服务器级硬件中的内核、MCU 和 SOC。另一方面，由于产品缺乏共同的参考点，这使得提供市场的高层概况变得更加困难。尽管如此，基于 RISC-V 的 MCU 生态系统正在蓬勃发展，它们无疑值得你考虑。

2021 年，[Eclipse IoT 和边缘开发者调查](https://outreach.eclipse.foundation/iot-edge-developer-2021)发现，9% 的受访者使用了基于 RISC-V 技术的 MCU。即使是 Arm 市场的成熟参与者，如 Espressif，也在宣布基于 RISC-V 的新产品。^(⁴⁰)



## 选择微控制器

面对众多选择，挑选微控制器可能是一项艰巨的任务。最终，你的用例需求应指导你的决策。以下是你需要自问的一系列问题：

*   **你是否需要每秒处理超过几个事件？** 16 位和 32 位微控制器相对于 8 位微控制器的主要优势在于速度。如果你只需要每十秒记录一次当前的温度和湿度，那么即使是 8 位微控制器也能胜任。当然，如果要在机车上运行一个集成上百个传感器实时值以进行预测性维护的 AI 模型，则需要更高端的微控制器。你的用例很可能介于这两个极端之间。

*   **你是否需要计算大数值？** 在 8 位微控制器上计算大数值是可行的。然而，这些计算在 16 位和 32 位硬件上会快得多。如果你需要执行图像处理、视频分析，甚至是利用快速傅里叶变换的数字信号处理，那么运行在低频兆赫兹的 8 位微控制器完全不可行。

*   **你是否需要高级连接能力？** 驱动高级 I/O 接口，尤其是同时驱动多个接口，所需的处理能力超出了 8 位微控制器的提供范围。如果你需要以太网、通用串行总线（USB）、控制器局域网（CAN）总线或多个通用异步收发器（UART），那么请选择 32 位微控制器。

*   **你需要哪些外设？** 有些微控制器只提供最基本的功能，而另一些则会集成大量内置外设甚至传感器。

*   **延长电池寿命有多关键？** 32 位微控制器比 8 位微控制器消耗更多电量。当然，这可以通过在不使用时让处理器内核和其他设备进入休眠状态来部分缓解。然而，8 位微控制器在电池寿命上的优势解释了它们为何仍在市场上存在。

决策的另一个维度是是否使用无线连接。如果你使用无线电连接到网络，它很可能成为板上最耗电的组件，无论涉及的是 BLE、LoRaWAN、LTE-M 还是其他技术。如果电池寿命是你的用例核心，那么功耗将影响哪种连接选项是最佳选择。

*   **你的目标微控制器支持哪些软件？** 并非所有微控制器都能获得同等水平的软件支持。你需要考虑软件开发工具包（SDK）、库，甚至是实时操作系统。更广泛的软件支持为你构建解决方案提供了更多选择。

最终，在长期承诺使用特定硬件平台之前，最好始终在尽可能接近真实环境的条件下进行实验。

## 传感器与执行器

选定微控制器后，下一步是寻找能够与其交互并满足你需求的传感器和执行器。要详细涵盖当前市场上种类繁多的传感器，需要一本专著。我将重点介绍将传感器和执行器与微控制器连接的各种方法。但在此之前，让我们先看看选择传感器时应关注的主要特性。

### 传感器特性

传感器测量质量受其设计特性的影响。这些特性大多会影响传感器的价格和耐用性。以下是应关注的主要特性列表：

*   **精度：** 精度反映了传感器返回值与实际值的符合程度。精度由多个其他因素决定，例如灵敏度、分辨率、噪声、系统误差和老化。在大多数情况下，灵敏度和噪声比精度更重要。你需要考虑所有可能的不精确来源。

*   **测量范围：** 这是传感器能够返回准确读数的数值范围。尝试测量超出测量范围的值将返回错误的信号值。

*   **可靠性与精确度：** 传感器应对同一测量值始终返回相同的结果。例如，在测量一块木头的尺寸时，传感器应在连续测量以及任意时间间隔后的测量中返回相同的值。换句话说，如果木头的实际长度保持不变，传感器应始终返回相同的值。话虽如此，环境因素可能会引入读数变化。特定传感器也可能随着老化而变得不那么可靠和精确。你可以通过软件对此进行补偿，前提是你通过实验或直接经验收集了足够的信息。定期校准也可以解决这个问题，尽管并非所有传感器都能做到。

*   **分辨率：** 传感器的分辨率是其能够捕捉到的最小差异。假设你想测量到一个物体的距离。该物体距离 2 米。如果你的传感器分辨率为 2 毫米，它返回的值可以是 1.998 米、2.000 米和 2.002 米。

*   **采样率：** 采样率，或称采样频率，是指进行测量的频率。采样率应至少是信号变化频率的两倍；这被称为奈奎斯特准则。

*   **灵敏度与噪声：** 这两个因素密切相关。灵敏度是输出信号与测量值之间的比率。例如，当使用电压输出传感器测量以摄氏度为单位的环境温度时，灵敏度以伏特/摄氏度表示。另一方面，噪声是测量信号中不希望出现的变化。对于传感器，它通常表示为信噪比（SNR）。噪声水平受多种外部因素影响，例如电气干扰、温度和振动。

如你所见，你需要恰当地引出并记录你的需求，才能选择合适的传感器。物联网工业流程中的许多问题可以通过使用不同类型的传感器或将现有传感器移至不同位置来解决。此外，在任何流程的关键步骤中，你都应始终部署冗余传感器。解决方案的整体可靠性将远远超过增加的成本。



### 建立连接

与其他计算机一样，微控制器（MCU）以数字形式处理信息。换句话说，它们可以直接解读二进制信号。此类信号可以表示门是否打开，或传达按钮按下状态。如果 MCU 工作在 3.3V 电压下，它会将 0V 测量值理解为二进制 0，将 3.3V 测量值理解为二进制 1。微控制器具有通用输入/输出（GPIO）引脚，用于发送或接收数字信号。然而，现实世界远比单纯的 0 和 1 复杂。因此，许多传感器和执行器都使用模拟值。也就是说，它们会在特定范围内发送或处理电压值。对于一个 3.3V 的 MCU，模拟传感器会返回 0V 到 3.3V 之间的任意值。大多数 MCU 都配备了一个或多个模数转换器（ADC）来解读此类信号。ADC 可以将引脚上的模拟电压转换为数字值。它们还能将电压电平转换为 MCU 支持的数值范围。

ADC 有多种不同类型。大多数 Arduino 板上的 ADC 是 10 位 ADC，这意味着它可以检测 1,024（2¹⁰）个离散的模拟电平。一些微控制器配备 8 位 ADC（2⁸ = 256 个离散电平），还有一些配备 16 位（2¹⁶ = 65,536 个离散电平）甚至 24 位（2²⁴ = 16,777,216 个离散电平）ADC。ADC 的另一个重要特性是其采样率：即 ADC 对输入信号进行采样的频率，以赫兹（Hz）为单位。务必记住，传感器也有其自身的采样率。因此，你需要确保计划使用的传感器和 ADC 组合能够共同提供你所需的采样率。

集成电路（IC）的 GPIO 有多种实现方式。该术语指的是专用于数字信号的引脚，而非统一的引脚布局标准。有时，会使用扩展器 IC 将 GPIO 引脚连接到诸如 I²C 和 SMBus 等串行通信总线。微控制器通常包含 GPIO。根据应用场景，微控制器的 GPIO 引脚可能是其与外部电路的主要接口，也可能是众多 I/O 选项中的一种。

I²C、SPI、UART 和 W1 是微控制器甚至工业计算机上常见的四种高度相关的 I/O 接口。下面我将总结这四种接口的特性。

*   **I**^(**2**)**C（内部集成电路）：** 一种在基于 MCU 的设备中常见的串行通信协议，尤其适用于与传感器、存储设备和 LCD 的接口连接。它是一种同步协议，因为它使用了一条时钟线。I²C 协议与 SPI 类似，但使用的线缆更少。然而，由于地址冲突，当使用多个具有固定地址的传感器实例时，它可能会带来困难。I²C 允许多个主节点，而 SPI 则不支持这一点。

*   **SPI（串行外设接口）：** 另一种在基于 MCU 的设备中常见的同步串行通信协议。它比 UART 和 I²C 都快，但需要更多的线缆。多个子节点可以共用同一个 SPI 主引脚，设备甚至可以菊花链连接。

*   **UART（通用异步收发传输器）：** 也称为串行端口，是 MCU 上常见的外设。异步串行端口为两个设备提供了一种无需共享时钟信号即可通信的简单方式。UART 仅需两根线缆，尽管它们通常还带有另外两根线（RTS 和 CTS）来实现硬件流控制，以防止数据丢失。是否需要硬件流控制取决于所使用的线缆协议。

*   **1-Wire（单总线）：** 一种通过单根线缆实现低速（16.3 kbit/s）连接的协议。该线缆同时传输信号和电力。1-Wire 在概念上与 I²C 类似，但以降低数据速率为代价提供了更远的传输距离。1-Wire 连接通常是从 MCU 到多个外设的一对多连接。

你可能已经注意到，我刚才讨论的四种选项都是串行和双向协议。这并非偶然。在部署大量设备时，串行协议减少了布线需求，使其更具成本效益。表 8-3 对这四种协议进行了相互比较。

尽管我在解释中只提到了传感器，但我所涵盖的协议也可用于控制执行器。另一种在处理它们时有用的技术是脉冲宽度调制（PWM）。PWM 是一种在机器人技术中用于驱动电机和舵机的技术。MCU 利用内部计数器调制方波的占空比，以控制提供给设备的功率大小。通过快速切换电源与负载之间的开关，可以控制提供给设备的电压（和电流）平均值。许多 MCU 都带有 PWM 控制器，这些控制器作为固件控制下的外设暴露于外部引脚。

## 端点、网关和边缘节点

与 MCU 相比，物联网端点（通常包含单个 MCU）、物联网网关和边缘计算节点在能力上更接近传统服务器。事实上，在某些情况下，边缘计算节点将是部署在连接提供商光纤网络边缘的本地数据中心中的传统服务器。然而，网关和边缘节点通常位于数据中心之外。例如，物联网网关可用于通过 CAN 总线连接集成传统的可编程逻辑控制器（PLC）。部署在风电场的边缘节点可用于聚合从附近涡轮机收集的数据。对于此类部署场景，依赖高性能但低功耗的硬件非常有意义。低压处理器可以被动散热，并且从设备中移除风扇等机械部件通常可以提高其平均无故障时间（MTBF）。使用低压部件还有一个额外的好处，即首先减少了热量输出。

[2021 年版 Eclipse 物联网与边缘开发者调查](https://outreach.eclipse.foundation/iot-edge-developer-2021)发现，32% 的受访者在其物联网网关和边缘节点中使用了基于 Arm v8 架构的处理器。AMD/Intel x86-64 架构以 29% 的比例位居第二，而 Arm v7（32 位）是 27% 受访者的选择。该问题允许参与者最多选择三个选项。

鉴于 Cortex-A 和 x86-64 架构的主导地位，现在让我们来看看它们所提供的处理器类型。

根据所选内核，芯片制造商可以在每个簇中包含最多四个或八个内核，并在单个芯片上部署多个簇。此外，部分 Cortex-A 内核可以部署在*big.LITTLE*配置中，即性能更强但功耗更高的内核（大核）与能效更高但速度较慢的内核（小核）配对使用。处理器设计者可以将 Cortex-A55 和 A78 部署在这种配置中；Cortex-A510 和 A710 则构成另一组可能的配对。

Cortex-A65、A76 和 A78 具有 AE（汽车增强型）变体，为安全应用增加了双核锁步功能。这种配置允许内核并行处理相同任务，并比较结果以发现差异。这极大地增强了解决方案的功能安全性。

大多数芯片制造商将以片上系统的形式交付其 Arm 产品，并增加对串行 ATA（SATA）、PCI Express（PCIe）、USB、以太网以及 I²C、SPI 和 GPIO 等低速接口的支持。Arm 还提供其 Mali 系列图形处理单元（GPU）的授权，这些 GPU 可以与处理器内核一起集成在芯片封装中。市场中的一些参与者从 Arm 获得处理器内核授权，并在芯片中添加自己的专有 GPU。

### x86-64

尽管 x86-64 生态系统只有两个主要参与者，即 AMD 和 Intel，但他们的产品线非常丰富且定期更新。与其介绍不断变化的产品线，不如介绍通常在物联网网关和边缘节点中使用的处理器概要。

AMD 和 Intel 都有面向嵌入式市场的产品。这些是真正的片上系统，集成了处理器内核、图形处理单元（GPU），在某些情况下还包括 I²C、SMBus、SPI、UART 等低速接口。由于它们与桌面和服务器处理器相近，因此也提供了与该类机器相关的接口，例如 PCIe、SATA 和 USB。对纠错码（ECC）内存的支持也并不罕见。高端且最昂贵的选择集成了 AMD EPYC 和 Intel Xeon 系列的内核。

为了控制成本，许多设备制造商将依赖来自 AMD 和 Intel 移动产品线的处理器。在这种情况下，他们需要额外增加电路来支持低速 I/O 接口。这类处理器的优势在于功耗和散热更低。

当然，某些边缘计算工作负载需要服务器级处理器的全部性能。此时，这些设备之所以成为边缘设备，并非因为它们使用了嵌入式技术，而是因为它们部署的物理位置。然而，通常可以找到适合边缘部署的服务器，它们采用加固型机箱或外壳。

为了让您了解市场上使用的 x86-64 处理器类型，以下是撰写本书时正在出货的四款特定设备中使用的 CPU：

*   **物联网网关：** AMD Athlon Silver 3050e 移动处理器（1.40 GHz，最高睿频 2.80 GHz，2 核，4 线程，4 MB 缓存）

*   **工业计算机：** 第 8 代 Intel Core i3-8145U 移动处理器（2.10 GHz，睿频最高 3.90 GHz，2 核，4 线程，4 MB 缓存）

*   **物联网网关：** Intel Atom x5-E3940 嵌入式处理器（1.60 GHz，睿频最高 1.80 GHz，4 核，4 线程，2 MB 缓存）

*   **边缘节点（服务器）：** 双路 Xeon E5-2690v4 服务器处理器（2.60 GHz，睿频最高 3.50 GHz，14 核，28 线程，35 MB 缓存）

最终，您应该选择适合您工作负载和目标功耗的处理器。请注意，服务器级和嵌入式处理器通常比替代产品具有更长的市场供货期，从而提供长期稳定的硬件平台。



## 开源硬件

本书聚焦于开源技术。如果不讨论成功的开源硬件项目，那将是不完整的。目前，开源处理器核心、微控制器、单板计算机以及传感器和执行器的生态系统正在不断壮大。在本节中，我将为前三类提供示例。

由于硬件对物联网解决方案的设计影响巨大，依赖开源组件为设备设计者带来了显著优势。物联网开发者会针对代码体积进行优化，并需要将功耗降至最低`–`至少对于现场部署的设备而言是如此。当你不仅控制软件，还控制硬件时，更容易达到理想的功耗特性。此外，开源硬件使得在解决方案的整个生命周期内拥有稳定的硬件平台变得更加容易；如果某个特定供应商停止供应某个组件，总能找到其他供应商或委托第三方来提供。最后，开源硬件使设计者能够扩展和改进组件。由于物联网和边缘计算解决方案通常部署在偏远地区或恶劣环境中，因此能够调整硬件以适应特定用例的能力是无价的。

### 处理器核心：OpenHW CORE-V 系列

Eclipse 基金会与 [OpenHW 组织](https://www.openhwgroup.org) 紧密合作，在开源处理器核心实现的基础上构建一个全面的 RISC-V 生态系统。OpenHW 组织是一个加拿大的非营利性全球组织，硬件和软件设计师在此协作开发开源处理器核心、相关知识产权、工具和软件。所有 OpenHW 生态系统的产物都在公开可访问的 GitHub 仓库中开发。核心设计使用 SystemVerilog 定义，并采用 [Solderpad 硬件许可证 0.51 版](http://solderpad.org/licenses/SHL-0.51/) 进行许可。该许可证基于 Apache 许可证 2.0，并促进了商业应用。

在撰写本文时，所有当前可用的核心都属于 [CORE-V](https://github.com/openhwgroup/core-v-cores) 系列。以下是对最重要的 CORE-V 核心的描述。从高层次来看，从用例角度出发，CORE-V 处理器的 CVA 系列可以为网关和边缘节点提供动力，而 CVE4 系列则面向微控制器级别的核心。

#### CVA6 系列

CORE-V CVA6 最初被称为 PULP Ariane 核心，是一个 6 级流水线、单发射、顺序执行的核心，实现了 RV32GC 或 RV64GC 扩展，并具有三个特权级别（M、S、U），以完全支持类 UNIX 操作系统。

#### CVE4 系列

CVE4 系列包含多个核心。我仅描述其中最重要的三个。

*   **CV32E40P：** CORE-V CV32E40P 最初被称为 PULP RI5CY 核心，是一个 32 位、4 级流水线核心，支持 DSP 操作，并实现了 SIMD 扩展等功能。

*   **CV32E40X：** CORE-V CV32E40X 是一个小巧高效的 32 位顺序执行 RISC-V 核心，具有 4 级流水线。CV32E40X 核心面向计算密集型应用，并提供了一个通用扩展接口，你可以利用该接口为核心添加外部自定义扩展。

*   **CV32E40S：** CORE-V CV32E40S 是一个小巧高效的 32 位顺序执行 RISC-V 核心，具有 4 级流水线。CV32E40S 核心面向安全应用，提供机器模式（M）和用户模式（U）、增强的物理内存保护以及多种防篡改功能。

### 微控制器：CORE-V MCU

OpenHW 组织的 [CORE-V MCU 项目](https://docs.openhwgroup.org/projects/core-v-mcu) 旨在构建一个基于 CV32E40P 核心的微控制器。该 MCU 支持 I²C、PWM、SPI 和 UART 等技术。在撰写本文时，最便捷的试用方式是将该芯片作为比特流部署在 Diligent 的 [Nexys A7](https://digilent.com/shop/nexys-a7-fpga-trainer-board-recommended-for-ece-curriculum/) 或 [Genesys2](https://digilent.com/reference/programmable-logic/genesys-2/start) FPGA 开发板上。然而，OpenHW 组织计划在 2023 年推出配备实际物理芯片的开发板以供分发。请关注此页面以获取有关 CORE-V MCU DevKit 项目的更多信息：[`www.openhwgroup.org/core-v-devkits/`](http://www.openhwgroup.org/core-v-devkits/)。

注意

现场可编程门阵列（FPGA）是可编程的半导体器件。换句话说，它们是可以通过重新配置来执行不同逻辑功能的芯片。对于芯片设计者来说，它们是无价的工具，因为可以在不生产实际半导体器件的情况下测试对处理器核心或 MCU 设计的更改。

### 单板计算机

目前许多流行的单板计算机的制造者都将其设计以开源许可证的形式提供。我现在将详细探讨几个示例。

#### Arduino

流行的 Arduino 8 位微控制器可能使用专有的 AVR 微控制器，但其整体设计是开源的。详细的引脚排列图和原理图可在 Arduino 网站上获取，并根据 [知识共享署名-相同方式共享 4.0 国际许可证](https://creativecommons.org/licenses/by-sa/4.0/) 发布。

这些原理图的可用性催生了一个强大的兼容板卡生态系统。

#### micro:bit

接下来是强大的 [BBC micro:​bit](https://microbit.org/)。micro:bit 是一款采用 Cortex-M MCU 的开源单板计算机。它于 2016 年推出，至今已有两个不同的版本。以下是每个版本的主要特性列表：

*   **版本 1：** [Nordic nRF51822](https://www.nordicsemi.com/products/nrf51822) MCU，16 MHz Arm Cortex-M0 核心，256 KB 闪存，16 KB RAM

*   **版本 2：** [Nordic nRF52833](https://www.nordicsemi.com/products/nrf52833) MCU，64 MHz Arm Cortex-M4 核心，512 KB 闪存，128 KB RAM

micro:bit 最初由 [英国广播公司](https://www.bbc.com/)（BBC）与多家合作伙伴共同设计。该项目的目标是鼓励儿童积极参与计算机相关的活动。基于这一重点，这些板卡配备了 LED、按钮、传感器（加速度计、磁力计、温度、光线）以及蓝牙连接功能。版本 2 增加了麦克风、扬声器和触摸传感器。官方板卡配备了一个 25 引脚的边缘连接器，提供 PWM 输出、GPIO 引脚、模拟输入、串行 I/O、SPI 和 I²C。

[Micro:​bit 教育基金会](https://www.microbit.org/about/) 于 2016 年 10 月由最初的 BBC 项目创立。该基金会是一个非营利组织，负责最大化全球学生使用 micro:bit 学习的机会。因此，它引导着 micro:bit 板卡及其相关软件栈的演进。尽管官方板卡仅提供原理图和物料清单，但基金会发布了开源的 [兼容参考设计](https://github.com/microbit-foundation/microbit-reference-design) 和 [详尽的文档](https://tech.microbit.org/hardware/reference-design/)。该参考设计采用 Solderpad 硬件许可证 0.51 版进行许可。它提供与官方版本 1 板卡相同的组件，并且 100% 软件兼容。



#### Raspberry Pi Pico

虽然 Raspberry Pi 单板计算机备受关注，但 Pi Pico 也是一款广受欢迎且文档完善的微控制器。它搭载了 Raspberry Pi 自主研发的 [RP2040 芯片](https://www.raspberrypi.com/documentation/microcontrollers/rp2040.html%2523welcome-to-rp2040)。该芯片配备了两个运行频率为 133 MHz 的 Arm Cortex-M0+ 内核、264 KB 的 SRAM，并支持通过专用总线连接高达 16 MB 的片外 Flash 存储。该微控制器提供 2 个 UART、2 个 SPI 控制器、2 个 I²C 控制器和 16 个 PWM 通道。此外，还包含一个低分辨率温度传感器。

[Raspberry Pi Pico](https://www.raspberrypi.com/documentation/microcontrollers/raspberry-pi-pico.html) 开发板配备 2 MB Flash，并引出 26 个 GPIO 引脚，其中 3 个支持 ADC 功能。尽管 RP2040 微控制器是专有产品，但 Pi Pico 开发板的设计文件是开源的。市面上已有多个第三方厂商推出了同样使用该芯片的开发板。
