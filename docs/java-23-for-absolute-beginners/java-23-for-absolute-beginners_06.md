**堆**内存在运行时用于为对象和 JRE 类分配内存。对象是 JDK 类或开发者定义类的实例。任何使用 `new` 创建的对象都将存储在堆内存中。堆内存中创建的对象可以被应用程序的所有线程访问。堆内存的访问和管理稍微复杂一些，将在**第** **13** **章**（关于垃圾回收）中详细讨论。`-Xms` 和 `-Xmx` JVM 参数用于设置 Java 程序在执行期间堆内存的初始大小和最大大小。堆大小可能因程序创建的对象数量而变化，如果分配给 Java 程序的所有堆内存都已满，则会抛出 `java.lang.OutOfMemoryError`。堆内存的默认大小取决于运行 JVM 的计算机的可用物理内存，其最小值和最大值以及其他附加数据可以从 `java -XX:+PrintFlagsFinal -version` 命令的输出中提取。
