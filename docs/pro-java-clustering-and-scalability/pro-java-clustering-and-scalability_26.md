# 22. 持续交付

持续交付是指能够以可持续的方式，安全、快速地将变更（例如新功能、配置更改、错误修复等）交付到生产环境。

当然，实现这一点是有代价的。整个交付流程（从开发到生产）必须高度自动化。请记住，人类擅长创造性任务，但在重复性任务上表现糟糕。机器不擅长创造性任务，但在重复性任务上表现出色。为什么不充分利用人类和机器两者的优势呢？换句话说，让人类和机器各司其职，做他们最擅长的事！

测试是一项重复、困难且乏味的任务。人类无法在不出错的情况下测试成千上万个用例。但机器可以。而且，机器可以在几秒或几分钟内完成测试任务，而人类可能需要花费好几天。这也意味着，在这类任务上用机器替代人类成本更低。

让人类来测试你的软件，或者手动参与任何本可以自动化的任务，这与软件的持续交付理念格格不入。如果你想实现持续交付，自动化是关键！

现在你能明白为什么自动化测试如此重要了吗？我认为，它们是实现高质量发布、进行持续交付的一种先决条件。在没有良好自动化测试的情况下实施持续交付，只会导致快速地将质量糟糕的代码交付到生产环境。这是你想要的吗？绝对不是！
