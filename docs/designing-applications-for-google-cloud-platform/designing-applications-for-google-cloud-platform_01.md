# 1. 引言

本章将介绍 Google Cloud Platform (GCP) 服务，以及如何使用它们来构建、部署和扩展应用程序。本章将指导您设置 GCP 环境，并简要概述最常见的 GCP 服务。学完本章后，您将能够理解 GCP 的基础知识，创建 GCP 项目，并了解可以使用哪些 GCP 服务来构建和部署应用程序。

您需要创建一个 Google 登录账号，或使用您现有的 Google 账号，以开始免费的 GCP 试用，从而跟随本书中的练习进行操作。

## 在 GCP 上开始免费试用

要在 GCP 上开始免费试用，您可以按照以下步骤操作：

1.  访问 GCP 网站 [`https://cloud.google.com/`](https://cloud.google.com/)，点击网页右上角的“免费开始使用”按钮。

2.  使用您的 Google 账号登录，如果没有 Google 账号，则创建一个新账号。

3.  填写您的个人信息和账单信息。您必须提供有效的信用卡以验证身份，但只有在升级到付费账户时才会被收费。

4.  查看免费试用的条款和条件，然后点击“同意并继续”按钮。

5.  创建账户后，您就可以免费使用 GCP 服务 90 天。在此期间，您将获得 300 美元的赠金，可用于任何 GCP 服务。请注意，某些服务可能不符合免费试用条件，因此在使用前请查看详细信息。

6.  探索 GCP 控制台并尝试一些服务。有许多资源可帮助您入门，包括文档、教程和社区论坛。

请记住在试用期内监控您的使用情况和账单，以避免意外收费。如果您决定升级到付费账户，可以随时通过向您的项目添加结算账户来升级。

一旦您开始免费试用，GCP 项目“我的第一个项目”将自动创建。您可以根据需要创建更多项目。

GCP 中的项目用于组织资源。GCP 中的每个项目都有自己的资源集和账单，您可以根据项目跟踪账单。

在 GCP 中，一个*项目*就像一个盒子或文件夹，包含与特定应用程序、系统或工作负载相关的所有资源、服务和配置。项目充当资源的边界，以确保它们被隔离并且可以独立管理。

项目用于在 GCP 中组织资源、定义资源所有权和访问权限、管理账单和使用情况，并促进团队成员之间的协作。您可以将项目视为资源的逻辑分组，这使得管理、保护和监控它们更加容易。

不使用项目就无法使用 GCP。您必须至少创建一个项目才能访问 GCP 服务和资源。当您注册 GCP 账户时，会自动创建一个默认项目，用于账单和其他管理目的。您还可以为工作负载和应用程序创建其他项目。




GCP 中的项目可以通过多种方式相互关联。例如，你可以关联项目以启用共享资源，例如共享 VPC 网络或结算账户。你还可以关联项目以控制跨多个项目的资源访问，或向用户或服务账号授予权限。

以下是关于 GCP 项目需要了解的一些关键事项：

*   项目必须使用 GCP 服务和资源。

*   每个项目在所有 GCP 服务中都有一个全局唯一的 ID、名称和编号。

*   一个项目可以包含多个资源，例如计算实例、存储桶和数据库。

*   项目可以组织到文件夹和层级结构中，以管理资源所有权和访问权限。

*   结算和使用情况在项目级别进行管理。

*   项目可以相互关联，以启用共享资源和访问控制。

*   每个项目都有自己的身份和访问管理 (IAM) 策略，用于控制谁可以访问和管理项目的资源。

*   你可以为每个项目配置监控、日志记录和告警，以了解其运行状况和性能。

*   项目可用于不同目的，例如开发、测试、预发布或生产。

*   你可以使用 GCP 控制台、Cloud SDK 或 API 创建和管理项目。

*   当项目不再需要时可以删除，但在此之前需要移除所有关联的资源和数据。

在使用 GCP 项目时，跟踪其配置和使用情况，并遵循安全、可扩展性和成本优化的最佳实践非常重要。你还应考虑使用 Terraform、Cloud Deployment Manager 或 Kubernetes Engine 等工具，将基础设施作为代码进行管理，并确保跨多个项目和环境的一致性和可重复性。

## GCP 及其服务概览

*GCP* *架构*指的是构成 Google Cloud Platform 的基础设施和服务的设计与布局。从高层来看，GCP 架构包括计算、存储、数据库、网络和安全。它被设计为可扩展、可靠且安全的。它使用多层抽象和隔离机制，确保应用程序和服务能够在同一基础设施上独立且安全地运行。它还提供了各种工具和服务，使开发人员能够自动化开发流程、监控性能并实时排查问题。

GCP *区域*和*可用区*是 GCP 资源的特定地理位置。GCP 区域是托管资源的特定位置。区域由一个或多个可用区（AZ）组成，这些可用区是物理上分离的位置，称为*数据中心*。可用区被设计为相互独立，因此如果一个可用区发生故障，不会影响同一区域内另一个可用区中资源的可用性。GCP 区域和可用区非常重要，因为它们会影响应用程序的性能和可用性。当您创建 GCP 资源时，需要指定要创建资源的区域和可用区。

GCP 的定价基于按需付费模式，这意味着您只需为使用的资源和服务付费。成本取决于多个因素，包括所用资源的类型和数量、使用时长以及资源的位置。价格可能因区域和可用区而异，某些区域和可用区的成本会高于其他区域。GCP 通过承诺在一定时期内使用一定量的 Compute Engine 和云存储来提供承诺使用折扣。

GCP 提供多种服务，包括计算、存储、网络、大数据、机器学习、物联网（IoT）、安全和应用程序开发。让我们快速了解一些最受欢迎的服务。

### Google Compute Engine

Google Compute Engine (GCE) 是 GCP 的一部分，它提供可扩展、高可用且高性能的虚拟机，用于运行应用程序和服务。它使客户能够创建、配置和管理在 Google 基础设施上运行的虚拟机（VM）。GCE 提供一系列可配置选项，包括可定制的机器类型、磁盘存储选项和操作系统，包括流行的 Linux 发行版和 Microsoft Windows。GCE 还提供高级网络功能，如负载均衡、防火墙和自动备份，使其成为部署和管理应用程序与服务的理想平台。

GCE 与其他 GCP 服务（如 Google Kubernetes Engine、Google Cloud Storage、Google Cloud SQL 和 Cloud Pub/Sub）的集成并非受限，但可能取决于网络连接和配置。这种集成能力使得构建复杂的多层应用程序变得容易。

让我们看一个例子：GCE 实例与其需要通信的其他 GCP 服务位于不同的区域或可用区。在这种情况下，可能会出现延迟和性能问题。如果网络配置设置不当，可能会影响 GCE 实例与其他 GCP 服务之间的连接和集成。为了优化 GCE 实例与其他 GCP 服务之间的集成，将网络连接和配置作为架构设计的一部分来考虑非常重要，这包括选择合适的区域和可用区、设置正确的网络配置，以及使用负载均衡和 Cloud CDN 等服务来提高性能和可用性。

### Google Kubernetes Engine

Google Kubernetes Engine（GKE）是 GCP 提供的一项完全托管的 Kubernetes 服务，可让您轻松利用 Google 基础设施部署、管理和扩缩容器化应用。GKE 为部署、管理和扩缩容器提供了完整的解决方案，使开发者能够专注于编写代码和交付功能。它还提供自动升级、自动扩缩和自修复等功能，让开发者可以专注于应用本身，而无需操心底层基础设施。此外，GKE 还与其他 GCP 服务（如 Cloud Load Balancing、Cloud Operations Suite、Cloud Monitoring and Logging 以及 Cloud DNS）集成，从而可以轻松地在单一平台上管理完整的应用栈。

*容器化应用*是指将其所有依赖项打包成一个称为*容器*的单一单元的应用。容器彼此隔离，也与宿主机操作系统隔离，这使得它们成为更高效、更安全的部署应用方式。可以将容器想象成一个盒子，里面包含了应用运行所需的一切，例如代码、库和设置。这使得将应用部署到不同环境（例如本地环境、云端或混合环境）变得非常容易。容器也比传统虚拟机更高效。虚拟机需要完整的操作系统，而容器只需要应用的资源。这可以节省硬件成本并提高性能。容器比传统虚拟机更安全。容器彼此隔离，也与宿主机操作系统隔离，这使得恶意软件更难传播，从而使容器成为需要高度安全的应用的理想选择。

Google Cloud Spanner 是 Google Cloud 提供的一种高度可扩展、全球分布式的托管关系型数据库服务。它旨在跨所有节点提供水平扩展性和强一致性，使其成为需要强一致性和高性能的应用程序的绝佳选择。凭借其对 SQL、事务和二级索引的支持，Spanner 提供了熟悉的数据库抽象，使开发人员易于使用并集成到现有应用程序中。Spanner 的全球分布式架构、自动故障转移和低延迟使其成为高可用性和关键任务应用程序的理想选择。凭借其可扩展性、可靠性和一致性，Cloud Spanner 为希望实现数据架构现代化并将其应用程序迁移到云端的组织，提供了一种经济高效且可扩展的传统关系型数据库替代方案。

## Java 及其功能概述

Java 是一种流行的编程语言，用于在 GCP 上构建各种应用。此外，GCP 还提供了许多服务和工具，你可以与 Java 结合使用来构建、部署和扩展应用。以下是适用于 Java 应用的 GCP 服务的一些示例：

*   *Google App Engine*：一个完全托管的平台，用于构建和托管多种语言（包括 Java）的 Web 应用。

*   *Google Cloud Storage*：一个完全托管的对象存储服务，可以为 Java 应用存储和检索数据。

*   *Google Cloud SQL*：一个完全托管的关系型数据库服务，可以为 Java 应用存储和检索数据。

*   *Google Cloud Datastore*：一个完全托管的 NoSQL 文档数据库服务，可以为 Java 应用存储和检索数据。

*   *Google Cloud Bigtable*：一个完全托管的高性能 NoSQL 数据库服务，可以为 Java 应用存储和检索大量数据。

*   *Google Kubernetes Engine*：你可以使用 GKE 创建一个完全托管的 Kubernetes 集群。你可以将容器化的 Java 应用部署到 Google Kubernetes Engine。

GCP 还提供了多种工具和库，可用于构建、部署和扩展 Java 应用，包括 Google Cloud SDK for Java，它提供了一套用于与 GCP 服务交互的命令行工具和库。接下来，我们来谈谈使用 Java 进行 GCP 应用开发的优势。



## 使用 GCP 进行应用开发的优势

GCP 提供了一系列用于开发和部署 Java 应用的工具和服务。它为各种规模的组织提供了可扩展、安全且经济高效的解决方案。在本书中，我们将探讨使用 GCP 进行 Java 开发的优势，以及如何在 GCP 平台上设计、构建和优化 Java 应用。

使用 GCP 进行 Java 开发的关键优势之一是可扩展性。GCP 提供自动扩展和负载均衡，确保您的应用能够应对流量高峰并提供一致的性能。这消除了手动干预的需要，使开发人员能够专注于构建新功能和改进应用。

使用 GCP 的另一个优势是安全性。GCP 提供安全的基础设施和一系列安全功能，例如访问控制和加密，以帮助保护您的应用和数据安全。这降低了数据泄露的风险，并有助于确保敏感信息保持机密。

GCP 还提供了一系列用于监控和日志记录的工具和服务，使识别和解决性能问题变得更加容易。借助 GCP，您可以轻松监控应用的性能，并深入了解用户如何与之交互。这可以帮助您确定需要改进的领域，并优化应用的性能。

GCP 提供经济高效且灵活的定价选项，使您只需为所需内容付费。这使得成本控制更加容易，并为需求不断变化的组织提供了更灵活的解决方案。

GCP 为包括 Java 开发在内的各种编程语言提供了全面且经济高效的解决方案，并提供了一系列工具和服务来帮助组织构建、部署和优化其应用。GCP 提供了一个专门用于 Java 开发的 BOM，称为 Google Cloud Libraries BOM。此 BOM 列出了在 GCP 上构建和运行 Java 应用所需的所有库和依赖项，包括 Spring、Apache Beam 等流行库。

在接下来的章节中，我们将更详细地探讨这些优势，并提供在 GCP 上设计、构建和优化 Java 应用的示例和最佳实践。

## 微服务概述

微服务架构是一种通过将软件应用分解为更小、独立且自包含的服务来构建软件应用的方式，如图 1-34 所示。每个微服务都有其特定的功能，通过 API 与其他服务通信，并且可以独立开发、部署和扩展。微服务方法可以实现更快、更敏捷的开发，提高可扩展性和弹性，并简化应用的维护和演进。

![](img/603470_1_En_1_Fig34_HTML.jpg)

微服务组件的流程图显示了一个客户端和两个微服务。每个微服务进一步连接到不同数量的微服务。最后一行有四个微服务，分别连接到 SQL、NoSQL、键值存储和 SQL。

微服务架构

在微服务架构中，不同的微服务可以使用不同的编程语言和技术进行开发，从而更容易为任务采用最佳工具。Java 是开发微服务的流行选择，因为它提供了丰富的工具、框架和库来构建可扩展且可靠的应用。

GCP 提供了许多非常适合开发和部署基于 Java 的微服务的服务和工具。例如，Google Kubernetes Engine 为部署和扩展容器化应用提供了托管环境，而 Operations Suite 为在 GCP 上运行的 Java 应用提供了强大的监控和日志记录功能。此外，GCP 还提供了多种数据库选项，包括 Cloud SQL、Cloud Datastore 和 Cloud Spanner，可用于存储微服务的数据。

使用 GCP 进行 Java 开发可以带来许多好处，包括访问一系列强大且可扩展的服务、简化的开发和部署流程，以及轻松扩展和监控应用的能力。
