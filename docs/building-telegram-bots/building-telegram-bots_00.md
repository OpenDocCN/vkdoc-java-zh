请注意，如果你将消息对象保留在 `let` 块中声明，编译器会“抱怨”（双关语）该消息无法更新，因为它是不可变的（图 2-17）。

```nim
import telebot, asyncdispatch, options

const API_KEY = slurp("secret.key")

proc updateHandler(b: Telebot, u: Update) {.async.} =
  var response = u.message.get
  if response.text.isSome:
    let
      text = response.text.get
    var message = newMessage(response.chat.id, text)
    message.replyToMessageId = response.messageId
    discard await b.send(message)

let bot = newTeleBot(API_KEY)
bot.onUpdate(updateHandler)
bot.poll(300)
```

**D 语言并发的一些要点**

从 Java 线程的烦恼中走出来，你会对 D 语言的并发特性感到兴奋。D 语言中的线程可以像常规 D 函数一样被创建和*生成*。

**简单线程**

负责并发的命名空间恰如其分地命名为 `std.concurrency`，我们将在这个第一个线程测试中使用它。

要从一个函数生成一个线程，我们将简单地使用 `spawn`，它接受一个函数的引用作为参数。这是通过在任何标准 D 函数前面使用 & 符号 (`&`) 来完成的。代码片段的其余部分很容易理解，所以这里是带线程的程序：

```d
import std.concurrency : spawn, thisTid;
import std.stdio : writeln;

void worker()
{
    writeln("Thread ", thisTid);
}

void main()
{
    int TOTAL_WORKERS=5;
    for (int i = 0; i < TOTAL_WORKERS; ++i) {
        spawn(&worker);
    }
}
```

**共享状态**

我们 D 语言线程子节的最后一个示例是关于共享状态的，其中有一个公共计数器，由每个线程分别递增。要创建这样一个共享数据，D 语言要求你特别将数据标记为 `shared`。

一旦完成，你可以使用 `core.atomic` 命名空间中的操作来安全地更新该共享数据的值。我们将使用的原子操作是 `atomicOp` 和 `+=` 的组合，编写如下：

```d
atomicOp!"+="
```

这将创建一个线程安全的 `+=` 函数版本。

让我们看看这在以下代码片段中是如何实际使用的。

```d
import std.concurrency : spawn, thisTid;
import std.stdio : writeln;
import core.atomic : atomicOp;

shared int flag = 0;

void worker()
{
    atomicOp!"+="(flag, 1);
    writeln("Thread ", thisTid,":global>",flag);
}

void main()
{
    for (int i = 0; i < 20; ++i) {
        spawn(&worker);
    }
}
```

**我对斐波那契的爱**

在永无止境地寻找最有效的斐波那契代码片段的过程中，我们可能在 D 语言中找到了一个令人瞩目的东西。第一个片段是通常的迭代实现，其中算法循环遍历不同的值并迭代计算斐波那契序列的元素。

```d
auto fib(const int n)
{
    import std.bigint;
    if (n == 0)
        return BigInt(0);
    if (n == 1)
        return BigInt(1);
    BigInt next;
    for (BigInt i = 0, j = 1, count = 1; count < n; ++count)
    {
        next = i + j;
        i = j;
        j = next;
    }
    return next;
}

void main()
{
    import std.stdio;
    writeln(fib(100_000));
}
```

第 5 章 第 5 周：D 语言

当你执行上述代码时，你会发现它非常快。但下一个实现（在论坛中找到，由 Biotronic 实现）简直是快得惊人。

```d
auto fib(ulong n) {
    import std.bigint : BigInt;
    import std.meta : AliasSeq;
    import std.typecons : tuple;
    BigInt a = 0;
    BigInt b = 1;
    auto bit = 63;
    while (bit > 0) {
        AliasSeq!(a, b) = tuple(
            a * (2*b - a),
            a*a + b*b);
        if (n & (BigInt(1) << bit)) {
            AliasSeq!(a, b) = tuple(b, a+b);
        }
        --bit;
    }
    return a;
}
```

这个生成斐波那契数的版本使用了以下内容：

• 一种不同的生成斐波那契数的基本算法，称为快速平方算法。
• 来自 `std.meta` 命名空间的 `AliasSeq`，与模板一起工作，允许在编译器的帮助下进行编译时改进。

• 你之前没有见过的主要导入是来自 `telega` 库的 `botapi`。我们使用包含在 `secret.key` 文本文件中的密钥创建一个机器人客户端。
• 更新以消息批次的形式出现，因此在 `listenUpdates` 函数中，我们创建一个循环来逐一处理这些消息。
• 为了告诉机器人 API 我们已完成更新处理，我们调用 `updateProcessed`，并将原始更新作为参数。

```d
    auto api = new BotApi(token);
```

要编译和安装该库，你需要一些工具，例如 `make`、`cmake`、`ssl` 开发库和 `boost`。

```
apt-get install g++ make binutils cmake libssl-dev libboost-system-dev
```

```
cmake .
make
sudo make install
```

安装默认到 `/usr/lib/`，你可以通过查找以下文件的存在来检查 Telegram 库是否正确安装：

```
/usr/local/lib/libTgBot.a
```
