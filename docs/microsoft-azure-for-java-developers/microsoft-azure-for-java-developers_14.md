# 13. 使用 Azure DevOps 和 Azure CLI 预配资源

基础设施即代码（IaC）是基于云的企业中广泛采用的做法。你可以使用脚本和 DevOps 工具自动创建云基础设施。你需要创建这些脚本或解决方案一次，并在多个环境中重复使用它们。这种方法可以为你节省大量时间和精力。有许多可用的基础设施即代码（IaC）实用程序和工具，可以帮助你自动预配 Azure 基础设施。

在上一章中，我们学习了 Azure Active Directory 的基本概念。然后，我们创建了一个可以使用 Azure Active Directory 执行用户身份验证和授权的 Java 应用程序。在本章中，我们将探讨 Azure 的基础设施即代码（IaC）选项，然后使用 Azure CLI 和 Azure DevOps 启动一个 Azure Web 应用。

## 结构

在本章中，我们将讨论使用 Azure DevOps 和 Azure CLI 创建基础设施的以下方面：

*   Azure 上的基础设施即代码（IaC）简介

*   创建用于启动 Azure Web 应用的 Azure CLI 脚本

*   创建用于启动 Azure Web 应用的 Azure DevOps 管道

## 目标

学习本章后，你应该能够获得以下知识：

*   探索可用于通过自动化创建 Azure 基础设施的选项

*   使用 Azure DevOps 和 Azure CLI 创建 Azure 资源



## Azure 基础设施即代码 (IaC) 简介

Azure 门户可帮助您轻松创建 Azure 资源。您会获得一个直观且基于向导的用户界面，并可使用它来创建资源。但是，您需要手动登录 Azure 门户，然后在门户中提供资源详细信息并创建资源。如果您只有一个环境，这种方法效果很好。然而，企业通常拥有多个环境，例如开发、测试、验收和生产环境。如果您使用 Azure 门户在这些环境中创建资源，最终可能会进行许多重复性活动。您需要在开发环境中为应用程序创建资源，然后在其他环境中创建相同的资源集。这种方法工作量大，并且由于涉及手动操作，存在引入错误的风险。但是，您可以自动执行资源创建活动，并在多个环境中触发自动化。这种方法将为您节省大量时间，并且不会出错。与使用 Azure 门户创建资源的方法相比，您可以更快地创建好环境。

以下是一些流行的用于自动化创建 Azure 资源的**基础设施即代码 (IaC)** 方案：

*   Azure CLI
*   Azure PowerShell
*   ARM 模板
*   Terraform
*   Bicep

Azure CLI 是一个跨平台方案，您可以在 Windows、Linux 或 Mac 系统上启动 Azure 资源。您可以从命令提示符、bash 提示符或 PowerShell 提示符运行 Azure CLI 脚本。

Azure PowerShell 为您提供了有助于启动 Azure 资源的 PowerShell 模块。如果您使用可在 PowerShell 6 及更高版本上运行的 Azure PowerShell Core，则可以构建跨平台脚本。

使用 ARM 模板，您可以将 Azure 基础设施表示为 JSON 模板，然后将该模板部署到 Azure 环境并启动您的 Azure 资源。Azure CLI 和 Azure PowerShell 都公开了可用于部署这些模板的命令。

Azure PowerShell、Azure CLI 和 ARM 模板是 Microsoft 提供的用于创建 Azure 资源的方案。Terraform 是一个第三方方案，可帮助您编写脚本来创建 Azure 资源。

您可以使用 Azure DevOps 管道来调用使用 Azure CLI、PowerShell、Terraform 或 ARM 模板开发的脚本，以启动 Azure 基础设施。在这种情况下，您可以构建发布管道以在各种环境中启动基础设施。这种方法将帮助您构建一键式自动化，从而启动 Azure 基础设施并部署应用程序。

## 创建用于启动 Azure WebApp 的 Azure CLI 脚本

让我们创建一个 Azure CLI 脚本来启动 Azure WebApp。作为先决条件，您应该在系统中安装 Azure CLI。请遵循清单 13-1 中提供的安装步骤。

```
清单 13-1
Azure CLI 安装 URL
```

让我们打开 PowerShell、命令提示符或 bash 提示符。第一步，我们需要使用清单 13-2 中的命令登录到 Azure。

```
az login
清单 13-2
登录到 Azure
```

系统会提示您在浏览器中提供凭据，如图 13-1 所示。

图 13-1

登录到 Azure

登录到 Azure 后，您需要选择一个订阅来创建资源。执行清单 13-3 中的命令来选择您的订阅。将 `{subscription}` 替换为您的订阅名称或订阅 ID。您可以使用 `az account show` 命令列出本章中的所有订阅。

```
az account set -s "{subscription}"
清单 13-3
选择一个订阅
```

您需要创建一个资源组，然后在该资源组内创建 WebApp。您也可以选择使用现有的资源组。在这种情况下，您可以忽略此步骤。将 `{name}` 替换为资源组名称。这里，我们在 eastus 位置创建资源组。您可以选择符合您需求的 Azure 位置。清单 13-4 演示了创建资源组的命令。

```
az group create --location eastus --name {name}
清单 13-4
创建一个资源组
```

我们需要为 WebApp 创建一个应用服务计划。执行清单 13-5 中的命令，了解如何使用应用服务计划的 create 命令。您还将了解此处可用的各种应用服务计划 SKU 以及其他详细信息。

```
az appservice plan create --help
清单 13-5
创建应用服务计划帮助
```

执行清单 13-6 中的命令，在 eastus 区域创建一个标准 SKU (S1) 的应用服务计划。将 `{resourcegroup-name}` 替换为资源组的名称，将 `{name}` 替换为应用服务计划的名称。

```
az appservice plan create --resource-group {resourcegroup-name} --name {name} --sku S1 --location eastus
清单 13-6
创建一个应用服务计划
```

执行清单 13-7 中的命令来创建一个 WebApp。将 `{name}` 替换为 WebApp 的名称，将 `{resourcegroup-name}` 替换为资源组的名称，将 `{plan-name}` 替换为应用服务计划的名称。

```
az webapp create --name {name} --resource-group {resourcegroup-name} --plan {plan-name}  --runtime "java:11:Java SE:11"
清单 13-7
创建一个 WebApp
```

您的 WebApp 将在提供的资源组中创建。您可以使用命令 `curl -s -o /dev/null -w "%{http_code}" https://{name}.azurewebsites.net/`，其中 `{name}` 是 WebApp 的名称。您应该会收到响应代码 200。
