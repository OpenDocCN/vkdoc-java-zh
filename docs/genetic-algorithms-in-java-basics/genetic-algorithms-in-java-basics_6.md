# 优化

在本章中，我们将探讨常用于优化遗传算法的不同技术。随着待解决问题的复杂化，额外的优化技术变得越来越重要。一个经过良好优化的算法在解决较大问题时可以节省数小时甚至数天的时间；因此，当问题达到一定复杂度时，优化技术至关重要。

除了探讨一些常见的优化技术外，本章还将使用前几章案例研究中的遗传算法，提供几个实现示例。

## 自适应遗传算法

自适应遗传算法（AGA）是遗传算法中一个流行的子集，在适当的情况下，它能比标准实现提供显著的性能提升。正如我们在前几章中所了解的，决定遗传算法性能的一个关键因素是其参数的配置方式。在构建高效遗传算法时，我们已经讨论了为变异率和交叉率找到正确值的重要性。通常，配置参数需要一些试错过程，并结合一些直觉，才能最终达到令人满意的配置。自适应遗传算法之所以有用，是因为它们能够根据算法的状态自动调整这些参数，从而辅助参数调优。这些参数调整发生在遗传算法运行期间，有望在执行过程中的任何特定时刻使用最佳参数。正是这种对算法参数的持续自适应调整，常常能带来遗传算法性能的提升。

自适应遗传算法利用平均种群适应度和种群当前最佳适应度等信息，以最适合其当前状态的方式计算和更新其参数。例如，通过将任何特定个体与种群中当前最适应个体进行比较，可以评估该个体相对于当前最佳个体的表现。通常，我们希望增加保存表现良好个体的机会，并减少保存表现不佳个体的机会。实现这一点的一种方法是允许算法自适应地更新变异率。

不幸的是，事情并没有那么简单。过了一段时间，种群将开始收敛，个体将开始更接近搜索空间中的单个点。当这种情况发生时，由于个体之间差异很小，搜索的进展可能会停滞。在这种情况下，稍微提高变异率可能是有效的，从而鼓励在搜索空间内探索其他区域。

我们可以通过计算当前最佳适应度与平均种群适应度之间的差异来判断算法是否已开始收敛。当平均种群适应度接近当前最佳适应度时，我们就知道种群已开始在搜索空间的一个小区域周围收敛。

然而，自适应遗传算法不仅可以用于调整变异率。类似的技术也可以应用于调整遗传算法的其他参数，例如交叉率，以便根据需要提供进一步的改进。



### 实现

与遗传算法的许多方面一样，更新参数的最佳方式通常需要一些实验。我们将探讨一种较为常见的方法，如果你愿意，可以自行尝试其他方法。

如前所述，在计算任何给定个体的突变率时，需要考虑的两个最重要特征是当前个体的表现如何，以及整个种群的整体表现如何。我们将用于评估这两个特征并更新突变率的算法如下：

p [m] = (f [max] - f [i] ) / (f [max] – f [avg] ) * m, f [i] > f [avg]

p [m] = m, f [i] ≤ f [avg]

当个体的适应度高于种群的平均适应度时，我们取种群中的最佳适应度（f[max]），并计算其与当前个体适应度（f[i]）的差值。然后，我们计算最大种群适应度与平均种群适应度（f[avg]）的差值，并将这两个值相除。我们可以用这个值来缩放初始化时设定的突变率。如果个体的适应度等于或低于种群的平均适应度，我们则直接使用初始化时设定的突变率。

为了方便起见，我们可以将新的自适应遗传算法代码集成到之前的课程表调度器代码中。首先，我们需要添加一个新方法来获取种群的平均适应度。为此，可以在 `Population` 类中的任意位置添加以下方法：

     `* 获取平均适应度`

     `*`

     `*/`

现在，我们可以通过更新突变函数来使用自适应突变算法，从而完成实现：

`/**`

     `* 对种群应用突变`

     `*`

     `* @param population 种群`

     `* @param timetable 时间表`

     `* @return 突变后的种群`

     `*/`

    `public Population mutatePopulation(Population population, Timetable timetable){`

        `// 初始化新种群`

        `Population newPopulation = new Population(this.populationSize);`

        `// 获取最佳适应度`

        `double bestFitness = population.getFittest(0).getFitness();`

        `// 按适应度遍历当前种群`

        `for (int populationIndex = 0; populationIndex < population.size(); populationIndex++) {`

            `Individual individual = population.getFittest(populationIndex);`

            `// 创建随机个体用于基因交换`

            `Individual randomIndividual = new Individual(timetable);`

            `// 计算自适应突变率`

            `double adaptiveMutationRate = this.mutationRate;`

            `if (individual.getFitness() > population.getAvgFitness()) {`

                `double fitnessDelta1 = bestFitness - individual.getFitness();`

                `double fitnessDelta2 = bestFitness - population.getAvgFitness();`

                `adaptiveMutationRate = (fitnessDelta1 / fitnessDelta2) * this.mutationRate;`

            `}`

            `// 遍历个体的基因`

            `for (int geneIndex = 0; geneIndex < individual.getChromosomeLength(); geneIndex++) {`

                `// 如果是精英个体则跳过突变`

                `if (populationIndex > this.elitismCount) {`

                    `// 该基因是否需要突变？`

                    `if (adaptiveMutationRate > Math.random()) {`

                        `// 交换为新基因`

                        `individual.setGene(geneIndex, randomIndividual.getGene(geneIndex));`

                    `}`

                `}`

            `}`

            `// 将个体添加到种群`

            `newPopulation.setIndividual(populationIndex, individual);`

        `}`

        `// 返回突变后的种群`

        `return newPopulation;`

    `}`

这个新的 `mutatePopulation` 方法与原始方法基本相同，只是增加了实现上述算法的自适应突变代码。

当启用自适应突变来初始化遗传算法时，所使用的突变率将是最大可能的突变率，并根据当前个体和整个种群的适应度进行缩放。因此，较高的初始突变率可能是有益的。

## 多重启发式

在优化遗传算法时，实现一个次要启发式是另一种常见方法，可以在特定条件下显著提升性能。在遗传算法中实现第二个启发式，使我们能够将多种启发式方法的最佳特性结合到一个算法中，从而进一步控制搜索策略和性能。

两种常被集成到遗传算法中的流行启发式是模拟退火和禁忌搜索。模拟退火是一种以冶金学中的退火过程为模型的搜索启发式。简而言之，它是一种爬山算法，旨在逐步降低接受较差解的速率。在遗传算法的背景下，模拟退火会随时间降低突变率和/或交叉率。

另一方面，禁忌搜索是一种搜索算法，它维护一个“禁忌”解列表，防止算法返回搜索空间中已知较弱的已访问区域。这个禁忌列表有助于算法避免反复考虑之前发现且已知较弱的解。

通常，只有在包含多重启发式能够为搜索过程带来某些必要改进的情况下，才会采用这种方法。例如，如果遗传算法在搜索空间中收敛过快，在算法中引入模拟退火可能有助于控制算法收敛的速度。



### 实现

让我们通过将模拟退火算法与遗传算法相结合，快速了解一个多启发式算法的示例。如前所述，模拟退火算法是一种爬山算法，它在初始阶段以较高的概率接受较差的解；然后随着算法的运行，它会逐渐降低接受较差解的概率。

将这一特性融入遗传算法的最简单方法之一，是调整变异率和交叉率，使其从较高的初始值开始，然后随着算法的推进逐渐降低。这种初始的高变异率和交叉率会使遗传算法搜索搜索空间中的大片区域。随后，随着变异率和交叉率逐渐降低，遗传算法应开始将搜索重点集中在适应度值较高的搜索空间区域。

为了改变变异和交叉的概率，我们使用一个温度变量，该变量初始值较高（即“热”），并随着算法的运行逐渐降低（即“冷却”）。这种加热和冷却技术直接受启发于冶金学中的退火过程。每一代之后，温度会略微降低，从而降低变异和交叉的概率。

开始实现时，我们需要在 `GeneticAlgorithm` 类中创建两个新变量。`coolingRate` 应设置为一个很小的分数，通常为 0.001 或更小——不过这个数字取决于你预期运行的代数以及你希望模拟退火算法的激进程度。

`private double coolingRate;`

    `}`

## 性能提升

除了改进搜索启发式算法，还有其他优化遗传算法的方法。其中可能最有效的方法之一就是编写高效的代码。在构建需要运行成千上万代的遗传算法时，即使每代处理时间只减少零点几秒，也能大幅缩短总运行时间。

### 适应度函数设计

由于适应度函数通常是遗传算法中计算量最大的部分，因此将代码改进的重点放在适应度函数上，以获得最佳的性能回报是合理的。

在改进适应度函数之前，最好先确保它能充分代表问题。遗传算法使用其适应度函数来衡量搜索空间中应重点搜索的最佳区域。这意味着设计不佳的适应度函数会对遗传算法的搜索能力和整体性能产生巨大的负面影响。例如，假设构建了一个遗传算法来设计汽车面板，但评估该面板的适应度函数完全是通过测量汽车的最高速度来进行的。如果该面板还需要满足一定的耐用性、人体工程学约束以及足够的空气动力学性能，那么这个过于简单的适应度函数可能无法提供足够的适应度值。

### 并行处理

现代计算机通常配备有多个独立的处理单元或“核心”。与标准的单核系统不同，多核系统能够利用额外的核心同时处理多个计算任务。这意味着任何设计良好的应用程序都应该能够利用这一特性，将其处理需求分布到可用的额外处理核心上。对于某些应用程序，这可能简单到在一个核心上处理与图形用户界面相关的计算，而在另一个核心上处理所有其他计算。

支持多核系统的优势是在现代计算机上实现性能提升的一种简单而有效的方法。正如我们之前讨论的，适应度函数通常是遗传算法的瓶颈。这使其成为多核优化的理想候选。通过使用多个核心，可以同时计算多个个体的适应度，当每代通常有数百个个体需要评估时，这将产生巨大的差异。

幸运的是，Java 8 提供了一些非常有用的库，使得在我们的遗传算法中支持并行处理变得更加容易。使用 `IntStream`，我们可以在适应度函数中实现并行处理，而无需担心并行处理的细节（例如需要支持的核心数量）；相反，它会根据可用的核心数量创建最佳数量的线程。

你可能想知道，为什么在第 5 章中，`GeneticAlgorithm` 的 `calcFitness` 方法在使用 `Timetable` 对象之前会先克隆它。当为并行处理编写多线程应用程序时，需要确保一个线程中的对象不会影响另一个线程中的对象。在这种情况下，一个线程对 `timetable` 对象所做的更改可能会对同时使用同一对象的其他线程产生意外结果——首先克隆 `Timetable` 允许我们为每个线程提供其自己的对象。

我们可以通过修改 `GeneticAlgorithm` 的 `evalPopulation` 方法，使其使用 Java 的 `IntStream`，从而在第 5 章的类调度器中利用多线程的优势：

`/**`

     `* 评估种群`

     `*`

     `* @param population`

     `* @param timetable`

     `*/`

    `public void evalPopulation(Population population, Timetable timetable){`

        `IntStream.range(0, population.size()).parallel()`

         `.forEach(i -> this.calcFitness(population.getIndividual(i), timetable));`

        `double populationFitness = 0;`

        `// 循环遍历种群中的个体，评估个体并累加种群适应度`

        `for (Individual individual : population.getIndividuals()) {`

            `populationFitness += individual.getFitness();`

        `}`

        `population.setPopulationFitness(populationFitness);`

    `}`

现在，如果系统支持多核，`calcFitness` 函数就能够跨多个核心运行。

由于本书中介绍的遗传算法使用了相当简单的适应度函数，并行处理可能不会带来太大的性能提升。测试并行处理能在多大程度上提升遗传算法性能的一个好方法，可能是在适应度函数中添加对 `Thread.sleep()` 的调用。这将模拟一个需要较长时间才能完成执行的适应度函数。



### 适应度值哈希

如前所述，适应度函数通常是遗传算法中计算开销最大的部分。因此，即使对适应度函数进行微小的改进，也能对性能产生显著影响。值哈希是另一种方法，它通过将先前计算过的适应度值存储在哈希表中，来减少计算适应度值所花费的时间。在大型分布式系统中，你可以使用集中式缓存服务（例如 Redis 或 memcached）来实现同样的目的。

在执行过程中，由于个体的随机变异和重组，先前找到的解决方案偶尔会被重新访问。随着遗传算法收敛并开始在搜索空间中越来越小的区域内寻找解决方案，这种对解决方案的重新访问会变得更加常见。

每次重新访问一个解决方案时，都需要重新计算其适应度值，这浪费了处理能力在重复的、重复的计算上。幸运的是，这可以通过在计算后将适应度值存储在哈希表中来轻松解决。当重新访问一个先前访问过的解决方案时，其适应度值可以直接从哈希表中获取，从而避免了重新计算的需要。

要将适应度值哈希添加到你的代码中，首先在 `GeneticAlgorithm` 类中创建适应度哈希表：

`// 创建适应度哈希表`

`private Map<Individual, Double> fitnessHash = Collections.synchronizedMap(`

`new LinkedHashMap<Individual, Double>() {`

`@Override`

`protected boolean removeEldestEntry(Entry<Individual, Double> eldest) {`

`// 最多存储 1000 个适应度值`

`return this.size() > 1000;`

`}`

`});`

在这个例子中，哈希表在开始移除最旧的值之前，最多会存储 1000 个适应度值。这可以根据需要更改，以实现最佳的性能权衡。虽然更大的哈希表可以容纳更多的适应度值，但代价是内存使用量增加。

现在，可以添加 `get` 和 `put` 方法来检索和存储适应度值。这可以通过如下更新 `calcFitness` 方法来实现。请注意，我们移除了上一节中的 `IntStream` 代码，以便我们可以一次评估一个改进。

`/**`

`* 计算个体的适应度值`

`*`

`* @param individual`

`* @param timetable`

`* @return fitness`

`*/`

`public double calcFitness(Individual individual, Timetable timetable){`

`Double storedFitness = this.fitnessHash.get(individual);`

`if (storedFitness != null) {`

`return storedFitness;`

`}`

`// 为线程创建新的时间表对象`

`Timetable threadTimetable = new Timetable(timetable);`

`threadTimetable.createClasses(individual);`

`// 计算适应度`

`int clashes = threadTimetable.calcClashes();`

`double fitness = 1 / (double) (clashes + 1);`

`individual.setFitness(fitness);`

`// 将适应度存储在哈希表中`

`this.fitnessHash.put(individual, fitness);`

`return fitness;`

`}`

最后，由于我们使用 `Individual` 对象作为哈希表的键，我们需要重写 `Individual` 类的 `equals` 和 `hashCode` 方法。这是因为我们需要哈希值基于个体的染色体生成，而不是像默认那样基于对象本身。这一点很重要，因为具有相同染色体的两个不同个体应该被适应度值哈希表识别为相同。

`/**`

`* 基于个体的染色体生成哈希码`

`*`

`* @return 哈希值`

`*/`

`@Override`

`public int hashCode() {`

`int hash = Arrays.hashCode(this.chromosome);`

`return hash;`

`}`

`/**`

`* 基于个体的染色体进行相等性判断`

`*`

`* @return 相等性布尔值`

`*/`

`@Override`

`public boolean equals(Object obj) {`

`if (obj == null) {`

`return false;`

`}`

`if (getClass() != obj.getClass()) {`

`return false;`

`}`

`Individual individual = (Individual) obj;`

`return Arrays.equals(this.chromosome, individual.chromosome);`

`}`

### 编码

另一个可能影响遗传算法性能的组件是所选择的编码方式。虽然理论上任何问题都可以使用 0 和 1 的二进制编码来表示，但这很少是最有效的编码选择。

当遗传算法难以收敛时，通常可能是因为为问题选择了糟糕的编码方式，导致其在搜索新解决方案时遇到困难。选择好的编码方式没有硬性科学，但使用过于复杂的编码通常会产生糟糕的结果。例如，如果你想要一种能够编码 0-10 之间 10 个数字的编码方式，通常最好使用 10 个整数的编码，而不是二进制字符串。这样更容易应用变异和交叉函数，这些函数可以应用于单个整数，而不是表示整数值的位。这也意味着你不需要处理无效的染色体，例如表示值 15 的“1111”，这超出了我们所需的 0-10 范围。

### 变异和交叉方法

在选择改进遗传算法性能的方案时，选择好的变异和交叉方法是另一个重要因素。要使用的最佳变异和交叉方法主要取决于所选的编码方式和问题本身的特性。一个好的变异或交叉方法应该能够产生有效的解决方案，同时也能够以预期的方式对个体进行变异和交叉。

例如：如果我们正在优化一个接受 0-10 之间任意值的函数，一种可能的变异方法是高斯变异，它向基因添加一个随机值，略微增加或减少其原始值。然而，另一种可能的变异方法是边界变异，其中选择一个介于下限和上限之间的随机值来替换该基因。这两种变异方法都能够产生有效的变异，但根据问题的性质和实现的其他细节，其中一种可能会优于另一种。一个糟糕的变异方法可能只是根据原始值将值向下舍入为 0 或向上舍入为 10。在这种情况下，发生的变异量取决于基因的值，这可能导致性能不佳。初始值 1 会被更改为 0，这是一个相对较小的变化。然而，值 5 会被更改为 10，这要大得多。这种偏差可能导致对接近 0 和 10 的值的偏好，这通常会对遗传算法的搜索过程产生负面影响。



## 总结

遗传算法可以通过不同的方式进行修改，以实现显著的性能提升。在本章中，我们探讨了多种不同的优化策略，以及如何将它们应用到遗传算法中。

自适应遗传算法是一种优化策略，能够比标准遗传算法带来性能提升。自适应遗传算法允许算法动态更新其参数，通常修改的是变异率或交叉率。这种参数的动态更新通常比静态定义的参数（即不根据算法状态进行调整的参数）能取得更好的结果。

本章讨论的另一种优化策略是多启发式算法。该策略涉及将遗传算法与另一种启发式算法（如模拟退火算法）相结合。通过将搜索特性与另一种启发式算法结合，可以在这些特性发挥作用的场景下实现性能提升。本章介绍的模拟退火算法基于冶金学中的退火过程。当将其应用于遗传算法时，它允许基因组在初始阶段发生较大变化，然后逐渐减少变化量，从而使算法能够聚焦于搜索空间中有潜力的区域。

实现性能提升的最简单方法之一是优化适应度函数。适应度函数通常是计算开销最大的部分，因此是优化的理想目标。同样重要的是，适应度函数需要定义良好，并能很好地反映个体的实际适应度。如果适应度函数不能很好地反映个体的表现，可能会拖慢搜索过程，并将算法引向搜索空间中较差的区域。

优化适应度函数的一个简单方法是支持并行处理。通过同时处理多个适应度函数，可以大幅减少遗传算法评估个体所花费的时间。

另一种可用于减少处理适应度函数所需时间的策略是适应度值哈希。适应度值哈希使用哈希表来存储最近使用过的若干染色体的适应度值。如果这些染色体在算法中再次出现，可以直接调用其适应度值，而无需重新计算。这可以避免对过去已经评估过的个体进行重复的繁琐处理。

最后，考虑改进遗传编码，或使用不同的变异或交叉方法来优化进化过程，也可能非常有效。例如，使用不能很好代表被编码个体的编码方式，或者使用无法在基因组中产生所需多样性的变异方法，都可能导致算法停滞，并产生较差的解。
