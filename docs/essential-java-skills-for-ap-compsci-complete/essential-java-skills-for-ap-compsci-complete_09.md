# 9. 简单的 Java 程序结构

在我们的第一个程序中，我们在计算机上显示了一条简单的消息。我们使用 `System.out.println()` 方法完成了这一点。这是 Java 中的一个命令，我们可以用它来执行某些操作。

嗯，实际上，`println()` 才是真正的方法。`System.out` 是包含我们运行的 `println()` 方法的类。并且 Java 中的每条语句都以分号结尾。每一条。它就像句子中的句号。

你看，Java 中几乎所有内容都保存在一个叫做类的东西中。类定义了程序各部分如何工作的规则。稍后，你将学习如何更多地使用类，甚至如何创建自己的类。目前，我们只需要使用我们已经拥有的类。

当我们运行之前的代码时

```
System.out.println("Hello, world!");
```

有几件事情正在发生。

首先，这行代码周围的一切都在 Java 中设置我们的项目。让我们简要地看一下。

```
public class Main {
public static void main(String[] args) {
System.out.println("Hello, World");
}
}

```

在这个示例代码中，我们的 `Hello, World` 语句位于两个特定的代码块内。代码块使用一对花括号 `{ }` 来定义。它们将内部的内容分组到一个块中。

代码块有一些规则，但我们稍后会介绍。现在，只需知道你可以将块相互嵌套。因此，在这个示例中，我们有一个 `public class Main` 代码块，从第 1 行开始，到第 6 行结束。然后我们有一个 `public static void main` 代码块，从第 3 行开始，到第 5 行结束。

我们的代码位于这个内部代码块中。

外部代码块定义了程序的整体包装器。Java 在构建和运行程序时会寻找要执行的 `main()` 方法。

所以，Java 寻找一个类。它找到了。

然后它寻找 `main()` 方法，它也找到了。

然后它运行该方法内的代码。这就是我们创建并添加到项目中的代码。

在我们学习面向对象编程之前，你所有的代码都将放在 Main.java 文件中。很快，我们也会在构造器之外创建代码。

那些 `public`、`class`、`static`、`void` 之类的词？暂时忽略它们，专注于我们放入 `main()` 方法中的内容。一切都会随着时间的推移而变得清晰。
