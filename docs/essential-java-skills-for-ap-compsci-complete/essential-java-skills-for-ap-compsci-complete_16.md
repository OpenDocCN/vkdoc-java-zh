# 16. Unicode

使用二进制，我们可以以计算机原生的格式来表示不同类型的值。对于文本，我们也可以这样做；只是我们需要在实现方式上更具创造性一些。



## 文本编码

也许你曾使用过解码环。借助解码环，你可以将包含字母的消息，通过它找到每个字母对应的数字或符号。你可以拿这样一句话为例，为每个独特的字母分配一个代表它的符号。利用这种方法，你可以将每个字母转换成新的代码。这被称为**编码**。你正在将原始文本转换为一套独特的代码。

在另一端，如果我收到一条使用这些符号的消息，我可以取出每个符号，并使用相同的解码环，将符号与字母匹配，从而将消息**解码**回其原始格式。

编码和解码过程必须在两端保持一致并达成共识。如果一端使用的解码环与另一端不同，我们就无法通信。这种达成共识的方法被称为**协议**，它在通信的两端都会被使用。

现在，我们可以用数字来代替符号来表示我们的协议。这些数字与我们想要通信的每个字符唯一对应。

但这仅仅是简单的字母表。我们可以将其表示为大写字母和小写字母。此外还有标点符号、常见的国际字符、希腊字母、特殊符号，以及用于在屏幕上绘制方框和线条的块状构建符号。

另外还有一些非打印字符，比如空格、换行、退格等。

现在你拥有了一套独特的字符集，每个字符都有一个对应的数字。但请记住，计算机需要以二进制格式存储数字，因此每个数字都会被转换成二进制。在这种情况下，我们有 256 个字符，它们存储在一个字节（即 8 位）中。

## ASCII + Unicode

这种单字节编码系统被称为 **ASCII**（美国信息交换标准代码），最早开发于 20 世纪 60 年代初。随着时间的推移，该系统不断扩展以包含更多字符。但在某个节点，世界上需要表示的符号或字形实在太多，ASCII 已经无法满足需求了。

一种名为 **Unicode** 的新标准使用多个字节来表示来自世界各地语言的成千上万个符号，甚至包括失落文明的古老符号和语言（甚至还有一些来自科幻小说！）。更常见的字符需要更少的位来存储，这可以在存储或传输时节省空间和时间。较不常见或更复杂的字符则需要更多位。

网络标准化采用了 **UTF-8** 格式，即 8 位 Unicode 转换格式。它向后兼容 ASCII。每个字符使用前缀 `U+` 后跟一个四位十六进制数来表示。因此，字母“D”表示为 `U+0044`。四位十六进制数需要 2 个字节（即 16 位）来存储。因此，仅使用 2 个字节，就可以编码和解码超过 65,000 个符号。这被称为 Unicode 的**平面 0**。对于更多字符，会添加额外的位来创建新的平面。目前，主要使用的是三个平面：平面 0、平面 1 和平面 2。还有其他未使用或可自定义的平面，最高可达平面 16。总的来说，整个 Unicode 范围可以存储 1,114,112 个符号，这些符号被称为“码点”。这可是非常多的字母！

## 表情符号

你每天都在使用 Unicode。你在手机和设备上使用的所有表情符号都是 Unicode 中的符号。

这就是为什么通常情况下，当表情符号被添加到一个平台时，它们也会被添加到其他平台。这就是标准的工作方式。为了使一项技术符合标准，它需要保持对其支持的更新。这包括标准随时间变化和适应的情况。
