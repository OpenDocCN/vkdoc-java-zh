# 2. Java、C# 和 Python 入门

在本章中，我们将了解 2021 年（及以后）最流行的三种编程语言。我们将详细介绍并讨论几个与编程相关的概念。这些术语起初可能看起来令人生畏和/或笨拙，但请注意，一旦掌握，它们实际上相当简单，并提供了一条直达计算机核心的特殊路径。我们将首先向你介绍这三种令人惊叹的语言：Java、C# 和 Python。

## Java

由*太阳微系统公司* 创建并于 1995 年发布，Java 迅速成为一种被广泛采用的通用编程语言，尤其适用于在线环境（即云计算）。如今，用 Java 编写的软件为无数智能手机、数据中心以及像谷歌这样无处不在的在线服务提供动力。截至 2021 年，它对于新手和经验丰富的程序员来说都是最流行的编程语言之一。

尽管被称为 *JavaScript* 的编程语言与 Java 共享前四个字母，但这两者几乎没有共同之处。JavaScript 是由互联网先驱*网景公司* 在九十年代创建的，用于当时新兴的浏览器技术。这种语言在互联网上仍然活跃且健康，为众多网站提供了额外的功能和可用性。

## C#

C#（发音为 C sharp）于 2002 年夏季向公众发布。该语言由微软开发，可用于创建从生产力软件到视频游戏等任何类型的现代应用程序。C# 的某些特性与包括 Java 在内的其他几种流行编程语言共享。

你可能听说过 C++，与 C# 相比，它是一种更早且稍微复杂的语言。尽管 C++ 通常能够产生更高效的输出，但使用 C# 开发移动和 Web 应用更为直接。至于 C++ 和 C# 的“祖父”呢？那就是 C（有时也称为 *过程式 C*），这是一种可追溯到 1972 年的语言。

## Python

Python 的名字来源于*蒙提·派森*（一档流行的轻松娱乐电视节目），于 1991 年发布。虽然 Python 的输出通常比用 C# 制作的软件慢，但近年来 Python 仍然变得相当流行。它几乎可以处理任何事情，从为 Web 制作简单的应用程序到更重的桌面软件。Python 甚至在视频游戏行业中找到了用于游戏原型设计的细分领域。

Python 值得注意的一点是它对空白/缩进（通过按空格键和/或 Tab 键创建的不可见字符）的敏感性。我们将在本书后面讨论这种独特的方法以及更多内容。

## 1 和 0，还有你

现在，你可能听说过关于二进制数字以及计算机如何喜爱它们的说法。是的，在最基础的层面上，微处理器确实在处理 1 和 0（它们真的很喜欢像 01011010110 这样的字符串）。这个输入到中央处理单元的最接近的层级被称为*机器语言* 或*机器码* 层级。在这个二进制层级之上，有多个抽象层来帮助我们人类使用计算机完成工作。

现在，在编程中，存在*高级语言* 和*低级语言* 的区别。这与工具的质量无关。这种分类指的是编程语言在多大程度上接近机器码（即二进制）。基本上，高级语言更接近人类语言，而较低级的语言对外行人来说往往显得相当晦涩。C# 和 Python 被认为是高级语言，而 C++ 则代表了所谓的中级语言，因为它为更高级的程序员提供了一些相当强大的特性。

等等，你现在是不是开始焦虑了？别担心：在本书中，我们将只关注高级内容。不需要深入了解二进制/机器码。



## 编译与解释代码

计算机无法直接执行任何编程语言。例如，Java 和 Python 都属于所谓的解释型语言。这意味着代码清单中的每一行都会在运行时被实时、逐步地“解释”成机器码。与另一种语言类型（编译型语言）相比，解释过程通常会导致程序执行速度较慢。

*编译型语言*的代码清单在运行之前需要经历一个（你猜对了）称为*编译*的过程。这指的是在程序执行之前，将其完整源代码翻译成机器语言。幸运的是，编译过程是自动化的，所以你不需要任何程序员到机器码的翻译手册。这是任何优秀编码开发环境的功能。关于编译型与解释型编程语言的主要区别，请参见表 2-1。

## 变量的魔力

变量是一种临时存储形式，供程序用户在操作软件期间使用。变量通常使用设备中的*随机存取存储器（RAM）*。这意味着如果你关闭设备电源，存储在变量中的数据就会消失，除非它先被保存到硬盘等存储设备中。

举一个基本的使用场景，变量可用于存储游戏中玩家的名字。从程序员的角度来看，变量在整个程序中被反复使用。可以对它们应用多种操作，从基本算术到复杂的数学公式。

每种编程语言中都有多种类型的变量。事实上，有用于文本字符串、单个字母数字字符以及不同数值范围的独立变量类型。但为什么要做这些区分呢？嗯，有时你只需要存储一个字符，比如字母“B”。在上述场景中，如果为原本用于存储 19 位数值的变量类型预留空间，那将是对计算机内存的浪费，因此大多数编程语言中才有了多种变量类型。

许多编程语言并不具备可以存储任何类型信息的通用变量。在大多数情况下，程序员需要在使用前定义*变量声明*。

计算机不擅长猜测。大多数编程语言会明确区分字符串 *happy_variable* 和 *Happy_Variable*。如果代码清单无法运行，很可能只是大小写方面的问题。

## 关于花括号、变量作用域和代码块

在本章中，你很快就会遇到几个包含花括号的代码清单。它们确实是许多编程语言中的一个重要元素。这些花括号的作用是表示*代码块*，代码块由分组的声明和/或语句组成。编译器或解释器软件将单个代码块视为单个指令。

此外，代码块可以限制变量的*作用域*，即代码清单中特定变量生效的部分。两个不同的代码块可能完全无法访问彼此的变量空间。

Python 处理花括号的方式与 Java 或 C# 不同。相反，该语言在表示代码块时使用空白字符（即空格或缩进）。在 Python 中，花括号保留用于定义字典或集合数据类型。

清单 2-5 的第一行（即*using System;*）激活了一个特定的*命名空间*。C#中的命名空间是容器元素，有助于组织代码。首先，它们能帮你节省时间。如果没有 System 命名空间，每次在屏幕上打印内容时，我们就要输入*System.Console.WriteLine*，而不是*Console.WriteLine*。你也可以声明自己的自定义命名空间，我们将在本书后面部分进行实践。目前，你只需了解它们的存在即可。

大多数编程语言都需要在源代码中包含特定的声明语句，而这些语句通常并非每个项目所独有。这被称为*样板代码*。例如，在清单 2-4 中，*public static void main(String args[ ])*这一行，以及在清单 2-5 中，*public static void Main( )*这一行，可以分别归类为 Java 和 C#方面的样板代码。我们将在后续章节中更深入地探讨这个概念。

现在，与 Java 相比，C#的许多函数使用了不同的词汇。要在屏幕上打印文本，我们有*Console.Writeline*。对于用户输入，我们有*Console.ReadLine*，如下行所示：

```
int YourNumber = Convert.ToInt16(Console.ReadLine());
```

这里我们初始化了一个整数变量 YourNumber，并将其传递给一个转换函数*Convert*。我们告诉它等待用户输入，并期望得到一个*有符号 16 位整数值*。这些是范围从-32,768 到 32,768 的整数。对于用户最可能输入的内容来说，这个范围已经足够宽裕了。

*无符号 16 位整数*的取值范围在 0 到 65,536 之间，这意味着它们不能存储负值。如果我们需要存储非常大的数字，可以选择使用*32 位整数*，它们也有有符号（-2,147,483,647 到 2,147,483,647）和无符号（0 到 4,294,967,295）两种版本。就本书而言，我们将严格使用较小的数字。

## 关于类与面向对象编程

你可能已经注意到*类*这个词在我们的清单中出现了多次。这是*面向对象编程（OOP）*中的一个关键概念，OOP 是一种流行的编程范式。本书中介绍的三种语言都在不同程度上融入了 OOP。

任何现实世界或抽象的场景都可以用 OOP 优雅地表达。这种方法中的两个基本构建块被称为*类*和*对象*。简单来说，类定义了对象的蓝图。例如，你可能正在开发一个关于园艺的软件，并编写了一个名为*Plant*的类。然后，你可以使用 Plant 类创建单个实例（称为对象）来填充你的虚拟花园。你以这种方式创建的各种植物个体将共享其源类中定义的特性与变量。日后修改 Plant 类的部分内容将影响所有属于该类的未来对象。你还可以创建 Plant 的子类，以满足你计划建模的各种玫瑰和郁金香（以及一个用于花园小矮人的类）的需求。

OOP 为软件开发人员提供了许多好处。其中包括代码的可重用性以及通过类封装实现的安全性。面向对象编程有许多迷人的方面。我们将在本书后面更深入地探讨这一范式。



## 流程控制基础：循环

除了变量之外，我们还有一系列逻辑结构可供使用，以便通过编码完成任务。这些结构构成了所谓的*流程控制*。当任何程序清单被输入后，计算机会从上到下读取它。很多时候，程序中的处理过程需要重复执行多次。因此，具备循环和条件程序流的能力就显得很有意义。

编程中的循环可能会让你接触到*迭代*的概念。迭代是为了获得理想结果，通常按预设次数重复特定步骤的过程。在编程语境中，重复的动作序列被称为*循环*。创建这些循环的方法有很多，这也取决于所使用的编程语言。有关这些结构的示例实现，请参见表 2-6。

## While 循环

表 2-6 中的第一种迭代方法是 Java 中演示的 while 循环。这种方法会持续执行操作，直到 while 函数中的条件得到满足。在我们的示例中，循环在变量 *i* 大于零时运行。除了在屏幕上打印文本的命令语法存在差异外，表 2-6 中的 while 循环在 Java 和 C# 中是相同的。

## For 循环详解

表 2-6 中 C# 的示例可能看起来有些复杂。这里讨论的结构，即 *for 循环*，是一种古老的技术，由几个元素组成，现在将对其进行讨论。

头部（即从 *for* 开始的部分）包含了关于循环体（即花括号包围的部分）需要执行多少次的指令。在我们的示例中，头部部分如下所示：

*   定义一个辅助数值变量 *i* 并将 *i* 的值设为零（int i=0;）。

*   只要变量 *i* 小于三（i<3;），就执行循环体部分，之后继续执行程序。

*   将 *i* 的值加一（i++）。

同样，除了命令语法（即 System.out.println 与 Console.WriteLine）的差异外，表 2-6 中的中间示例在 Java 和 C# 中是相同的。

## Python 中的循环

你可能会注意到我们的 Python 循环中明显缺少分号和花括号。包含 print 命令的行仅通过缩进被包含在循环的代码块中。此外，与 Java 和 C# 中相对复杂的结构不同，Python 中的 for 循环可以简单地使用一个名为 *range* 的巧妙函数来设置迭代次数。

有时我们需要记录对代码的更改。虽然纸笔也可以，但更好的做法是在代码清单内部进行记录。为了添加计算机不会解析的行，我们可以在代码清单中插入特殊字符。对于 Java 和 C#，我们使用两个正斜杠，而对于 Python，我们使用井号，如清单 2-5 所示。
