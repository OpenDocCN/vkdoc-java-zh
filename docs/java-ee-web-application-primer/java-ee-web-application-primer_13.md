# 13. Web 的无状态特性

Web 应用程序不维护状态。它没有记忆。对 Web 服务器的每个请求都是一个独立的事件。每个请求都不知道之前的请求。当你发送用户名和密码时，Web 服务器将其视为一个独立的事件。它不会跟踪你正在登录什么。信息只是被发送到服务器。

当你提交表单时，关于如何处理表单数据的所有信息都必须与表单一起发送。为什么？因为每个请求都是一个独立的事务。

在现实生活中，这就像你去银行，每问一个问题就换一个柜员。而且柜员之间互不交谈——只和你交谈。每个柜员都想看你的身份证、检查你的余额，并做其他柜员已经做过的所有事情。为了让这种情况更容易处理，你可以保留每个事务的运行日志，每个柜员都可以用它来验证已经完成的操作。

那么，Web 应用程序如何维护状态呢？答案是使用会话变量或传递来自先前事务的称为参数的信息。参数通过 URL 或作为请求的一部分发送到服务器的其他信息，在客户端（Web 浏览器）和服务器之间传递。这被称为请求包；我们在讨论 Servlet 时已经提到过这一点。

会话变量存在于 Web 服务器的内存中。每个请求都包含一个会话 ID。会话 ID 将该请求与该用户的会话数据关联起来。会话 ID 在请求之间自动传递。你不需要做任何事情。它始终存在。

由于每个用户有一个会话，你可以在每个用户的会话中存储变量。这是内存中的一个空间，在用户使用网站时保存数据。由于 Java 始终知道会话 ID，因此它可以访问会话中的任何数据。

所以，是请求包和会话将整个房间联系在一起。而你以为是地毯！（不好笑？再看一遍《谋杀绿脚趾》）。会话使得服务器能够轻松地将一个请求与另一个请求连接起来。



## 数据传递过程

以下列表总结了数据从 Web 表单通过 Servlet 发送到 JSP 所遵循的步骤：

1.  表单传递请求。
2.  Servlet 接收请求。
3.  Servlet 使用 `request.getParameter()` 处理请求。
4.  Servlet 根据请求中的数据生成响应。
5.  Servlet 在将发送到 JSP 的对象中构建响应。
6.  JSP 包含一个属性 `${user}`。
7.  Servlet 设置属性 `request.setAttribute("user",myUser);`。
8.  Servlet 通过调用 `getServletContext().getRequestDispatcher(url).forward(request, response);` 将 JSP 发送回原始浏览器。
