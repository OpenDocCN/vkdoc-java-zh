# 1. 面向对象编程概念

欢迎来到面向对象编程（OOP）的世界。在开始之前，我们先讨论一些基本问题与解答。例如，为什么需要这种编程方式？这些概念如何让你的工作更轻松？如果你知道这些问题的答案，学习之路将会变得顺畅，你也能从多个角度理解这些概念。但在开始之前，有两点警告需要告知你。

*   如果第一次阅读后未能完全理解所有内容，请不要失去动力。很多情况下，内容看似复杂，但你会逐渐掌握。

*   许多开发者批评 OOP 的概念。但请记住，人类思维往往倾向于批评新事物。因此，即使你想批评这些概念，我建议你先正确理解它们，在各种应用中使用这些概念，然后再自行决定是赞赏还是批评。

现在，让我们开始这段旅程……

计算机编程始于二进制代码，当时使用机械开关加载程序。不难想象，那时的程序员生活极具挑战性。为了让程序员的工作更轻松，一些高级编程语言被开发出来，这些语言使用类似英语的简单指令。但你不应忘记一个事实：计算机只能理解二进制语言的指令。因此，编译器的主要任务是将这些类似英语的指令翻译成二进制代码，最终这些高级语言逐渐流行起来。

随着时间的推移，计算机的容量和能力大幅提升。随后，开发者开始开发复杂的应用程序。不幸的是，当时可用的编程语言没有一种足够成熟，能够有效处理所有应用程序。一些主要关注点如下：

*   如何避免重复劳动？或者说，如何重用现有代码？

*   如何在共享环境中控制全局变量的使用？

*   当代码中存在过多跳转时（例如，在应用程序的多个地方使用 `goto` 关键字），如何进行调试？

*   如何让新工程师的工作更轻松？

*   如何更好地维护大型代码库？

为了解决这些问题，专家级程序员开始将大问题分解为小问题。这一理念背后的思想非常简单：*如果你能解决这些小问题，最终你就能解决大问题*。因此，他们开始将大问题划分成小块，函数（或过程、子程序）的概念由此发展而来。每个函数专门解决一个小问题。因此，管理这些函数及其之间的交互成为关键焦点，**结构化编程**的概念应运而生。结构化编程大获成功，因为管理小函数很容易，调试起来也很方便。同时，开发者也开始限制全局变量的使用，在大多数情况下，它们被函数中的局部变量所取代。

结构化编程流行了近二十年。在此期间，硬件容量显著提升。因此，开发者希望解决更复杂的任务，结构化编程的局限性也逐渐凸显；例如，考虑以下情况：

*   假设在你的应用程序中，你在多个函数中使用了某种特定数据类型。后来你发现需要更改该数据类型。结果，你需要在应用程序的所有函数中实施更改。

*   用结构化编程的关键组件（即数据和函数）来建模所有现实场景是很困难的。在现实世界中，每当你创建一个产品时，有两个方面需要关注：
    *   *目的*。为什么需要这个产品？

*   *行为*。这个产品如何让你的生活更轻松？

随后，对象的概念应运而生。艾伦·凯伊被广泛认为是面向对象编程之父之一，他与一些同事在帕洛阿尔托研究中心（PARC，前身为施乐帕克研究中心）命名了这一概念。

### 要点回顾

结构化编程与面向对象编程的根本区别可以总结如下：*在面向对象编程中，重点不是对数据的操作，而是数据本身。*

OOP 侧重于一些关键原则。我将在本书中详细讲解它们。在本章中，你将初步了解每个原则。第一次阅读时你可能无法理解所有术语，因此建议你再次回顾这些主题。

## 类与对象

这些是 OOP 的核心。**类**是其对象的蓝图或模板。**对象**是类的实例。每个对象都有自己的状态、行为和标识。简单来说，你可以认为在结构化编程中，你将问题分解为小函数；而在 OOP 中，你将问题分解为对象。你熟悉诸如 `int`、`double`、`float` 等数据类型。你知道这些是*内置数据类型*或*原始数据类型*，因为它们已在计算机语言中定义。但当你创建自己的数据类型时，比如 `Student`，你需要创建一个 `Student` 类。就像当你需要创建一个整型变量时，你首先提到 `int`；同样，当你需要创建一个 `Student` 对象（例如 `john`）时，你需要首先提到你的 `Student` 类。因此，当你熟悉 OOP 后，你可能会这样说：狗是 `Mammal` 类的一个对象，你的汽车是 `Vehicle` 类的一个对象，等等。



## 封装

在面向对象编程中，你不允许数据在系统内自由流动。相反，你将数据和函数封装到一个单一单元中（即一个类中）。封装的目的至少包含以下之一：

*   设置限制，使得对象的组件无法被直接访问
*   将数据与作用于该数据的方法绑定在一起（即形成一个胶囊）

在某些面向对象编程语言中，信息隐藏并非默认实现。因此，它们引入了一个额外的术语，称为*信息隐藏*。

稍后你会看到，数据封装是类中的关键特性之一。如果你想提升安全性，你的数据就不应该对外部世界可见。只有通过类内部定义的方法，你才能访问这些数据。因此，你可以将这些方法视为对象数据与外部世界（即你的程序）之间的接口。

在 Java 中，你可以通过多种方式实现封装。例如，你可以在此上下文中使用访问说明符（或修饰符）以及 getter-setter 方法。

### 注意

你将在第 3 章中学习访问说明符和 getter-setter 方法。

## 抽象

抽象的关键目的是只展示必要的特征，并隐藏实现的背景细节。抽象也与封装密切相关，但两者的区别可以通过一个简单的日常场景轻松理解。

当你按下遥控器上的按钮打开电视时，你并不关心电视的内部电路，也不关心遥控器如何控制电视的操作。你只知道遥控器上不同的按钮有不同的功能，只要它们正常工作，你就满意了。因此，用户与复杂的实现细节隔离开来，这些细节被*封装*在遥控器（和电视）内部。同时，可以通过遥控器执行的常见操作可以被视为一种*抽象*。当同一个遥控器也能在不同型号或产品上使用时，制造商可以增强这一特性。例如，DVD 播放器的遥控器也可以用来控制电视的音量。

## 继承

每当我们谈论可重用性时，通常指的是继承，这是一个对象获取另一个对象属性的过程。考虑这个例子。`Bus` 是 `Vehicle` 的一种类型，因为它满足了用于运输目的的 `Vehicle` 的基本标准。类似地，`Train` 是 `Vehicle` 的另一种类型。即使 `GoodsTrain` 和 `PassengerTrain` 不同，我们也可以说两者都继承自 `Train` 类别（或类），因为最终它们都满足了 `Train` 的基本标准，而 `Train` 本身又是一种 `Vehicle`。因此，你可以简单地说，层次化分类是通过继承的概念来支持的。

在编程世界中，继承从一个现有的父类创建一个新的子类。这个父类有时有不同的名称。例如，在 C# 中，你将这个父类称为基类，而在 Java 中，你可能将其称为超类。所以，简单来说，父类位于该层次链中高一级的位置。然后，你可以向子类中添加新的功能（方法）或修改超类的功能（稍后你会称之为重写功能）。你必须记住，由于这些修改，核心架构不应受到影响。换句话说，如果你从 `Vehicle` 类派生出 `Bus` 类，并在 `Bus` 类中添加/修改功能，这些修改不会影响为 `Vehicle` 类描述的原始功能。

因此，关键优势在于，通过这种机制，你可以避免大量重复的代码。

## 多态

多态通常与*一个名称，多种形态*相关联。想想你家宠物的行为。当它看到一个陌生人时，它会生气并开始狂吠。但当它看到你时，它会发出不同的声音并表现出不同的行为。在编程世界中，你也可以想到一个常见的方法：加法。对于两个整数的加法，你期望得到整数的和。但对于字符串操作数，你期望得到一个拼接后的字符串。

多态可以分为两种类型：

*   ***编译时多态**：* 一旦程序被编译，编译器可以很早就决定在哪种情况下调用哪个方法。这也被称为静态绑定或早期绑定。

*   ***运行时多态**：* 实际的方法调用在运行时才被解析。在编译时，你无法预测程序运行时将调用哪个方法（例如，程序可能因不同的输入而表现不同）。考虑以下情况：假设你想在程序执行的第一行生成一个随机数。如果生成的数字是偶数，你将调用一个方法 `Method1()`，它会打印“Hello”；否则，你将调用一个同名的方法，但它会打印“Hi”。现在，你会同意，在程序执行后，你才能看到哪个方法被调用了（即编译器无法在编译时解析该调用）。在这种情况下，在程序执行之前，你完全无法得知会看到“Hello”还是“Hi”。因此，它有时也被称为动态绑定或晚期绑定。在第 5 章中，你将看到关于多态的详细讨论，并深入体会运行时多态的必要性。

## 问答环节

**1.1 面向对象编程的关键特性是什么？**

以下是面向对象编程的关键特性：

*   封装
*   抽象
*   继承
*   多态

**1.2 对象与类有何不同？**

对象由类创建。对象是类的一个实例，而类只是对象的模板或蓝图。对象是物理实体，可以在系统中分配内存，但类是逻辑实体，不在系统中分配内存。

**1.3 抽象与封装有何不同？**

抽象关注对象的可见行为，而封装关注该行为的实现部分。封装帮助你捆绑数据，同时可以隐藏一些你不想向外部世界披露的信息。

**1.4 继承机制的关键优势是什么？**

重用现有代码，你可以节省时间和精力。同时，这种机制有助于避免应用程序中的重复代码。

**1.5 面向对象编程的特点是什么？**

以下是一些重要特点：

*   你的关注点在数据，而不是函数。因此，你将程序划分为对象，而不是函数。
*   你不允许数据自由流动。你使用方法访问它们。
*   你的对象通过方法进行通信。
*   外部世界不应访问你的数据。
*   你的应用程序可以轻松适应新的变化。同时，它也易于维护。
