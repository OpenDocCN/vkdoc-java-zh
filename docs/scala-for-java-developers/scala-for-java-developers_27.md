# 23. 转换你的代码库

此时，你很可能已经有一个现有的 Java 代码库，并且可能在想是否应该尝试将 Java 转换为 Scala。

这可能需要很多努力，但我认为，如果你认真考虑采用 Scala，那么投资将整个代码库转换为 Scala 绝对是值得的。不要试图留下一半。要接受这需要一些时间和精力，但要致力于实现它。如果你没有完成这项工作，冷漠情绪可能会滋生，更不用说旧的 Java 和 Scala 之间的互操作会变得非常尴尬。如果你坚持完成这项工作，你的生活会轻松很多。

IntelliJ IDEA 有一个转换功能，这是一个很好的起点。它可以让你快速地将一个 Java 类转换为 Scala，但要做好准备：它通常不符合惯用写法，甚至可能不经强制转换就无法编译。将任何自动转换视为一次学习练习。不要只看表面就接受转换结果，而要审视结果并运用你的判断力进行整理。例如，IntelliJ 通常会在 Scala 类中创建字段，而不是创建一个 case class。你通常可以用更符合 Scala 风格的方式重新表述逐字转换的结果。

考虑你的构建工具。这些工具需要更新。SBT 和 Maven 都支持 Java/Scala 混合项目，所以没有理由不继续运行你的持续集成和测试环境。你可能还想考虑是否值得将你现有的构建迁移到 SBT。
