关键语法差异

本书的这一部分是关于 Java 和 Scala 语言语法之间的关键差异。给定一些典型的 Java 代码，我们将查看等效的 Scala 语法。在第三部分中，我们将更多地研究 Scala 中 Java 没有直接对应项的特性。

我们将研究以下内容：

*   许多与类和对象、创建类、字段和方法相关的内容。我们将进行从 Scala 生成的字节码回到 Java 的往返转换，以便让你了解 Scala 与 Java 的关系。
*   继承、接口、抽象类和混入。
*   常见的控制结构，如 `for` 循环。
*   泛型。



## 灵活性

Scala 非常灵活。通常有多种方式可以实现同一目标。我指的不是使用 `while` 循环或 `for` 循环的区别；而是指该语言提供了不同的语法选项来表达相同的事物。这种灵活性带来了极大的自由度，但在阅读不同作者的代码时可能会令人困惑。

一个例子是我们之前看到的中缀表示法。调用方法时，你通常可以省略点和括号。Scala 没有强制规定；是否使用点号完全取决于你。

相比之下，Java 则非常严格；通常只有很少的方式来表达相同的事物。一眼识别出代码含义往往更容易。而在 Scala 中，你可能需要多花些功夫才能识别出某些更特殊的语法选项。

这一点同样适用于代码结构：你可以在函数内部创建函数，在类中间插入 import 语句，或者让一个类存在于一个名称不相关的文件中。当你习惯了 Java 的严谨性后，这一切可能会让你感到有些迷失方向。

## 不可变性与声明式编程

由于 Scala 推崇不可变性，你可能会注意到解决问题的方式有所不同。例如，你可能会发现很少使用循环来操作可变变量。Scala 程序通常更倾向于使用函数式编程风格来实现相同目标。

这种更具声明性的做事方式说的是“告诉我做什么，而不是怎么做”。你可能更习惯于 Java/命令式的做事方式，它说的是“准确地告诉我怎么做”。最终，当你放弃命令式编程那种微观管理风格时，你就赋予了语言在处理事务时更多的自由。

例如，Java 中一个传统的命令式 for 循环如下所示：

```
// java
for (int count = 0; count < 100; count++) {
System.out.println(count);
}
```

这是一个典型的命令式循环。我们明确地告诉它要从零到一百按顺序枚举。相反，如果我们使用更具声明性的机制，如下所示：

```
// scala
(0 to 100).foreach(println(_))
```

……枚举是在 `foreach` 方法内部完成的，而不是通过语言结构。我们说的是，“对于一个数字范围，对每个数字执行某个函数。”虽然只有细微差别，但我们并没有说明如何枚举这个序列。这意味着 Scala 可以自由地以它喜欢的任何方式实现枚举。例如，它可以选择并行执行。

有趣的是，Oracle 已经采纳了这些想法，自 Java 8 起提供了类似的 for 循环枚举。上述代码在 Java 中可以这样写：

```
// java
IntStream.range(0, 100).forEach(System.out::println)
```
