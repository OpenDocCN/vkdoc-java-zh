# 22. 技巧

回顾我的经验，我有三个主要技巧。

1.  明确你想从 Scala 中得到什么。  
2.  寻求指导并分享经验。  
3.  深思熟虑并制定计划。

## 明确目标

*   明确你想从中获得什么。你应该问自己，它对你有什么好处。如果你打算使用 Scala，你应该能够解释原因。你认为更简洁的代码会帮助你吗？是因为函数式编程的好处吗？不可变性？你打算如何衡量它？整个团队是否要追求像 Haskell 那样的纯函数式编程风格？还是面向对象风格？提前决定好。达成共识。
*   了解团队动态。了解你的团队以及他们的需求和愿望。评估团队的意愿和目标。最终，每个人都需要认同这个想法并朝着同一个方向前进。如果有一两个人感到被冷落或朝着另一个方向努力，你以后会遇到问题。
*   设定期望。与团队和管理层沟通。解释风险，并分享你的动机和期望。设定你的期望：全面学习函数式编程（想想 Haskell）是困难的。做好准备。
*   回顾。一旦开始，要持续回顾你所处的阶段。你是否从中得到了你预期的收获？你能证明或反驳你最初持有的任何假设吗？业务是否受益？如果你能更量化并使用度量标准，你就可以更客观。并且不要害怕改变你的想法。也许你会发现这对团队来说并非正确的选择。你随时可以退出。

## 寻求指导

*   聘请专家。这是我最重要的一个技巧。如果团队中有一位真正的专家，他们可以指导你掌握语法和特性，更重要的是，指导你完成采纳过程。当你还不应该加速时，他们可以拉住你；当你准备好时，他们可以为你打开新技术的大门。找到真正的专家可能很难，但他们可能价值连城。
*   积极参与社区。参加活动、会议和聚会。创建一个社区。向他人学习并分享你自己的经验。
*   查看开源项目。但不要看得太多。看看别人是怎么做的很好——我确实通过开源项目接触到了一些更奇特的语法——但请记住，适合他们的不一定适合你。要带着批判性思维去看待。利用别人的代码来激发学习，但不要照搬。

## 制定计划

*   制定计划。不要只是用 Scala 开始你的下一个项目；要深思熟虑并制定计划。
*   思考你的目标。把它们写下来，然后勾勒出能让你更接近目标的步骤。例如，如果目标是快速上手，你可能想和你的团队举办一些午餐时间的知识分享会。
*   决定你想在函数式编程连续谱系中处于什么位置。你是想直接跳入深水区，还是稍后再采用函数式习惯用法？一个最小化学习曲线的技巧是避免一开始就使用原生的 Scala 库。它们常常会引导你走向函数式风格，而且说到底，这又是一个需要学习的 API。你完全可以坚持使用你熟悉的 Java 库，至少一开始可以这样。
*   做出承诺。致力于你的目标；不要像我一样把 Scala 边缘化到测试中。主动决定使用 Scala，并做出个人承诺去实现它。

如果你遵循这些技巧，你应该能清楚自己的目标和期望，知道从哪里获得指导，并规划好下一步。你将面临使用 Scala 代码库的前景，很可能需要转换 Java 代码，并使用新的工具和库。

在接下来的章节中，我们将探讨一些关于如何进行的实用技巧，包括转换 Java 代码和管理你的代码库。
