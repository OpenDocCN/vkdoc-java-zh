# 7. 总结

在这次高层概览中，我们讨论了 Scala 如何既是一门面向对象语言，又是一门函数式语言。我提到 Scala 实际上只有对象；没有基本类型，一切都是对象。

我们讨论了 Scala 的背景，它如何从一门学术语言发展成为一门有商业支持的主流语言，以及它如何在 JVM 上运行。从理论上讲，这通过使其易于部署到现有服务器上，降低了采用的门槛。

在 JVM 上运行也意味着 Scala 有大量的库可用，因为 Java 互操作性是内建的。尽管一些工具相对落后，编译时间仍然较慢，但 Scala 已被广泛采用，我们看到许多大公司今天都在使用它。

我们了解了如何安装和运行 Scala。你看到了 Scala 程序通常运行的三种方式，并在此过程中介绍了 Scala REPL。

然后我们转向了语法概览，并查看了一些有趣的语言特性。

在语法概览中，我们看到了如何定义变量和值、函数和方法，并看到了 Scala 如何通过类型推断和自动识别终止条件（如 `return` 和分号）来减少样板代码的噪音。

我们看到了中缀表示法如何让你避免使用经典的点表示法，并且重要的是，我们看到了方法名可以包含符号。这就是我们能够在代码中自然地使用数学符号的原因；它们实际上是方法，所以我们可以重写它们，并使用中缀表示法，而无需使用嘈杂的点和括号。

我们还使用了一些集合类型，并看到了枚举它们的两种基本方式：`foreach` 和 `for` 循环语法。我们看到了处理 Java 对象是多么容易；像 `Int` 和 `Boolean` 这样的值类型在 Scala 和 Java 中基本相同。我们深入了解了 Scala 如何在其类层次结构中表示类型，并学习了如何在 ScalaDoc 中查找信息。

在语言特性方面，我们研究了在处理源文件和包、方法及方法参数时语言的一些有趣方面，以及一些面向函数式编程的特性。
