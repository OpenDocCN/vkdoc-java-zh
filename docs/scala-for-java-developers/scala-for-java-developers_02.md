# 1. Scala 语言

Scala 既是一门函数式编程语言，也是一门面向对象编程语言。作为 Java 程序员，你会对面向对象的定义感到熟悉：Scala 拥有类、对象、继承、组合、多态——所有你在 Java 中习惯使用的特性。

事实上，Scala 在某些方面比 Java 更进一步。没有“非”对象。一切都是对象，因此没有像 `int` 这样的原始类型，也没有静态方法或字段。函数是对象，甚至值也是对象。

Scala 可以被准确地描述为一种函数式编程语言，因为它允许并推广函数式编程中重要的技术使用。它提供了语言级别的特性，用于实现不可变性以及无副作用的编程函数。

像 Lisp 或 Haskell 这样的传统函数式编程语言只允许你使用这些技术进行编程。这些语言通常被称为纯函数式编程语言。从这个意义上说，Scala 并非纯函数式语言；它是一种混合体。例如，你仍然可以处理可变数据，利用语言处理不可变数据，或者两者兼用。这对于灵活性和易采用性来说很棒，但对于设计的一致性和统一性来说则不太理想。

## 作为函数式编程语言

通常，函数式编程语言支持：

1.  一等函数和高阶函数。  
2.  匿名函数（lambda 表达式）。  
3.  纯函数和不可变数据。  

可以说 Java 也支持这些特性，而且 Java 确实一直在努力提供更好的支持。然而，任何朝这个方向的进展都感觉像是事后才想到的，并且通常会导致语法冗长，或与现有习语和语言 API 产生冲突。

尽管 Java 有所进步，但人们不太可能将其描述为一种函数式编程语言。部分原因是它在这种风格下使用起来很笨拙，另一部分原因是它作为面向对象语言的悠久历史。

另一方面，Scala 从一开始就被设计为一种函数式编程语言。它拥有更好的语言结构和库支持，因此以函数式风格编码时感觉更自然。例如，它有定义不可变值的关键字，并且库集合类默认都是不可变的。

## 历史

Scala 始于 2001 年，是瑞士洛桑联邦理工学院（EPFL）的一个研究项目。在 2003 年内部发布后，于 2004 年公开发布¹。该项目由 Martin Odersky 领导，他之前曾在 Sun Microsystems 参与 Java 泛型和 Java 编译器的工作。

一门学术语言跨界进入工业界是相当罕见的，但 Odersky 和其他人创立了 Typesafe Inc.（后更名为 Lightbend Inc.），这是一家围绕 Scala 建立的商业企业。自那时起，Scala 已稳固地成为主流的开发语言。

Scala 提供了比 Java 更简洁的语法，但运行在 JVM 上。在 JVM 上运行（理论上）意味着可以轻松迁移到生产环境；如果你的生产环境中已经安装了 Oracle JVM，那么字节码是由 Java 编译器还是 Scala 编译器生成的并无区别。

这也意味着 Scala 内置了 Java 互操作性，进而意味着 Scala 可以使用任何 Java 库。Java 相对于其竞争对手的优势之一始终是拥有大量可用的开源库和工具。这些几乎全部对 Scala 也可用。Scala 社区也拥有同样的开源精神，因此涌现出越来越多优秀的 Scala 库。

## 未来

Scala 无疑已成为一种流行的主流语言。它已被许多大公司采用，包括 Twitter、eBay、Yahoo、汇丰银行、瑞银集团和摩根士丹利，并且短期内不太可能失宠。如果你对在生产环境中使用它感到不安，大可不必；它由一个国际组织支持，并且在流行度指数中经常名列前茅。

不过，工具支持仍然落后于 Java。像 IntelliJ IDEA 和 Eclipse 这样强大的 IDE 可以轻松重构 Java 代码，但对于 Scala 来说还不够完善。编译时间也是如此：Scala 的编译速度比 Java 慢得多。这些问题会随着时间推移而改善，总体而言，它们并不是我在开发中遇到的最大障碍。

Scala 的未来与 JVM 的未来息息相关，而 JVM 依然强劲。然而，各种其他函数式语言正在涌现；特别是 Kotlin 和 Clojure 很有趣，可能会形成竞争。如果你对基于 JVM 的语言不感兴趣，而只是看重函数式编程的优势，那么 Haskell 和 ELM 在工业界正得到越来越广泛的应用。
