# 6. 语言特性

在我们的探索之旅中，我们已经看到了一些示例语法，了解了类层次结构，并简要了解了 ScalaDoc，但 Scala 还提供了大量其他有趣的语言特性。

在本章中，我们不会真正讨论语法，而是会探讨一些使 Scala 在处理源代码、使用方法以及利用其函数式编程特性时，成为一门有趣且强大语言的因素。

## 处理源代码

源文件。在 Scala 中，源文件的内容比 Java 灵活得多。在 Java 中，文件名必须与其包含的公共类名称相同，但对于 `.scala` 文件则没有这样的限制。一个名为 `Customer.scala` 的文件可能包含一个名为 `Customer` 的类，但并非必须如此。同样，它可能包含四个类，但没有一个叫 `Customer`。

包。虽然本质上与 Java 中的包相同，但 Scala 中包内的类不必像 Java 那样存放在同名的文件夹中。在作用域方面存在一些差异——例如，Scala 中没有 `protected` 关键字，但你可以使用特殊语法（`variable[package]`）来实现相同的效果。

包对象。Scala 还有包对象的概念。这些对象可以放置有用的代码块，以便在包作用域内重用。它们对包中的其他类可用，如果有人导入了该包，那么包对象中的所有内容对他们也是可用的。库经常使用这些对象，以便你可以一次性导入它们的所有类。

导入别名。导入与 Java 大致相同，但一旦你在 Scala 中导入了一个类，你就可以在你的类中重命名它。换句话说，你可以为你的类中的某个类创建别名。这在遇到名称冲突时非常有用——例如，在库之间。

类型别名。Scala 也支持类型别名。你可以为复杂的类型定义指定别名，以帮助阐明意图。这类似于 C 语言中无结构的 `typedef` 或 `#define` 宏，或者 Haskell 中所谓的类型同义词。

特质。虽然 Scala 有类和对象，但没有“接口”关键字。相反，它有 `trait`（特质）的概念，它类似于接口，但可以包含方法。它介于 Java 8 的默认方法和 Ruby 的混入（mixin）之间。



## 使用方法

泛型。Scala 对泛型类型的协变和逆变提供了比 Java 更好的支持。这意味着当泛型类型作为参数使用时，你可以在方法签名中拥有更高的通用性和灵活性。暂时不必担心具体细节，后面会有专门一节来讲解变型。

变型简述

泛型类型的协变意味着，如果某个类型的泛型参数（尖括号中的部分，如 `<T>`）是某个类型的子类型，那么当该类型的参数是超类型时，该泛型类型本身也是另一个泛型类型的子类型。例如，假设 `Cat extends Animal`，那么 `List<Cat>` 也是 `List<Animal>` 的（子类型）。逆变也保持了这种关系，但方向相反。Java 不支持这种类型变型，但支持协变返回类型。在 Scala 和 Java 中，方法参数和变量也可以是协变或逆变的。

可变参数。在处理方法时，Scala 和 Java 一样支持可变参数或 `varargs`。

```
public void add(String... names)   // java
def add(names: String*)            // scala
```

命名方法参数。Java 不提供命名方法参数。在 Scala 中，只要你为参数命名，就可以不按顺序调用方法。因此，对于函数 `def swap(first: Int, second: Int)`，你可以显式地调用它，并为其参数命名。由于参数已命名，编译器可以区分它们，无论其位置如何。所以，以下写法都是正确的：

```
swap(first = 3, second = 1)
swap(second = 1, first = 3)
```

默认值。你可以在参数声明后使用 `=` 添加默认值。例如，`def swap(first: Int, second: Int = 1)`。如果在调用函数时省略第二个值，它将默认为 `1`。你仍然可以提供值来覆盖默认值，并且仍然可以使用命名参数。

```
swap(3)
swap(3, 2)
swap(first = 3)
swap(first = 3, second = 1)
```

函数字面量。函数字面量只是一个创建匿名函数的表达式。你可以将它们存储为变量、传递它们，或将它们传入方法。即时创建它们意味着你可以创建简洁的功能片段，而无需样板代码。函数字面量是引用 lambda 的一种方式。

Lambda 表达式。Scala 支持 lambda 表达式或匿名函数。你可以将函数字面量作为参数传递给方法，并在方法签名中使用函数签名作为参数。因此，下面的 `test` 方法接受一个无参数、返回 `Boolean` 的函数。

```
def test(f: () => Boolean) = ...
```

当你调用它时，可以传入一个函数字面量作为参数。

```
test(() => if (!tuesday) true else false)
```

再举一个例子，你可以创建一个函数签名来表示一个从 `String` 值到 `Boolean` 的函数，如下所示：

```
def test(f: String => Boolean): Boolean = ...
```

……然后像这样用一个函数字面量调用它：

```
test(value => if (value == "tuesday") true else false)
```

## 函数式编程

还有一些 Scala 特性更侧重于函数式编程。

模式匹配。这是一个非常强大的特性，乍一看类似于 switch 语句，但用途远不止于此。

for 推导式。这与常规的 for 循环有细微差别，并且在处理函数式结构时非常有用。当你第一次遇到它们时，它们看起来像是 Java 的 `for` 循环的另一种语法。

柯里化。虽然你可以在任何语言中编写自己的柯里化函数，但 Scala 将柯里化作为语言特性来支持。如果你不确定什么是柯里化，现在不必担心。更多细节请参见柯里化部分。

函数字面量。该语言支持字面量来表示一些有用的类型，例如元组。流行的 Java 函数式库，如 totally-lazy¹ 或 functional-java²，都有这类东西；而 Scala 只是让它们更容易使用。

递归。大多数语言都支持递归，但 Scala 的编译器支持尾调用优化，这意味着它可以支持在 Java 等语言中会导致栈溢出的递归调用。如果你使用 `@tailrec` 注解，编译器甚至可以为你执行一些检查。
