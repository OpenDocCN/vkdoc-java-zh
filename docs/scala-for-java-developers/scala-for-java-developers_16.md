从 Java 到 Scala

本书的这一部分主要探讨 Scala 与 Java 之间的差异。Scala 中有大量语言特性在 Java 中没有明显的对应物。在本部分中，我们将深入探讨其中一些特性，并探索 Scala 相较于 Java 能为我们带来什么。

具体来说，我们将探索许多使编写 Scala 代码更具表现力的语言特性，并研究一些 Scala 广为人知的函数式编程惯用法。

## 富有表现力的 Scala

Scala 提供了若干特性，使编写代码更加简洁。除了我们已经见过的一些特性外，它还提供了以下机制：

*   使用特殊方法 `apply` 让方法看起来像函数。
*   使用特殊方法 `update` 为赋值操作符提供默认行为。
*   让常规的方法调用看起来像语言结构，这实际上意味着你可以定义自己的控制结构。

Scala 还提供了模式匹配；我们将研究 `unapply` 方法及其在模式匹配中的作用。

## 函数式编程惯用法

我们还将研究 Scala 的一些函数式编程方面。

*   用于映射值的内置方法（`map`、`flatMap`）。
*   什么是单子（monad）以及为什么你应该关注它。
*   `Option` 类作为避免空值检查的一种方式。
*   链式单子调用。
*   for 推导式及其底层工作原理。

这仅仅是函数式编程的冰山一角，但我希望本书的这一部分能让你在接触 Scala 和函数式编程时，有一个有用的先发优势。
