# 3. 表达式

计算机程序通过表达式来操作数据。本章将介绍简单表达式和复合表达式。

## 表达式简介

*表达式*是字面量、变量名、方法调用（在第 6 章中讨论）和运算符的组合。在运行时，表达式会求值为一个类型，称为表达式的类型。

如果表达式被赋值给一个变量，它们的类型必须*一致*（必须能够在不丢失信息的情况下从一种类型转换为另一种类型），否则编译器将报告错误。

## 简单表达式

*简单表达式*是字面值、变量名（包含值）或方法调用（返回值）——我将在第 6 章中介绍方法和调用。以下示例演示了这些简单表达式用于初始化变量：

```
int i = 2; // 类型为 int 的字面值 (2)
int j = i; // 类型为 int 的变量名 (i)
boolean success = open("file.txt"); // 返回布尔值 true/false 的方法调用
```

第一个示例将字面量 `2` 赋值给变量 `i`。字面量和变量具有相同的类型：`int`。

第二个示例将变量 `i` 赋值给变量 `j`。两个变量具有相同的 `int` 类型。

第三个示例调用 `open()` 方法，向该方法传递要打开的文件名，并将结果赋值给变量 `success`。我们假设 `open()` 的返回类型是 `boolean`，与 `success` 的返回类型匹配。

这些示例假设只有类型与它们正在初始化的变量类型匹配的表达式才能被赋值给这些变量。但是，也可以赋值具有不同类型的简单表达式。例如，Java 允许你将各种 `int` 字面量赋值给 `short` 类型的变量，例如 `short x = 90;`，并将 `short` 赋值给 `int` 变量，例如 `int y = x;`。

前一个赋值是可以的，因为 `90` 可以表示为短整数——可表示的最大 `short` 是 `32767`，因此 `90` 适合 `short`。相反，当遇到 `short x = 80000;` 时，编译器会报错，因为 `80000` 无法在 16 位空间中表示，并且会丢失信息。

后一个赋值是可以接受的，因为当你从值集较小的类型转换为值集较大的类型时，不会丢失信息。`short` 的值范围可以轻松放入 `int` 更大的值范围内。

Java 使用拓宽转换规则来支持以下基本类型转换：

*   `byte` 到 `short`、`int`、`long`、`float` 或 `double`

*   `short` 到 `int`、`long`、`float` 或 `double`

*   `char` 到 `int`、`long`、`float` 或 `double`

*   `int` 到 `long`、`float` 或 `double`

*   `long` 到 `float` 或 `double`

当将较小的整数转换为较长的整数时，Java 会将较小整数的符号位复制到较大整数的额外位（左侧）。如果你想了解有符号整数表示，请查看维基百科的“有符号数表示”（[`http://en.wikipedia.org/wiki/Signed_number_representations`](http://en.wikipedia.org/wiki/Signed_number_representations)）条目。

前面的示例侧重于变量声明上下文中的简单表达式。其中一种简单表达式类别，即方法调用，也可以出现在非变量声明上下文中。以下是一个示例：

```
success = open("file.txt");
```

此示例假设之前已经声明了 `boolean success;`。



## 复合表达式

*复合表达式*是由一系列简单表达式通过*运算符*连接而成的序列，运算符是以符号形式表示的指令序列，用于将操作数转换为新值。例如，`-1`是一个复合表达式，其中`-`是运算符，`int`字面量`1`是其操作数。该表达式将`1`转换为其相反数：`-1`。同样，`4 * y`也是一个复合表达式，其中`*`是运算符，`4`和`y`是其操作数。当该表达式被求值时，会获取`y`的值并与`4`相乘，表达式的值即为乘法运算的结果。

注意

如果`x`的类型是`byte`或`short`，该变量的值会被拓宽为`int`类型。但如果类型是`long`、`double`或`float`，则`4`会被拓宽为相应的类型。加法运算在拓宽操作完成后执行。

Java 根据运算符所操作的操作数数量对其进行分类：

*   *一元运算符*接受单个操作数。在前面的`-1`示例中，`-`（一元负号）运算符接受单个操作数。
*   *二元运算符*接受两个操作数。`4 * y`中的`*`（乘法）运算符就是二元运算符的一个例子。
*   Java 支持一个*三元运算符*，它接受三个操作数。该运算符由两个符号（`? :`）表示，被称为*条件运算符*。我将在后面讨论这个运算符。

Java 还根据运算符的位置将其分为前缀、后缀和中缀：

*   *前缀运算符*是位于其单个操作数之前的一元运算符（如`-1`）。
*   *后缀运算符*是位于其单个操作数之后的一元运算符（如`x++`——递增`x`中的值）。
*   *中缀运算符*是位于二元运算符的两个操作数之间（如`4 * y`）或三元运算符的三个操作数之间的二元运算符。

在求值表达式时，编译器会考虑每个运算符的优先级和结合性：

*   *优先级*决定了当表达式包含两个或多个不同优先级的运算符时，表达式的求值顺序。优先级高的运算符先被求值。例如，在表达式`4 + 2 * 3`中，子表达式`2 * 3`先被求值，因为乘法的优先级高于加法。
*   *结合性*决定了当表达式包含两个或多个相同优先级的运算符时，表达式的求值顺序。例如，`4 + 2 + 3`会先求值`4 + 2`，因为`+`具有从左到右的结合性。如果`+`具有从右到左的结合性，则会先求值`2 + 3`，然后将结果与`4`相加。

### 运算符示例

表 3-1 中的一些运算符可能比较熟悉；另一些则可能令人费解。接下来的几节将按优先级从低到高的顺序简要描述所有运算符。

#### 赋值运算符

Java 支持简单赋值（`=`）和复合赋值（`+=`、`-=`、`*=`、`/=`、`%=`、`&=`、`^=`、`|=`、`<<=`、`>>=` 和 `>>>=`）运算符。

##### 简单赋值

*=* *运算符*将表达式的值赋给一个变量。该运算符的语法如下：

```
变量 '=' 表达式 ';'
```

注意

你必须指定单引号之间的内容（但不包括单引号本身）。此外，斜体项是你必须输入的占位符。这也适用于表 3-1 中的 `(`*类型*`)`。（在该表中，我没有使用单引号来包围你需要逐字输入的文本，因为我不想造成混淆。）

*变量*和*表达式*的类型必须一致，以避免编译器错误。例如，你不能直接将一个`double`值赋给一个`int`变量。

考虑以下示例，它将`self_employed`初始化为`true`：

```
boolean self_employed = true;
```

##### 复合赋值

*+=*、*-=*、**=*、*/=*、*%=*、*&=*、*^=*、*|=*、*<<=*、*>>=* 和 *>>>=* *运算符*先执行其运算，然后将结果赋给变量。它们的语法如下所示：

```
变量 '+=' 表达式 ';'
变量 '-=' 表达式 ';'
变量 '*=' 表达式 ';'
变量 '/=' 表达式 ';'
变量 '%=' 表达式 ';'
变量 '&=' 表达式 ';'
变量 '^=' 表达式 ';'
变量 '|=' 表达式 ';'
变量 '<<=' 表达式 ';'
变量 '>>=' 表达式 ';'
变量 '>>>=' 表达式 ';'
```

同样，变量和表达式的类型必须一致，以避免编译器错误。

看看这些例子：

```
int x += 1; // 递增 x。
int x -= 1; // 递减 x。
int x *= 2; // 将 x 乘以 2。
int x /= 2; // 将 x 除以 2。
int x %= 2; // 将 x 除以 2 的余数赋给 x。
int x &= 0xFFFFFF00; // 关闭 x 中最右边的八位。
int x ^= 0xFFFFFFFF; // 关闭 x 中所有已设置的位。
int x |= 0xFF000000; // 打开 x 中最左边的八位。
int x <<= 1; // 将 x 乘以 2。
int x >>= 1; // 将 x 除以 2。如果 x 为负数，结果也为负数。
int x >>>= 1; // 将 x 除以 2。如果 x 为负数，结果为正数。
```

在乘以或除以 2 的幂时，`<<=` 和 `>>=` 运算符比其对应的 `*` 和 `/` 运算符执行得更快。此外，在处理二进制数时，它们也更直观。



#### 条件运算符

*?: 运算符*根据条件决定对两个表达式中的哪一个进行求值。其语法如下：

```
condition '?' expression1 ':' expression2
```

*条件*是一个布尔表达式。当条件求值为 true 时，将对 *expression1* 进行求值并返回其结果。然而，当条件求值为 false 时，将对 *expression2* 进行求值并返回其结果。*expression1* 和 *expression2* 的类型必须一致，以避免编译器错误。例如，你不能让 *expression1* 是 `int` 类型而 *expression2* 是 `double` 类型。

以下示例展示了如何使用 `?:` 根据一位女性的 `age` 来判断她是否有资格领取养老金：

```
int age = 60;
boolean eligibleForPension = (age >= 55) ? true : false;
```

因为 `age` 大于 `55`，所以 `true` 被赋值给 `eligibleForPension`。

#### 逻辑或运算符

*||* *运算符*会对其左操作数进行求值，并且仅当左操作数为 false 时，才对右操作数进行求值。这被称为*短路求值*。其语法如下：

```
operand1 '||' operand2
```

*operand1* 和 *operand2* 都必须为 `boolean` 类型。当 *operand1* 和 *operand2* 中有一个或两个都为 true 时，此表达式返回 true；否则，返回 false。

考虑以下示例：

```
int x = 93;
boolean outOfRange = (x < 10 || x > 50);
```

该示例首先将 `x` 初始化为 `93`。然后，它使用逻辑或运算符结合 `x` 来判断 `x` 是否在范围 `10` 到 `50` 之外。因为 `x` 不在此范围内（其值为 `93`），所以 `true` 被赋值给 `outOfRange`。

由于 `x` 等于 `93`，表达式 `x < 10` 和 `x > 50` 都会被求值。但是，如果 `x` 等于 `5`，由于短路求值，则只需要对 `x < 10` 表达式进行求值。

短路求值可以帮助你避免在空引用上调用方法。考虑以下示例：

```
str == null || str.isEmpty()
```

在此示例中，`str` 引用了一个 `String` 对象。我们想在此对象上调用 `String` 的 `isEmpty()` 方法，以判断该字符串是否包含任何字符。然而，如果 `str` 包含 null，调用 `isEmpty()` 将导致 `NullPointerException`。我们通过首先使用表达式 `str == null` 来测试 `str` 是否为空，从而避免了这种可能性。如果 `str` 包含 null，则 `str == null` 为 true，并且 `||` 不会对 `str.isEmpty()` 进行求值，从而防止了 `NullPointerException`。但是，如果 `str` 不包含 null，则 `str == null` 为 false，并且 `||` 会对 `str.isEmpty()` 进行求值，从而防止了 `NullPointerException`。你将在后面的第 13、6 和 11 章中分别了解 `String`、方法和 `NullPointerException`。

#### 逻辑与运算符

*&&* *运算符*会对其左操作数进行求值，并且仅当左操作数为 true 时，才对右操作数进行求值。这也被称为*短路求值*。其语法如下：

```
operand1 '&&' operand2
```

*operand1* 和 *operand2* 都必须为 `boolean` 类型。当 *operand1* 和 *operand2* 都为 true 时，此表达式返回 true；否则，返回 false。

考虑以下示例：

```
int x = 93;
boolean inRange = (x >= 10 && x <= 50);
```

此示例与上一节中的示例基本相同。它只是反转了该示例的行为。

短路求值可以帮助你避免在空引用上调用方法。考虑以下示例：

```
str != null && !str.isEmpty()
```

此示例与上一节中的示例基本相同。它只是反转了该示例的行为。

#### 按位包含或运算符

*|* *运算符*对其操作数的位进行包含或运算。对于每一位，如果其中一个或两个位都为 1，则结果位为 1；否则，结果位为 0。此运算符的语法如下：

```
operand1 '|' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回包含或运算后的结果。

考虑以下示例：

```
byte status = readPort(DEVICE);
writePort(DEVICE, status | ENABLE_PRINTER);
```

此示例从一个假设计算机的 `DEVICE` 端口读取一个状态字节。然后，它通过将 `ENABLE_PRINTER` 值与 `status` 字段进行或运算（以防打印机未启用）来启用打印机，并将此值写入 `DEVICE` 端口。

`|` 运算符可以像 `||` 一样使用，但不执行短路求值。它总是对其右操作数进行求值。示例如下：

```
x < 10 | x > 50
```

在此示例中，右操作数（`x > 50`）总是被求值。

#### 按位异或运算符

*^* *运算符*对其操作数的位进行异或运算。对于每一位，如果其中一个位为 1 但两个位不都为 1，则结果位为 1；否则，结果位为 0。此运算符的语法如下：

```
operand1 '^' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回异或运算后的结果。

考虑以下示例：

```
int i = 10;
int j = 20;
i = i ^ j; // 30
j = i ^ j; // 10
i = i ^ j; // 20
```

此示例使用 `^` 来交换变量 `i` 和 `j` 的内容。

#### 按位与运算符

*&* *运算符*对其操作数的位进行与运算。对于每一位，如果两个位都为 1，则结果位为 1；否则，结果位为 0。此运算符的语法如下：

```
operand1 '&' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回与运算后的结果。

考虑以下示例：

```
byte status = readPort(DEVICE);
writePort(DEVICE, status & ~ENABLE_PRINTER);
```

此示例从一个假设计算机的 `DEVICE` 端口读取一个状态字节。然后，它通过将 `ENABLE_PRINTER` 值的取反值与 `status` 字段进行与运算（以防打印机已启用）来禁用打印机，并将此值写入 `DEVICE` 端口。

`&` 运算符可以像 `&&` 一样使用，但不执行短路求值。它总是对其右操作数进行求值。示例如下：

```
x >= 10 & x <= 50
```

在此示例中，右操作数（`x <= 50`）总是被求值。

#### 相等运算符

Java 支持等于（`==`）和不等于（`!=`）相等运算符。

##### 等于

*==* *运算符*比较两个操作数是否相等。此运算符的语法如下：

```
operand1 '==' operand2
```

当两个操作数相等时，`==` 运算符返回 true；否则，返回 false。

示例如下：

```
int x = 10;
int y = 15;
boolean equal = x == y;
```

在此示例中，`x` 和 `y` 各自包含一个整数。`x == y` 表达式返回 false，因为 `x`（`10`）不等于 `y`（`15`）。



##### 不等于

*!=* *运算符*比较两个操作数是否不相等。该运算符的语法如下：

```
operand1 '!=' operand2
```

当两个操作数不相等时，`!=` 运算符返回 true；否则返回 false。

示例如下：

```
int x = 10;
int y = 15;
boolean equal = x != y;
```

在此示例中，`x` 和 `y` 各自包含一个整数。表达式 `x != y` 返回 true，因为 `x`（`10`）不等于 `y`（`15`）。

对于这两个运算符，两个操作数都必须具有可比性。例如，你不能将整数常量与字符串常量进行比较。

当涉及对象时（在第 6 章中讨论），这些运算符不会比较它们是否相等或不相等。相反，它们会比较这些对象的引用。例如，`"abc" == "abc"` 会比较第一个字符串常量的引用与第二个字符串常量的引用。由于一种称为“驻留”的机制（在第 13 章讨论 `String` 类时会提到），这些引用是相同的，因此表达式返回 true。对于 `"abc" != "abc"`，表达式返回 false。

#### 关系运算符

Java 支持小于（`<`）、小于等于（`<=`）、大于（`>`）、大于等于（`>=`）以及用于对象的类型比较（`instanceof`）关系运算符。

##### 关系小于

*<* *运算符*比较两个操作数，以判断左操作数是否小于右操作数。该运算符的语法如下：

```
operand1 '<' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int`、`long`、`float` 或 `double` 类型。当 *operand1* 小于 *operand2* 时，此表达式返回 true；否则返回 false。

考虑以下示例：

```
x < y
```

当 `x` 小于 `y` 时，此表达式计算结果为 true；否则计算结果为 false。

##### 关系小于等于

*<=* *运算符*比较两个操作数，以判断左操作数是否小于或等于右操作数。该运算符的语法如下：

```
operand1 '<=' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int`、`long`、`float` 或 `double` 类型。当 *operand1* 小于或等于 *operand2* 时，此表达式返回 true；否则返回 false。

考虑以下示例：

```
x <= y
```

当 `x` 小于或等于 `y` 时，此表达式计算结果为 true；否则计算结果为 false。

##### 关系大于

*>* *运算符*比较两个操作数，以判断左操作数是否大于右操作数。该运算符的语法如下：

```
operand1 '>' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int`、`long`、`float` 或 `double` 类型。当 *operand1* 大于 *operand2* 时，此表达式返回 true；否则返回 false。

考虑以下示例：

```
x > y
```

当 `x` 大于 `y` 时，此表达式计算结果为 true；否则计算结果为 false。

##### 关系大于等于

*>=* *运算符*比较两个操作数，以判断左操作数是否大于或等于右操作数。该运算符的语法如下：

```
operand1 '>=' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int`、`long`、`float` 或 `double` 类型。当 *operand1* 大于或等于 *operand2* 时，此表达式返回 true；否则返回 false。

考虑以下示例：

```
x >= y
```

当 `x` 大于或等于 `y` 时，此表达式计算结果为 true；否则计算结果为 false。

##### 对象的类型比较

*instanceof* *运算符*比较其左操作数（一个对象）与其右操作数（一个引用类型），以确定该对象是否为该引用类型的实例。该运算符的语法如下：

```
operand1 'instanceof' operand2
```

*Operand1* 必须是某个引用类型（例如类——关于对象和类的讨论请参见第 6 章）的实例（对象）。*Operand2* 必须是一个类或其他引用类型。当 *operand1* 是 *operand2* 的一个实例时，此表达式返回 true；否则返回 false。

考虑以下示例：

```
"abc" instanceof String
```

此表达式计算结果为 true，因为 `"abc"` 是 `String` 类的一个实例。

#### 移位运算符

Java 支持按位左移（`<<`）、带符号扩展的按位右移（`>>`）以及带零扩展的按位右移（`>>>`）移位运算符。

##### 按位左移

*<<* *运算符*将其左操作数中的位向左移动，移动的位数由右操作数指定。每次移位，最右侧位补 0，最左侧位被丢弃。该运算符的语法如下：

```
operand1 '<<' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回 *operand1* 向左移动 *operand2* 位后的结果。

考虑以下示例：

```
2 << 3
```

此表达式将 `2` 向左移动 `3` 位。结果为 16。本质上，我们是将 `2` 乘以 8。这比使用 `*` 运算符快得多。然而，它仅限于乘以 2 的小次幂。

##### 带符号扩展的按位右移

*>>* *运算符*将其左操作数中的位向右移动，移动的位数由右操作数指定。每次移位，最左侧位（*符号位*，指示其余位表示正数还是负数——0 表示正数，1 表示负数）的副本被移到右侧，最右侧位被丢弃。这被称为*有符号移位*。该运算符的语法如下：

```
operand1 '>>' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回 *operand1* 向右移动 *operand2* 位后的结果。

考虑以下示例：

```
4 >> 1
-4 >> 1
```

第一个表达式将 `4` 向右移动 `1` 位。结果为 2。第二个表达式将 `-4` 向右移动 `1` 位。结果为 -2。本质上，我们是将 4 或 -4 除以 2。这比使用 `/` 运算符快得多。然而，它仅限于除以 2 的小次幂。



##### 无符号右移

*>>>* *运算符*将其左操作数的二进制位向右移动，移动的位数由右操作数指定。每次移位时，右侧会补入一个 0，而最右侧的位则被丢弃。这被称为*无符号移位*。该运算符的语法如下：

```
operand1 '>>>' operand2
```

*operand1* 和 *operand2* 都必须为 `char`、`byte`、`short`、`int` 或 `long` 类型。此表达式返回将 *operand1* 向右移动 *operand2* 位后的结果。

请参考以下示例：

```
4 >>> 1
-4 >>> 1
```

第一个表达式将 `4` 向右移动 `1` 位，结果为 2。第二个表达式将 `-4` 向右移动 `1` 位，结果为 2147483646（即 -2 的无符号等效值）。本质上，这是将 4 或 -4 除以 2。这比使用 `/` 运算符快得多。然而，它仅限于除以 2 的小次幂。

对于涉及 32 位表达式的每种移位操作，仅使用 *operand2* 的低 5 位，以防止移位超过 32 位整数的位数。对于涉及 64 位表达式的每种移位操作，仅使用 *operand2* 的低 6 位，以防止移位超过 64 位表达式的位数。

#### 加法运算符

Java 支持加法/字符串连接 (`+`) 和减法 (`-`) 加法运算符。

##### 加法/字符串连接

*+* *运算符*将其两个数值操作数相加并返回总和。该运算符的语法如下：

请参考以下示例：

结果是一个新的 `String` 对象，包含序列 `abcdef`。（我将在第 13 章讨论 `String` 类。）

##### 减法

此示例从变量 `x` 的值中减去变量 `y` 的值，这两个变量分别被赋值为 `10` 和 `20`。然后将差值赋给变量 `difference`。

#### 乘法运算符

#### 其他运算符

Java 还支持括号 [`()`]、数组索引（`[]`）和成员选择（`.`）运算符。

##### 括号

*()* *运算符*允许你更改数值表达式的优先级顺序。例如，考虑以下表达式：

```
4 + 2 * 3
```

在此表达式中，`2 * 3` 会首先被计算。然后，其结果与 `4` 相加。

你可以通过使用括号来更改计算顺序。考虑以下表达式：

```
(4 + 2) * 3
```

在此表达式中，`4 + 2` 会首先被计算。然后，其结果与 `3` 相乘。

你可以将括号嵌套到任意深度。考虑以下表达式：

```
(4 + (5 – 3)) * 2
```

最内层的括号会首先被计算，因此 `5 – 3` 会首先被计算。然后，其结果与 `4` 相加。最后，结果再乘以 `2`。

*操作数 1* 是对对象或引用类型（例如类）的引用，通常表示为变量名（包括对象名）或引用类型名。*操作数 2* 是引用类型的字段或方法的名称。（我将在第 6 章讨论对象、类、字段和方法。）考虑以下示例：

在此表达式中，字符串字面量 `"abc"`（实际上是一个 `String` 对象）提供了对此对象的引用，而 `length()` 是 `String` 类中的方法之一。此方法返回 `"abc"` 中的字符数，恰好是 3。（我将在第 13 章介绍 `String` 及其 `length()` 方法。）



## 玩转表达式

表达式可以很有趣，因为你永远不知道会发现什么。我创建了一个 `ExprDemo` 应用程序，并在其中填充了几个示例表达式，作为本次探索的起点。清单 3-1 展示了 `ExprDemo` 的源代码。

```
class ExprDemo
{

{

System.out.println("-4 << 1: " + (-4 << 1));
System.out.println("1 << 31: " + (1 << 31));
System.out.println("1 << 32: " + (1 << 32));
System.out.println("-1 << 31: " + (-1 << 31));
System.out.println("-1 << 32: " + (-1 << 32));

System.out.println("x++: " + x++);
System.out.println("x: " + x);

```

`ExprDemo` 重点关注 `instanceof`、`+`（字符串拼接）、`()`（括号）、`<<`（左移）、`-`（取反）、`=`（赋值）、`++`（后置自增）和 `/`（除法）这些运算符，它们出现在表达式中，而这些表达式的值作为参数传递给 `System.out.println()` 方法调用。

第一个方法调用中的表达式使用 `instanceof` 来判断 `"abc"` 是否是 `String` 的实例，`String` 是一个类的例子。（第 6 章将教你关于类的知识。）回想一下，当 `instanceof` 的左操作数（一个引用）是其右操作数（一个引用类型）的实例时，它返回 true；否则返回 false。

事实证明，`"abc"` 字符串字面量是 `String` 的一个实例，这使得 `"abc"` 很特殊。你将在第 13 章（我会探讨 `String`）中了解到字符串字面量有多么特殊。

第一个方法调用中的表达式还使用了 `+` 来将两个字符串拼接成一个更大的字符串。该运算符的左操作数是字符串字面量 `"\"abc\" instanceof String: "`（注意 `\"` 转义序列用于在字符串字面量中放置双引号字符），但右操作数不是字符串：带括号的 `"abc" instanceof String` 表达式求值为一个布尔值 true/false。这是怎么回事？

Java 的 `+` 运算符对其数值操作数进行加法运算并返回它们的和。然而，当其中一个操作数是字符串而另一个操作数不是字符串时，`+` 会使用*字符串转换*将非字符串转换为字符串。如果该字符串是一个对象（你将在第 6 章学习对象），`+` 会使用该对象的 `toString()` 方法来获取该对象的字符串表示。如果操作数是基于原始类型的值（例如布尔值 true），`+` 会将该操作数转换为字符串，而不依赖 `toString()`。`+` 运算符将右操作数字符串追加到左操作数字符串之后，并返回拼接后得到的字符串。

将括号运算符与字符串拼接一起使用

你可能想知道为什么我在 `"\"abc\" instanceof String: " + ("abc" instanceof String)` 中给 `"abc" instanceof String` 加上了括号。我这样做是因为我希望输出如下所示：

```
"abc" instanceof String: true
```

如果没有括号，我会看到以下输出：

```
true
```

字符串拼接 (`+`) 的优先级高于对象的 relational 类型检查 (`instanceof`)。如果没有括号，编译器生成的字节码就好像表达式被括号括起来如下：

```
("\"abc\" instanceof String: " + "abc") instanceof String
```

字节码首先计算 `"\"abc\" instanceof String: " + "abc"`，结果是 `"abcabc"`。然后计算 `"abcabc" instanceof String`，返回 true。最后，它执行 `System.out.println()`，参数为 `true`，并输出该值。

这是另一个没有括号就无法按预期求值的表达式示例：

```
System.out.println("4 + 2: " + 4 + 2);
```

此方法调用输出 `4 + 2: 42` 而不是 `4 + 2: 6`。

回想一下，结合性决定了当一个表达式包含两个或多个具有相同优先级的运算符时，它是如何求值的。例如，表达式 `"4 + 2: " + 4 + 2` 由子表达式 `"4 + 2: "`、子表达式 `4`（在左侧 `+` 运算符之后）和子表达式 `2`（在右侧 `+` 运算符之后）组成。

表达式 `"4 + 2: " + 4 + 2` 的求值过程如下：

1.  计算 `"4 + 2: " + 4`。结果是字符串 `"4 + 2: 4"`。

2.  计算 `"4 + 2: 4" + 2`。结果是字符串 `"4 + 2: 42"`。

这个求值顺序可以通过使用括号运算符来阐明，如下所示：

```
(("4 + 2: " + 4) + 2)
```

要更改此表达式以便看到 `4 + 2: 6`，你需要在 `4 + 2` 周围加上括号，如下所示：

```
"4 + 2: " + (4 + 2)
```

这次，`(4 + 2)` 会首先被求值，因为括号运算符具有最高优先级。

`<<` 运算符将其左操作数向左移动其右操作数指示的位置数。接下来的六个方法调用演示了这个运算符。

这些方法调用中第一个的表达式将 `4` 向左移动一个位置。结果是 `8`。这相当于将 `4` 乘以 2，但速度更快。

第二个方法调用的表达式将 `-4` 向左移动一个位置。结果是 `-8`。这并不意外！

第三个方法调用的表达式将 `1` 向左移动 31 个位置。结果是 -2147483648，这可能看起来令人惊讶，因为 `1` 是正数而结果是负数。然而，当你考虑到一个 32 位的值正在被左移，其最左边的位（位置 31——最右边的位在位置 0）是符号位（0 表示正数，1 表示负数），并且 1 被移入了符号位，这就不足为奇了。

Java 以*二进制补码*（[``en.wikipedia.org/wiki/Two's_complement``](http://en.wikipedia.org/wiki/Two%2527s_complement)）格式表示整数，这种格式允许你通过将所有位翻转为相反状态并将结果加 1，来“将正二进制数可逆地转换为具有等效负值的负二进制数”。例如，假设一个 4 位整数，位置 3 是符号位，你会将 5 表示为 0101。翻转这些位会得到 1010。将结果加 1 会得到 1011，这是 -5 的位模式（1111 是 -1 的模式，1110 是 -2 的模式，1101 是 -3 的模式，1100 是 -4 的模式）。你可以通过翻转所有位并将结果加 1 来转换回 5。

第四个方法调用中 `1 << 32` 表达式的结果可能令人困惑。你可能期望结果是 0，因为 `1` 被移过了位置 31，并且 0 被移到了它后面。然而，结果是 1。如果你回想一下我之前关于移位运算符的讨论，我说过：

“对于一个 32 位的表达式，只使用 *operand2* 的低五位，以防止移位超过 32 位整数中的位数。”

右操作数 (*operand2*) 是 `32`，其 32 位二进制表示为 00000000000000000000000000100000。低五位是 00000，等于 0。因此，左操作数 (`1`) 没有被移位，保持不变。

在演示 `<<` 运算符的六个方法调用中，倒数第二个的表达式 `-1 << 31` 生成的结果与表达式 `1 << 31` 相同，因为在这两种情况下最右边的位都是 1，并且左边的所有位都被移出了结果。在这两种情况下，你最终都会得到 100000000000000000000000000000000。这导致 -2147483648。

最后，表达式 `-1 << 32` 产生的结果是 -1，原因与 `1 << 32` 产生的结果是 1 相同。在移位中只使用了右操作数的低五位，这导致尝试将左操作数移动零个位位置。

现在 `<<` 运算符已经讨论完毕，考虑 `-`（取反）运算符。在六个 `<<` 运算符示例的第二个中，取反反转了其 `4` 操作数的符号。它还在这些示例的第五个和第六个中反转了其 `1` 操作数的符号。

现在我们转到 `int` 变量 `x` 的声明。这个变量通过 `=` 运算符被赋值为 `3`。

接下来，在 `System.out.println("x++: " + x++)` 方法调用中，表达式 `x++` 的输出是 `3`，因为后置自增运算符 `++` 在递增变量值之前返回其变量操作数中的当前值。本质上，它将当前值保存在另一个内存位置，执行递增，然后返回保存的值。后置递减运算符 (`--`) 也是如此。相比之下，前置自增 (`++`) 和前置递减 (`--`) 分别递增和递减其变量的值，然后返回结果。为了证明后置自增操作生效了，倒数第三个方法调用输出了 `x` 中的值。

最后一个 `System.out.println()` 方法调用的表达式试图除以 0，这样你就可以看到 Java 对此的反应。你会发现抛出了一个 `ArithmeticException` 对象，并带有“`/ by zero`”消息，表明试图除以 0。（我在第 11 章讨论异常。）
