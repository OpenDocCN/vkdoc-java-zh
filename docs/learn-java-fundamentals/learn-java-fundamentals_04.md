# 4. 语句

计算机程序通过语句执行任务。本章将介绍赋值语句、决策语句、循环语句、循环分支语句以及其他语句。

## 语句简介

**语句**是一种语法单元，用于表达要执行的某个动作。语句用于给变量赋值、通过决策控制执行流程、重复执行其他语句特定次数或无限次，以及执行其他任务。

通过将语句放在`{`和`}`字符之间，可以将多条语句组合在一起，视为一条语句。这种复合语句称为**块**。除块之外的每条语句都必须以分号（`;`）结尾。虽然可以用分号结束一个块，但这不符合惯例。

## 赋值语句

**赋值语句**将表达式的结果赋给一个变量。它有两种形式：简单赋值语句和复合赋值语句。

### 简单赋值语句

**简单赋值语句**计算`=`运算符右侧的表达式，并将结果赋给左侧的变量。其语法如下：

```
变量 '=' 表达式 ';'
```

该语句以**变量**名开头，接着是`=`运算符，最后以**表达式**和分号结尾。以下是两个示例，假设变量已事先声明：

```
healthyBodyTemp = 98.6; // healthyBodyTemp 是 double 类型
name = "John Doe"; // name 是 String 类型
```

注意

在变量声明中初始化变量是简单赋值语句的一种形式，例如 `int age = 30;`。

### 复合赋值语句

与简单赋值语句密切相关的是**复合赋值语句**，它一步完成操作并将结果赋给变量。该语句的语法如下：

```
变量 ('+=' | '-=' | '*=' | '/=' | '%=' | '&=' | '^=' | '|=' | '=' | '>>>=') 表达式 ';'
```

该语句以**变量**名开头，接着是指定的复合赋值运算符之一，最后以**表达式**和分号结尾。以下是一个示例，假设变量已事先声明：

```
counter += 1; // 将 counter 的值加 1，并将结果和存储在 counter 中。
```

## 决策语句

**决策语句**允许 Java 程序在两条或多条执行路径之间做出选择。决策语句有三种：`if`、`if`-`else` 和 `switch`。

### If 语句

**if 语句**计算一个布尔表达式，当该表达式为 true 时执行另一条语句。该语句的语法如下：

```
'if' '(' 表达式 ')'
语句
```

`if` 语句以保留字 `if` 开头，接着是括号内的布尔**表达式**。随后是**语句**，当**表达式**为 true 时执行该语句。

`if` 语句的示例如下：

```
if (temperature < 70)
turn_on_furnace();
```

如果 `temperature` 的值降至华氏 70 度以下（通过计算布尔表达式 `temperature < 70` 并发现其值为 true 来确定），则调用方法 `turn_on_furnace()` 来打开暖气并加热房间。（我将在第 6 章讨论方法。）

### If-Else 语句

**if-else 语句**计算一个布尔表达式，并根据该表达式计算结果为 true 还是 false 来执行两条语句中的一条。该语句的语法如下：

```
'if' '(' 表达式 ')'
语句 1
'else'
语句 2
```

`if`-`else` 语句以保留字 `if` 开头，接着是括号内的布尔**表达式**。随后是**语句 1**，当**表达式**为 true 时执行；接着（在保留字 `else` 之后）是**语句 2**，当**表达式**为 false 时执行。

该语句的示例如下：

```
if (temperature < 70)
turn_on_furnace();
else
turn_off_furnace();
```

如果 `temperature` 的值降至华氏 70 度以下（通过计算布尔表达式 `temperature < 70` 并发现其值为 true 来确定），则调用方法 `turn_on_furnace()` 来打开暖气并加热房间。否则，调用方法 `turn_off_furnace()` 来关闭暖气（房间已达到 70 度）。

你可以将多个 `if`-`else` 语句链接在一起，如下所示：

```
'if' '(' 表达式 1 ')'
语句 1
'else'
'if' '(' 表达式 2 ')'
语句 2
'else'
'if' '(' 表达式 3 ')'
语句 3
'else'
...
语句 N
```

如果**表达式 1**为 true，则执行**语句 1**。然而，如果**表达式 2**为 false，则执行 `if '('` **表达式 2** `')'` 并计算**表达式 2**。如果为 true，则执行**语句 2**。此过程持续进行，直到其中一个表达式为 true 并执行其对应的语句。如果所有表达式都不为 true，则到达 `else` 并执行**语句 N**（默认语句）。

以下示例演示了这种链接能力：

```
if (direction == "North")
travelNorth();
else
if (direction == "East")
travelEast();
else
if (direction == "South")
travelSouth();
else
travelWest();
```

悬挂 else 问题

当 `if` 和 `if`-`else` 一起使用时，如果源代码缩进不当，可能难以确定哪个 `if` 与 `else` 配对。例如：

```
if (file.isOpen())
if (file.isReadable())
file.read();
else
file.open();
```

开发者是否意图让 `else` 匹配内部的 `if`，但代码格式不当导致看起来并非如此？例如：

```
if (file.isOpen())
if (file.isReadable())
file.read();
else
file.open();
```

如果 `file.isOpen()` 和 `file.isReadable()` 都返回 true，则执行 `file.read();`。如果 `file.isOpen()` 返回 true 而 `file.isReadable()` 返回 false，则执行 `file.open();`。尝试打开一个已打开的文件毫无意义。

开发者本意一定是想让 `else` 匹配外部的 `if`，但忘记了 `else` 匹配最近的 `if`。这个问题可以通过用花括号将内部的 `if` 括起来解决，如下所示：

```
if (file.isOpen())
{
if (file.isReadable())
file.read();
}
else
file.open();
```

当 `file.isOpen()` 返回 true 时，执行紧随其后的块。当此方法返回 false 时，执行 `file.open();`，这合情合理。

忘记 `else` 匹配最近的 `if` 并使用糟糕的缩进来掩盖这一事实，被称为**悬挂 else 问题**。



### Switch 语句

*switch* *语句*让你能够比等效的链式 `if`-`else` 语句更高效地选择多个执行路径之一。该语句的语法如下：

```
'switch' '(' 选择器表达式 ')'
'{'
'case' 值 1 ':' 语句 1 ['break' ';']
'case' 值 2 ':' 语句 2 ['break' ';']
...
'case' 值 N ':' 语句 N ['break' ';']
['default' ':' 语句]
'}'
```

`switch` 语句以保留字 `switch` 开头，后跟括号内的*选择器表达式*以及由多个 `case` 组成的主体。*选择器表达式*是任何计算结果为整数、字符或字符串的表达式。

每个 `case` 以保留字 `case` 开头，后跟一个字面值和一个冒号（`:`）字符，再跟一个要执行的语句，并可选地以 `break` 语句（本章稍后讨论）结尾。`break` 语句会使执行流程跳转到 `switch` 语句之后继续执行。

在计算完*选择器表达式*之后，`switch` 会将该值与每个 `case` 的值进行比较，直到找到匹配项。如果匹配，则执行该 `case` 的语句。例如，如果*选择器表达式*的值与*值 1*匹配，则执行*语句 1*。

可选的 `break` 语句（方括号内的任何内容都是可选的）由保留字 `break` 后跟一个分号（`;`）组成，它可以防止执行流程继续执行下一个 `case` 的语句。相反，执行流程会继续执行 `switch` 之后的第一条语句。

BREAK 语句的放置与难以发现的错误

你通常会在 `case` 的语句之后放置一个 `break` 语句。忘记包含 `break` 可能会导致一个难以发现的错误，即执行流程会从一个 `case` 的末尾流向下一个 `case` 的开头，而不是在第一个 `case` 完成后立即离开 `switch` 语句。然而，在某些情况下，你可能希望将几个 `case` 组合在一起，让它们执行共同的代码。例如，你编写了代码提示用户输入字符 `Y`（表示 Yes）或字符 `y`（表示 yes）以继续程序，并且你希望无论输入哪个字符都执行相同的代码。在这种情况下，你应该省略参与组合的 `case` 中的 `break` 语句，但该组中的最后一个 `case` 除外。请考虑以下示例：

```
System.out.print("输入 Y 或 y 以结束程序：");
// 用于输入 Y 或 y 并将该字符存储在 char 变量 ch 中的代码。
switch (ch)
{
case 'Y':
case 'y': System.out.println("已输入 Y 或 y。");
break;
}
```

如果没有 `case` 的值与*选择器表达式*的值匹配，并且存在一个默认 `case`（由保留字 `default` 后跟一个冒号表示），则会执行默认 `case` 的语句。以下示例演示了该语句：

```
String direction = "East";
switch (direction)
{
case "North": travelNorth(); break;
case "East": travelEast(); break;
case "South": travelSouth(); break;
default: travelWest();
}
```

在声明变量 `direction` 并将其初始化为 `"East"` 之后，该示例进入 `switch` 语句。该语句读取 `direction` 的字符串，然后按从上到下的顺序将该值与每个 `case` 的字符串进行比较，直到找到匹配项。

它首先将 `"East"` 与 `"North"` 进行比较。由于它们不匹配，`switch` 继续处理下一个 `case`，其字符串值为 `"East"`。由于它们匹配，`switch` 调用一个名为 `travelEast()` 的方法（我将在第 6 章讨论方法）。当此方法返回时，`switch` 执行 `break` 语句，这会导致执行流程继续执行 `switch` 之后的第一条语句。

假设 `direction` 被初始化为 `"SouthWest"`。由于没有 `case` 的字符串匹配，`switch` 将执行 `default` case：其 `travelWest()` 方法将被调用。

## 循环语句

*循环语句*允许 Java 程序重复执行其他语句。循环语句有三种：`for`、`while` 和 `do`-`while`。

注意

每次重复执行称为一个*循环*。此外，每次循环执行称为一次*迭代*。

### For 语句

*for* *语句*可以按特定次数或无限次地迭代执行另一条语句。该语句的语法如下：

```
'for' '(' [初始化] ';' [测试] ';' [更新] ')'
语句
```

`for` 语句以保留字 `for` 开头，后跟括号内的头部，再跟*语句*。头部以一个可选的*初始化*部分开头，后跟一个非可选的分号字符。接着是一个可选的*测试*部分，后跟一个分号字符。最后以一个可选的*更新*部分结束。

*初始化*部分由逗号（`,`）分隔的变量声明或变量赋值列表组成。这些变量中的一部分或全部通常用于控制循环的持续时间，被称为*循环控制变量*。

*测试*部分由一个布尔表达式组成，该表达式决定循环的长度。只要此表达式为真，循环就会继续。

*更新*部分由逗号分隔的表达式列表组成，这些表达式通常用于修改循环控制变量。

`for` 语句首先执行*初始化*部分。然后执行*测试*部分以确定循环是否应该继续。*测试*部分由一个布尔表达式组成，返回 true 表示继续，返回 false 表示不继续。在*语句*执行之后，执行*更新*部分，然后依次执行*测试*、*语句*、*更新*、*测试*、*语句*、*更新*，依此类推。

以下示例演示了 `for` 语句：

```
for (int i = 0; i < 10; i++)
{
for (int j = 0; j < i; j++)
System.out.print("*");
System.out.println();
}
```

外层的 `for` 语句以一个包含 `int i = 0;` 的初始化部分开始。这个复合赋值语句声明了一个名为 `i` 的 `int` 变量并将其初始化为 `0`。执行完此部分后，`for` 将注意力转向测试部分，该部分由 `i < 10` 组成。

`i < 10` 布尔表达式将 `i` 中的值与 `10` 进行比较。只要 `i` 的值小于 `10`，`for` 语句就会继续迭代其*语句*。

接着，该示例进入 `for` 语句头部后面的代码块。

该代码块首先执行内层的 `for` 语句，其初始化部分引入了另一个 `int` 变量 `j`，并将其初始化为 `0`。此外，其测试部分计算 `j < i`，比较 `j` 和 `i` 中的当前值。只要 `j` 的值小于 `i` 的值，此布尔表达式将计算为 true，并且这个内层 `for` 语句将继续迭代。

假设表达式为真，执行流程传递到 `System.out.print("*");`，该语句输出一个星号（`*`），并且随后不前进到下一输出行。输出之后，内层 `for` 语句的更新部分执行 `j++`，这会增加存储在 `j` 中的值。

内层 `for` 语句通过计算 `j < i` 开始另一次迭代。一旦 `j` 的值超过 `i` 的值，执行流程将离开内层 `for` 语句，并执行 `System.out.println();`，导致后续输出从下一输出行的开头开始。

### While 语句

*while* *语句*在其布尔表达式为真时重复执行一条语句。该语句的语法如下：

```
'while' '(' 表达式 ')'
语句
```

`while` 语句由保留字 `while` 组成，后跟一个带括号的布尔*表达式*，再后跟*语句*，该语句会被重复执行。

`while` 语句首先计算*表达式*。如果返回真，程序执行*语句*。然后再次计算*表达式*。如果仍为真，则再次执行*语句*。此行为持续进行，直到*表达式*计算为假，此时执行流程转到*语句*之后的第一条语句。

以下示例演示了 `while` 语句：

```
long fact = 1;
int i = 1;
while (i <= 4)
fact *= i++;
System.out.println(fact);
```

此示例计算 4!（阶乘）。*阶乘*是从 1 开始到所求阶乘的整数为止的整数乘积。例如，4! = 1 x 2 x 3 x 4（即 24）。（注意，0! 计算结果为 1。）

在声明变量 `fact`（用于保存阶乘结果）和 `i`（一个记录当前循环迭代次数的循环计数器）之后，示例进入一个 `while` 循环。此循环首先通过计算 `i <= 4` 来验证 `i` 是否未超过四次迭代。只要此布尔表达式计算结果为真，`while` 循环就会继续。

每次迭代执行 `fact *= i++;`。这个复合赋值语句完成了以下几件事：

1.  它获取 `i` 的值并将其存储在一个内存位置，然后递增该值并将结果和存储回 `i`。
2.  它获取 `fact` 的值以及从 `i` 读取的已存储值，并将这些值相乘。
3.  它将相乘得到的结果值存储在 `fact` 中。

当 `while` 循环结束时，执行 `System.out.println(fact);`。检索存储在 `fact` 中的值，然后输出到控制台。

注意后置递增运算符 `++` 的存在。此运算符递增变量 `i`，使循环最终能够结束。如果它不存在，循环将永远不会结束，因为 `i` 永远不会递增，始终为 1，而 1 小于 4。

注意

永不结束的循环被称为*无限循环*。

一个 `for` 语句可以很容易地通过等价的 `while` 语句编写，如下例所示（该示例计算从 1 到 10 的整数之和）：

```
int sum = 0;
for (int i = 1; i < 10; i++)
sum += i;
```

声明变量 `sum` 后，示例进入 `for` 语句，该语句首先声明变量 `i` 并将其初始化为 `1`。然后计算布尔表达式 `i < 10`。只要 `i` 的值小于 10，`for` 语句就会持续循环执行其后的 `sum += i;` 语句。

这后一个表达式语句使用 `+=`（复合赋值运算符之一）将 `i` 的值加到 `sum` 的值上。在此更新之后，`for` 语句执行 `i++` 来递增 `i`。接下来，它计算 `i < 10`，并在 `i` 的值小于 10 时重新执行 `sum += i;`。

等价的 `while` 语句如下所示：

```
int sum = 0;
int i = 1;
while (i < 10)
{
sum += i;
i++;
}
```

### Do-While 语句

*do-while* *语句*在其布尔表达式为真时重复执行一条语句。与在循环顶部计算布尔表达式的 `while` 语句不同，`do`-`while` 在循环底部计算布尔表达式。该语句的语法如下：

```
'do'
语句
'while' '(' 表达式 ')' ';'
```

`do`-`while` 语句由保留字 `do` 组成，后跟*语句*（该语句会被重复执行），再后跟保留字 `while`，接着是一个带括号的布尔表达式，最后是一个分号。

`do`-`while` 语句首先执行*语句*。然后计算*表达式*。如果为真，`do`-`while` 执行*语句*。再次计算布尔表达式。如果仍为真，`do`-`while` 重新执行*语句*。此循环模式持续进行，直到*表达式*为假。

以下示例演示了 `do`-`while` 语句：

```
long fact = 1;
int i = 1;
do
fact *= i++;
while (i <= 4)
System.out.println(fact);
```

此示例在功能上与上一节中的示例相同。唯一的区别是测试表达式的位置。

提示

当循环必须执行零次或多次时，使用 `while` 语句。当循环必须执行一次或多次时，使用 `do`-`while` 语句。

## 循环分支语句

计算机程序通常需要终止循环或跳过循环的当前迭代。它们通过执行 `break` 或 `continue` *循环分支语句*来完成这些任务。

### Break 语句

*break* *语句*允许你终止 `switch` 语句，这你已经见过了。然而，*break* *语句*也允许你终止循环。当循环是无限循环时，例如 `for (;;)`、`while (true)` 或 `do` … `while(true)` 循环，这是必要的。在有限循环的上下文中也可能需要它。

`break` 语句的语法如下：

```
'break' ';'
```

以下示例展示了如何跳出无限循环：

```
while (true)
{
System.out.println("按 x 退出循环。");
// 下面的方法调用从键盘读取一个字符，并将其 int 值赋给 ch。
// 我将在第 14 章中详细说明 System.in.read()。
int ch = System.in.read();
if (ch == 'x')
break;
}
```

当 `ch == 'x'` 为真时，`if` 语句执行 `break`，它将执行流程转移到 `while` 循环之后的第一条语句。

以下示例展示了如何跳出有限循环：

```
for (int i = 0; i < 1000; i++)
if (i > 500)
break;
else
System.out.println(i);
```

当 `i` 中的值超过 500 时，此示例跳出循环。

#### 带标签的 Break

`break` 语句可以指定一个*标签*（一个标识符后跟一个冒号），这允许你跳出*嵌套循环*（循环内部的循环）。这种 `break` 语句的变体允许你将执行流程转移到以标签为前缀的循环之后的第一条语句，其语法如下：

```
'break' 标签 ';'
```

以下示例演示了带标签的 `break`：

```
outer:
for (int a = 0; a < 5; a++)
{
for (int b = 0; b < 5; b++)
{
System.out.println("a = " + a + ", b = " + b);
if (b == 3)
break outer;
}
System.out.println();
}
System.out.println("两个循环都已结束");
```

此示例展示了一对 `for` 循环。外层循环可能执行五次。对于每次外层迭代，内层循环可能执行五次。然而，当 `b == 3` 计算结果为真时，执行流程离开内层和外层循环，并继续执行 `outer:` 标签标记的 `for` 循环之后的第一条语句。

生成以下输出：

```
a = 0, b = 0
a = 0, b = 1
a = 0, b = 2
a = 0, b = 3
两个循环都已结束
```



### Continue 语句

*continue* *语句*允许你跳过循环的当前迭代。当在迭代中较早满足某个条件，并且你不想继续执行本次迭代的剩余部分时，这很有必要。

`continue` 语句的语法如下：

```
'continue' ';'
```

以下示例展示了如何取消迭代的剩余部分：

```
int count = 0;
String s = "To be or not to be, that is the question.";
for (int i = 0; i < s.length(); i++)
{
if (s.charAt(i) != 'e')
continue;
count++;
}
System.out.println("Number of 'e' characters = " + count);
```

此示例计算字符串中字母 ‘`e`’ 的数量。在初始化一个 `count` 变量来记录 ‘`e`’ 的数量，以及一个 `s` 变量来引用示例字符串后，程序进入一个 `for` 循环，该循环遍历字符串的长度。表达式 `s.length()` 在变量 `s` 上调用 `String` 的 `length()` 方法，该变量持有对正在处理的字符串的引用，以返回此字符串的长度。（我将在第 13 章中探讨 `String` 及其 `length()` 方法。）

对于每次循环迭代，都会执行一个 `if`-`else` 语句。该语句调用 `String` 的 `charAt()` 方法，以返回字符串中的下一个字符。（我将在第 13 章中探讨 `charAt()`。）如果此字符不等于 `e`，则执行 `continue` 语句以跳过本次循环迭代的剩余部分，从而不执行 `count++`。然而，如果此字符等于 `e`，则不执行 `continue`，而是执行 `count++` 以增加已见 ‘`e`’ 的计数。一旦整个字符串被检查完毕，循环结束，并输出 ‘`e`’ 的计数。

#### 带标签的 Continue

`continue` 语句可以指定一个标签，这允许你跳过嵌套循环当前迭代的剩余部分。这种 `continue` 语句的变体允许你将执行转移到由该标签前缀的循环语句的布尔表达式，其语法如下：

```
'continue' label ';'
```

以下示例演示了带标签的 `continue`：

```
outer:
for (int a = 0; a < 5; a++)
{
for (int b = 0; b < 5; b++)
{
System.out.println("a = " + a + ", b = " + b);
if (b == 3)
continue outer;
}
System.out.println();
}
System.out.println("Both loops have finished");
```

此示例与我之前讨论带标签的 `break` 时的示例类似。然而，输出略有不同：

```
a = 0, b = 0
a = 0, b = 1
a = 0, b = 2
a = 0, b = 3
a = 1, b = 0
a = 1, b = 1
a = 1, b = 2
a = 1, b = 3
a = 2, b = 0
a = 2, b = 1
a = 2, b = 2
a = 2, b = 3
a = 3, b = 0
a = 3, b = 1
a = 3, b = 2
a = 3, b = 3
a = 4, b = 0
a = 4, b = 1
a = 4, b = 2
a = 4, b = 3
Both loops have finished
```

之前，输出在 `a = 0, b = 3` 处停止。现在，它对于 `a` 从 `0` 到 `4` 的每个值都继续执行。但是，你仍然不会在输出中看到 `b = 4`，因为当 `b` 等于 `3` 时，内部循环被跳过了。

## 其他语句

Java 支持其他值得一提的语句：`assert` 语句、空语句、`import` 语句、方法调用语句、`package` 语句、`return` 语句、`try` 语句和 `try`-`with`-`resources` 语句。

### Assert 语句

*assert* *语句*允许你通过一个布尔表达式来表达对程序正确性的假设。本书不讨论此语句。

### 空语句

*空语句*是一个单独的分号（`;`）字符。有时，让 `for` 语句重复执行此语句会很方便，因为实际工作是在 `for` 的头部中执行的。考虑以下示例：

```
for (String line; (line = readLine()) != null; System.out.println(line));
```

此示例使用一个 `for` 循环来读取并打印所有行（可能来自文件），直到没有更多行可读。我们正在将行从源复制到目标。

通过调用 `readLine()` 方法（你需要在其他地方指定该方法）来读取一行。通过 `System.out.println()` 方法调用来打印该行。注意行尾的分号。这个分号就是空语句。

警告

空语句可能会在你的代码中引入微妙的错误。例如，以下循环本应在 15 行上输出 `Java is great!`。但实际上，只会出现一条此消息，因为空语句被执行了 15 次。

`for (int i = 0; i < 15; i++);`

`System.out.println("Java is great!");`

### Import 语句

*import* *语句*从包中导入一个类型。我将在第 10 章中全面讨论 `import`。

### 方法调用语句

*方法调用语句*调用一个方法，该方法可能会也可能不会返回结果。如果方法的返回类型是 `void`，则该方法不返回值。例如，`System.out.println()` 方法调用上下文中的各种 `println()` 方法具有 `void` 返回类型——这个保留字表明每个 `println()` 方法不返回值。我将在第 6 章中全面讨论方法调用语句。

### Package 语句

*package* *语句*将一个或多个引用类型放入一个包中。我将在第 10 章中全面讨论 `package`。

### Return 语句

*return* *语句*将执行从方法返回到调用该方法的代码。它可以选择返回一个值，该值的类型必须与方法的返回类型匹配。我将在第 6 章中全面讨论 `return`。

### Try 语句

*try* *语句*提供了一种方法，用于检测从代码块中抛出的*异常*（偏离正常执行流程），然后将其传递给另一个代码块以处理该异常。我将在第 11 章中全面讨论 `try`。

### Try-with-resources 语句

*try-with-resources* *语句*是一个 `try` 语句，它声明一个或多个*资源*（在不需要时必须关闭的对象）。它会关闭每个资源。我将在第 11 章中全面讨论 `try`-`with`-`resources`。



## 玩转语句

语句玩起来很有趣，因为你永远不知道会发现什么。我创建了一个 `StmtDemo` 应用程序，并用几个示例语句填充了它，作为本次探索的起点。清单 4-1 展示了 `StmtDemo` 的源代码。

```
class StmtDemo
{
public static void main(String[] args)
{
if (args.length != 1)
{
System.out.println("usage: java StmtDemo ([-h] | [-v] | " + "[positive integer count])");
return;
}
switch (args[0])
{
case "-H":
case "-h": System.out.println("help");
return;
case "-V":
case "-v": System.out.println("version 1.0");
return;
default  : int count = 0;
for (int i = 0; i  '9')
{
System.out.println("invalid character detected: " + digit);
return;
}
count *= 10;
count += digit - '0';
}
for (int i = 0; i < count; i++)
System.out.print("*");
System.out.println();
}
}
}
清单 4-1
StmtDemo.java
```

`StmtDemo` 主要关注 `if`、方法调用、`return`、`switch`、简单赋值、`for` 和复合赋值语句。

`main()` 方法中的第一个示例语句使用 `if` 来确保指定了单个命令行参数。它计算布尔表达式 `args.length != 1`，并在不满足此条件时执行一个代码块（布尔表达式计算结果为 true，因为指定了零个参数或多个参数）。

第二个示例语句使用 `System.out.println()` 方法调用来输出一条用法信息。`System.out.println()` 是该代码块中的第一条语句。

第三个示例语句使用 `return` 退出 `main()` 方法并终止程序。这是该代码块中的第二条语句。

第四个示例语句使用 `switch` 来识别传递给程序的第一个且唯一的命令行参数。该命令行参数存储在 `args[0]` 中，这是 `args` 数组中的第一个位置。（我将在第 5 章中介绍数组。）

`switch` 语句通过其 `case` 分支展示了其他语句。

前两个 `case` 分支分别将 `args[0]` 与 `"-H"` 和 `"-h"` 进行比较。注意，`case "-H"` 是空的：它没有指定任何语句，甚至没有 `break` 语句。因为没有 `break` 语句，当在命令行中指定 `-H` 时，执行流程会落入 `case "-h"` 的语句中。

`case "-h"` 的第一条语句是一个 `System.out.println()` 方法调用，用于输出一条简短的帮助信息。可以推测，`-h` 会显示关于程序用法的有用帮助信息。在这个例子中，我提供了一个简单的消息，让你了解可能出现的内容。

`case "-h"` 的第二条语句是一个 `return` 语句，它将执行流程从 `switch` 语句和 `main()` 方法中转移出来，从而终止程序。

接下来的两个 `case` 分支分别将 `args[0]` 与 `"-V"` 和 `"-v"` 进行比较。它们在功能上与前面两个 `case` 分支相同，但输出的是简短的版本号信息。

注意

许多程序都有版本号。例如，`1.0` 通常表示程序的第一个版本。句点（`.`）之前的数字通常称为主版本号，句点之后的数字通常称为次版本号。在这个例子中，“`1`”是主版本号，“`0`”是次版本号。

主版本号表示程序的主要变更。次版本号表示程序的次要变更。请查看维基百科的“软件版本号”条目（[`http://en.wikipedia.org/wiki/Software_versioning`](http://en.wikipedia.org/wiki/Software_versioning)）以了解更多关于版本号的信息。

当在命令行中指定了 `-H`、`-h`、`-V` 或 `-v` 以外的内容时，将执行 `default` 分支。

`default` 分支执行的第一条语句是一个简单赋值语句的变体，它声明了一个 `count` 变量并将其初始化为 0。

接下来要执行的是一条 `for` 语句，它遍历命令行参数的字符。

`for` 语句在其初始化部分声明了一个名为 `i` 的 `int` 变量，并将其初始化为 `0`。然后，它将此变量与 `args[0].length()` 进行比较，后者返回构成命令行参数的字符数。

`args[0].length()` 表达式由两个操作数组成，它们由成员选择运算符（`.`）分隔。`args[0]` 操作数使用数组索引运算符（`[]`）从数组中返回第一个命令行参数，该参数以字符串形式表示。`length()` 操作数是一个 `String` 方法，它返回字符串中的字符数。

`for` 语句执行的代码块以 `char digit = args[0].charAt(i);` 开头，该语句从 `args[0]` 字符串中提取下一个字符（预期是一个数字），并将其赋值给 `char` 变量 `digit`。`String` 提供了一个 `charAt()` 方法，该方法返回由其单个参数（必须是 `int` 类型）指定的索引处的字符。

该代码块接下来执行一条 `if` 语句，其布尔表达式将 `digit` 中存储的字符与 `'0'` 和 `'9'` 进行比较。如果此字符在数值上小于 `'0'` 的 Unicode 编号或大于 `'9'` 的 Unicode 编号，则会调用一个代码块。

注意

在创建用于那些使用 0 到 9 之外数字字符的国家（例如，参见 [`http://en.wikipedia.org/wiki/Tamil_numerals`](http://en.wikipedia.org/wiki/Tamil_numerals)）的 Java 程序时，硬编码诸如 `digit < '0' || digit > '9'` 这样的表达式并不是一个好主意。相反，你应该调用 `Character` 类的 `boolean isDigit(char ch)` 方法来确定一个字符是否代表数字。（`Character` 是 Java 引用类型库的成员。）

该代码块首先执行一个 `System.out.println()` 方法调用，以输出一条错误消息以及有问题的字符。（注意 `+` 运算符用于将两个字符串连接成第三个字符串。）

然后，该代码块执行一条 `return` 语句，这会导致执行流程离开 `main()` 并终止程序。

如果存储在 `digit` 变量中的字符确实是一个数字，则执行流程会转到一对复合赋值语句：

```
count *= 10;
count += digit - '0';
```

第一条复合赋值语句读取 `count` 的值，将其乘以 `10`，并将结果存储回 `count`。第二条复合语句读取 `count` 的值，将 `digit` 中的 Unicode 编号加到此值上，然后减去 `'0'` 的 Unicode 编号。结果存储回 `count`。

`main()` 方法中的最后两个示例语句包括一条用于输出 `count` 个星号（`*`）的 `for` 语句，以及一条 `System.out.println();` 方法调用语句，用于使后续输出从下一输出行的开头开始。

按如下方式编译清单 4-1：

```

```

按如下方式运行生成的应用程序：

```

```

你应该会看到以下输出：

```
usage: java StmtDemo ([-h] | [-v] | [positive integer count])
```

现在，使用 `-h`（或 `-H`）命令行选项运行该应用程序：

```

```

应用程序会输出以下消息作为响应：

```
help
```

接着，使用 `-v`（或 `-V`）命令行选项运行该应用程序：

```
java StmtDemo -v
```

你应该会看到以下输出：

```

```

最后，使用参数 `10` 运行该应用程序：

```

```
