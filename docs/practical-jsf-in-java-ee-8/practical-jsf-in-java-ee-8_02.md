# 前言

当我最初在 2010 年底想要写一本关于 JSF 的书时，使用 Java 和 JavaServer Faces (JSF) 开发 Web 应用程序已经给我带来了多年的巨大乐趣（和成功）。我与一些德国出版商联系，目标是写大约 200-250 页关于这个主题的内容，仅此而已。“不，谢谢，太专业了”是一个回答。另一个回答是：“很好。再加一些页数，再写一千页，写写整个 Java 企业版。让 JSF 保持在 200 页以内。”令人沮丧的回答。

于是，我在 2011 年初开始写关于 JSF 的博客。并且我成为了 JSF 2.2 (JSR 344) 专家组的成员。与大多数其他志愿者不同，我不是 JSF 的实现者，而是一位专家级的 JSF 用户。我也成为了 JSF 2.3 (JSR 372) 专家组的成员。我仍然是一位专家级的 JSF 用户，但我开始在 JSF 源码中编写一些代码。如果我能被接纳为 JSF 下一版本（无论其在过渡到 Eclipse 基金会后叫什么名字）的专家组成员，我希望贡献代码。

我关于使用 JSF 进行 Web 开发的教程仍然是我博客（[blog.mueller-bruehl.de](https://blog.mueller-bruehl.de)）中最受欢迎的部分，我从未放弃写一本关于这个主题的书的意图。随着时间的推移，我从用德语写博客转变为主要用英语写开发文章。而那位德国出版商如果知道我如今也在写相关的 Java EE 内容，一定会很高兴。

以我博客的文章为坚实基础，我开始撰写我的书《使用 Java 和 JSF 进行 Web 开发》，该书于 2014 年首次通过 Leanpub（[www.leanpub.com/jsf](http://www.leanpub.com/jsf)）自行出版。它变成了一本“活”书。第一个版本只涵盖了基础知识。每一位购买它的读者都能下载后续的更新。因此，这本书逐渐成长起来。

2016 年，Apress 首次邀请我按原样出版这本书。我拒绝了，因为我想添加更多内容。当他们在 2017 年再次邀请我时，这本书已经达到了让我同意的阶段。当然，还有更多内容可以写，但对于每一本书，你都需要对内容做出最终决定。

今天，你手中拿的是我前一本著作的增强版。它遵循相同的方法，将理论背景与实践开发相结合。书名《Java EE 8 中的实用 JSF》暗示了这不仅仅是关于 JSF——你不能孤立地使用这种 UI 技术。相反，它嵌入在我们称为 Java 企业版 (Java EE) 的完整技术栈中。Java EE 8 于 2017 年 9 月 21 日发布，Oracle 已开始将其捐赠给 Eclipse 基金会。即使未来的版本将被称为 Jakarta EE，最新的版本仍然是包含 JSF 2.3 的 Java EE 8。因为本书是关于实践开发的，所以它专注于应用程序，而不是最新特性。尽管本书中的所有应用程序都运行在 Java EE 8 上，但大多数应用可能使用 Java EE 7 也能实现，而 Java EE 7 在大多数企业环境中仍然是主流技术。只有那些依赖较新 Java EE 特性的功能才需要相应的版本——本书描述的最后一个应用程序 Alumni 就利用了这些新特性。

从某种意义上说，本书教你使用 JSF 作为用户界面来开发 Java EE 应用程序。这是一本为 Java 爱好者准备的书。了解 Web 技术会有所帮助，但并非必需。对于不熟悉 HTML、CSS 和其他内容的开发者，我在附录中添加了一些介绍性章节。

祝阅读愉快！

> —Michael Müller
