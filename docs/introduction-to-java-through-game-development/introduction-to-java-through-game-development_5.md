如果你觉得这个主题过于令人生畏，那么这一事实应该能让你放松下来。让我们尝试给 Java 中的 OOP 下一个简单的定义。首先，你在同名的 Java 类文件中定义你的类。例如，如果你创建了一个新类来管理一艘宇宙飞船，该文件可能命名为

## 类

在开始讨论 Java 类之前，让我们先看看 Java 文档中对类的定义^(⁹)：

> *“类是创建对象的蓝图或原型。”*

这个定义引出了一些有趣的概念。首先，它暗示了你定义的类用于创建某种东西——对象。我们之前已经简单讨论过对象，但现在让我们开始将它们视为类的实例，并记住类在某个地方有一个文件，文件名与类名相同，扩展名为 `.java`。这就是 Java 编程语言中类的一般性质。让我们进一步了解它们，以及如何开始定义我们自己的类。通常，Java 中类的定义可以包含以下信息：

*   **访问修饰符**：类可以是 `public` 或 `private`，也可以使用默认访问修饰符。
*   **类关键字**：使用 `class` 关键字来创建类。
*   **类名**：名称应以字母开头，在 Java 中，通常首字母大写。
*   **超类（可选）**：类的父类（超类）名称（如果有），前面加上关键字 `extends`。一个类只能继承（子类化）一个父类。
*   **接口（可选）**：类实现的接口列表（如果有），以逗号分隔，前面加上关键字 `implements`。一个类可以实现多个接口。可以回想一下我们之前提到的 `Enumerable` 接口。
*   **类体**：类体由花括号 `{}` 包围。这里定义了类的成员——包括字段、方法和构造器。

让我们重点关注如何声明一个类。至于类的扩展和接口，我们将在下一章讨论。现在，我们将专注于访问修饰符和基本的类声明。让我展示几个标准类声明的版本，然后我们再来分析它们的区别。

### 方法

方法是另一种类型的类成员，这意味着它们总是在类定义的上下文中定义的。Java 中的方法类似于其他编程语言（如 C 和 C++）中的函数。它们是仅在调用时运行的代码块。它们可以接收参数，也可以返回值。

到目前为止，我一直避免讨论作用域的概念，因为它是一个更高级的编程概念，与语言本身的细微差别关系不大，而更多与任何给定情况下语言元素的结构有关。Java 中的变量具有作用域。变量的作用域是指该变量有效且可访问的代码块（一个或多个代码块）。

Java 中的字段与其他变量一样具有作用域，你不仅可以从类外部访问它们（如我们在清单 7-4 中所见），还可以在类内部的方法中访问它们。这允许你创建可重用的代码块，这些代码块与类的字段以及传递给方法的任何参数一起工作，以便在需要时生成返回值。实际上，方法可以不接受任何参数，也不返回任何数据。让我们通过几个不同方法声明的示例来进一步讨论。我们将继续使用前两个清单中虚构的 `GameData` 类。让我们直接看代码！

### 类设计

类设计指的是你构建的类的结构。你使用哪些类字段，定义哪些方法等等。通常，你应该保持类设计简单。根据需要或预期来增加复杂性。例如，如果你在处理类时经常使用相同的几行代码，你可能希望将其移入一个类方法中。

同样，不要在你设计的类中过度支持功能。例如，如果你以类方法的形式添加新功能，你应该问自己，这段代码是否属于给定的类，是你的类负责它，还是另一个类负责。结果可能是需要一个新类来处理你期望的功能。

继承

在 Java 中，**继承**是面向对象编程的关键特性之一，它允许我们基于现有类创建新类。创建的新类称为子类，而派生子类的现有类称为超类。让我们看看如何在 Java 中实际声明一个子类。我们将基于清单 8-1 中使用的类进行构建。请留意封装和继承是如何协同工作的。

```

01 public class BasicEncapClass {
02     private int field1 = 0;

04     public BasicEncapClass(int i) {
05         this.field1 = i;
06     }

08     public int getField1() {
09         return this.field1;
10     }

12     public void setField1(int i) {
13         this.field1 = i;
14     }
15 }

17 public class BasicSubClass extends BasicEncapClass {
18     private int newField1 = 1;

20     public BasicSubClass(int i) {
21           super(i);
22         this.newField1 = i;
23     }

25     @Override
26     public int getField1() {
27         return this.newField1;
28     }

30     @Override

32         this.newField1 = i;
33     }

异常
在之前讨论 Java 编程语言中的流程控制时，我们曾简要接触过`Exception`类以及一般的 Java 错误处理。我们将重新讨论这个主题，并谈谈在 Java 程序中使用异常的一些更细致的要点。

处理异常
当一行 Java 代码由于某种原因无法正常执行时，就会生成异常。发生这种情况时，会生成一个包含错误信息和当前执行堆栈跟踪的异常。为了响应这些错误，我们必须在代码中使用 Try-Catch 语句。

正如我们之前所见，Try-Catch 语句允许我们尝试运行一段代码，并响应在此过程中可能出现的任何错误。这可能会导致某些实现决策，这些决策可能对你的程序产生更广泛的影响，值得讨论。让我们看一些 Try-Catch 语句的示例。请注意代码中的一些细微差别，我们接下来会讨论它们。

```
//code
01 System.out.println("Exception #1:");
02 try {
03     String s = null;
04     int l = s.length();
05 } catch (Exception e) {
06     e.printStackTrace();
07 }
```


09 System.out.println("异常 #2：");

接下来的 Try-Catch 语句（异常 #2）与之类似，但它旨在捕获 `NumberFormatException`，这是一个更具体的异常类。这个版本的 Try-Catch 语句更直观，因为你可以清楚地看到代码设计用于处理哪种类型的错误。总的来说，我赞同这种方法。但有时，我也会反其道而行之，实现更通用的 Try-Catch 语句。为什么呢？

嗯，在编写各种 Java 程序和游戏时，你会遇到很多情况，你其实并不关心错误是什么。你真正关心的只是是否发生了错误。如果你觉得使用更具体的异常类没有价值，那么这样做完全没问题。只需记住，最佳实践建议我们使用更具体的异常类。

在这种情况下，我们对 Try-Catch 块中的代码了解的信息要少得多。我们不知道它可能会抛出哪些异常。请注意，代码必须进行调整以抛出每种类型的异常，因为无论首先遇到哪个异常，都会退出 Try 子句。

这又回到了形式与功能的相同点上。如果了解异常类型对你有益，并且能让你的代码更直观、更可复用，那么就使用这种方法。如果你只关心知道何时出错，而不关心原因，那么就不要在 Try-Catch 语句中使用更具体的异常。在下一节中，我们将讨论如何在 Java 程序中定义自己的异常。

堆栈跟踪

Java 编程的一个重要方面是能够处理不可预见的错误。这些错误会以异常的形式显现出来，并在你最意想不到的时候以最糟糕的方式使程序崩溃。当你遇到异常时，通常会收到一条消息和打印到标准错误的堆栈跟踪。

有时，你执行程序的环境可能不允许你访问输出流。你可能需要尝试在程序本身中捕获并指示错误。在这种情况下，明智的做法是实现一个非常高级的、包罗万象的 Try-Catch，以防止程序完全崩溃。这将让你有机会显示一些关于错误的信息，你可以利用这些信息来解决问题。

无论如何，大多数时候你都应该能够访问异常堆栈跟踪。NetBeans 会在项目执行期间将任何输出直接显示到 IDE 中的一个面板中。让我们看一个堆栈跟踪示例，以便更好地理解它为我们提供的信息以及我们如何利用它来修复错误。
