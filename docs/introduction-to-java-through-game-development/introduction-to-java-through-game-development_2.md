# 2. 什么是 Java 编程

如果你已经决定开始学习如何用 Java 编程，并且已经读到了这里，那么讨论一下 Java 编程到底是什么可能是个好主意。在本章中，我们将简要回顾历史，从宏观层面讨论计算机编程以及用 Java 编程的含义。我们还将讨论游戏编程，并介绍那个备受推崇的主游戏循环。

在深入探讨 Java 编程语言的起源故事之前，让我们先花点时间讨论一下计算机编程的总体概念。在接下来的几节中，我们将探索使用不同工具进行计算机编程，并逐步过渡到基于 IDE 的现代方法。让我们来看一看。

## 计算机与编程

在本节中，我们将讨论计算机编程的实际含义。如果你在网上搜索这个短语，可能会找到类似如下的定义：

“计算机编程是专业人员用来编写代码的过程，这些代码指示计算机、应用程序或软件程序如何执行^(¹)。在最基本的层面上，计算机编程是一组用于促成特定动作的指令。”

虽然这个定义相当准确，但我对其中的一点有异议，那就是“专业人员”这个词。编程并不需要是“专业人员”才能做；任何人都可以！让我们进一步探讨这个想法，并讨论几种不同的计算机编程方式。

### 计算机编程

计算机编程可以有几种不同的形式。我想专注于计算机编程的某个特定方面，并用它来说明 Java 的重要性。我们之前说过，程序是计算机上“一组用于促成特定动作的指令”。这句话中巧妙地隐藏了编程语言的概念。

### 程序/编程的类型

你可以编写许多不同类型的计算机程序。有些是非常底层的，用汇编语言或系统编程语言（如 C、Rust 或 Go）编写。有些是支持面向对象编程（OOP）设计的高级编程语言，如 C++、Java 或 Python。

此外，还有一些专为特定程序设计的语言，例如用于 Web 编程的 JavaScript，或用于统计计算的 R。不同的编程语言就像不同的工具一样，各有其独特的属性。有些语言是编译型语言，而有些则是解释型语言。编译型语言通常会产生一个包含原始程序二进制表示形式的可执行文件。

其他语言则是解释型编程语言。解释型编程语言也会被编译，但并非编译成机器码。相反，解释型编程语言会被编译成一种中间形式，这种形式能被编程语言的解释器所理解。解释器是一个特殊的程序，它会运行中间代码，并随后执行该程序。

编程语言有很多重要的方面，但我们只触及其中最重要的部分。Java 编程语言的主要属性是：它是一种强类型、支持 OOP 的解释型编程语言。当我们介绍 Java 中的变量时，会详细说明 Java 是强类型语言的含义。目前，简单理解其含义就足够了。

Java 会跟踪并关注程序中任何给定变量的数据类型，并且不允许你将不同类型的数据存储到变量中。有些语言，如 JavaScript，采用动态类型。这意味着在 JavaScript 中，你可以将任何数据类型存储到任何变量中，而无需重新声明。在后续的编程语言回顾中，会有更多相关内容。

就像任何工具一样，不同的编程语言有不同的最佳用途。Java 编程语言适用于许多不同的领域，从应用程序到 Web 服务器，甚至游戏。不过，公平地说，Java 通常不是构建游戏的首选语言。这主要是因为该语言的解释特性会带来一些微小的开销，这是游戏开发者不喜欢的。然而，绝大多数游戏并不会在意这点微小开销，反而可能会受益于使用 Java 所带来的有序性和简洁性。

尽管如此，Java 仍然是一种可靠的编程语言，其解释特性和对垃圾回收的内置支持使其成为学习编程，特别是学习游戏编程的绝佳选择。视频游戏的设计与大多数其他程序不同。游戏编程要求你构建一个能实时响应用户输入的程序，同时还要处理游戏的所有视频、音频、碰撞检测、敌方 AI 等，以维持游戏的沉浸感。并且在此过程中不能丢失任何帧，保持动画流畅。

由于游戏具有特定且有时极为苛刻的要求，游戏编程与其他类型的软件开发截然不同。话虽如此，Java 是一门很棒的语言，通过开发视频游戏来学习 Java 也是一个很棒的方法！本节内容到此结束。在下一节中，我们将快速回顾历史，聊聊 Java 编程语言的发展历程。

## Java 编程语言

Java 诞生于 1991 年的 Sun Microsystems 公司，当时由 James Gosling、Mike Sheridan 和 Patrick Naughton 组成的 Sun 工程师团队试图创建一种独立于处理器、能轻松在小型电子设备上运行的新编程语言。这是 Java 历史中一个重要的被忽视的方面，或许也是 Java 成为 Android 项目基础部分的原因。

该团队创建了一种具有类似 C++ 语法的编程语言，它可以在小型数字遥控器上运行，并控制按钮和屏幕等电子元件。这个名为“Green Project”的项目旨在探索数字控制的家用电子设备与计算机之间的融合，但最终失败了。然而，在 Java 语言即将被历史遗忘之前，它找到了一个潜在的新应用领域——万维网。

1993 年，HTTP 协议和最早的浏览器之一 Mosaic 登上了技术舞台。在此期间，Java 团队意识到互联网非常适合 Java 编程语言与硬件无关的特性。1995 年，James Gosling 发布了一款名为 WebRunner 的浏览器，它支持 HTML 内容和嵌入式 Java 小程序。

这巩固了 Java 的流行地位，随着早期互联网的兴起，Java 也随之蓬勃发展。在 Flash 出现之前很久，在线创建交互式体验的唯一真正方法就是使用 Java 小程序。事实上，当我刚上大学时，使用 AWT 的 Java 小程序是计算机科学课程中最早教授的内容之一。

如今，大多数浏览器已不再支持小程序功能。但 Java 已渗透到现代云技术的许多其他方面，既包括使用 Java EE/Jakarta 的服务器端，也包括基于 Java 编程语言开源克隆版的 Android 客户端。

Java 能够“一次编写，到处运行”的能力近乎神奇。Java 是如何在所有这些不同的设备和地方运行的呢？答案在于 Java 运行时环境（JRE）和 Java 虚拟机（JVM）。

### JRE

JRE，更具体地说是 Java 虚拟机，负责实际执行编译 Java 源代码后生成的 Java 字节码。请记住，Java 是一种解释型语言，这意味着编译后的 Java 程序并非可由给定 CPU 直接运行的二进制形式。相反，一个虚拟 CPU（即虚拟机）会运行字节码并驱动底层硬件。

关于 JRE 的定义，可以在亚马逊云服务（AWS）网站上找到。其内容如下：

> *Java 运行时环境（JRE）是 Java 程序正确运行所需的软件。Java 是一种为许多当前 Web 和移动应用程序提供动力的计算机语言。JRE 是连接 Java 程序和操作系统的底层技术。它充当翻译和促进者的角色，提供所有资源，以便一旦你编写了 Java 软件，它就可以在任何安装了 JRE 的操作系统上运行，无需进一步修改。*
>
> `—`摘自亚马逊云服务网站

我认为这个定义非常简洁准确。最后一句是神奇之处所在。一旦“你编写了 Java 软件，它就可以在任何安装了 JRE 的操作系统上运行，无需进一步修改”；依我拙见，这极其强大，也是 Java 编程语言的第二大卖点。

请注意，Java 不仅包含一种编程语言和库，它还包含/需要一个运行时环境来执行 Java 字节码。这与之前使用 C 和 C++ 等语言的开发环境截然不同，在那些环境中，必须非常小心地确保语言、其数据类型和核心库在不同计算机硬件上表现一致。

现在，有了 JRE 的强大功能，你可以利用它在任何操作系统上运行 Java 程序，包括不同的硬件架构（如 ARM），只要该操作系统有对应的 JRE。在下一节中，我们将介绍 Java 编程语言的第二个最重要的部分——JDK。



### JDK

Java 开发工具包（JDK）是使用 Java 编程语言开发软件的基础。JDK 是开发 Java 应用程序所需的一系列软件工具的集合。你可以下载 JDK，仅凭一个文本编辑器就能开始编写 Java 程序。事实上，我们已经安装了 JDK（Java 11 SE LTS）和集成开发环境 NetBeans，因此我们已经准备好开始编码了。

## 语法与语义

什么是正确的语法？在软件开发领域，语法是一套规则，它定义了哪些符号组合被视为该语言中结构正确的语句或表达式^(³)。这听起来很简单。我们在日常生活中经常与语法打交道。语言和语法都有其语法规则。这通常被用来类比解释编程语言的语法。

我是否使用了正确的语义？嗯……也许吧？编程语言中的语义是一个略显复杂的话题。让我们进一步探讨语义的概念。如果说语法指的是语言的文法，那么语义则指的是该语言中语法正确的语句所表达的含义。

在人类语言中，这大致相当于你说出了有意义的话。当然，你可以遵守所有语法规则，却仍然说不出连贯或合理的内容。计算机语言也是如此。但在这里，语义的概念变得有些模糊，因为正确表达一个解决方案的方式肯定有很多种。就像在人类语言中，传达相同含义的方式也有很多种。

在 Java 编程语言的学习过程中，你首先需要学习和掌握语法，因为存在语法错误的程序是无法编译的。但你可以，也一定会编写出语义不正确的程序。随着时间的推移，你将培养出必要的技能，能够用结构正确的 Java 程序来定义你试图解决的问题。可以把它想象成学习一门新人类语言中的地道表达——在你掌握了其语法之后。你需要一些时间才能用这门语言流利地表达自己。

编程语言也是如此。你必须学习构建程序和解决问题的正确方法，这是一项需要你随着时间推移不断培养和完善的技能。现在，让我们来看看 Java 编程语言的核心语法规则。

### 基本语法规则

在本节中，我们将简要回顾 Java 编程语言的基本语法规则。我们将重点关注该语言的基础知识。在编写 Java 代码时，以下几点需要牢记^(⁴)：

*   **区分大小写**：Java 是区分大小写的，这意味着标识符`Hello`和`hello`在 Java 程序中具有不同的含义。

*   **类名**：所有类名的首字母应大写。如果类名由多个单词组成，则每个内部单词的首字母也应大写，这被称为大驼峰命名法，例如：`SomeClassName`。

*   **方法名**：所有方法名应以小写字母开头。如果方法名由多个单词组成，则每个内部单词的首字母应大写，即驼峰命名法，例如：`someMethodName`。

*   **程序文件名**：程序文件的名称必须与类名完全一致。事实上，如果不存在与当前 Java 文件名匹配的公共类，Java 编译器会报错。

*   **类**^(⁵)：每个类应位于一个独立的文件中，扩展名为`.java`。类文件通常分组到文件夹中，这些文件夹被称为包。

*   **程序**：Java 程序的处理始终从`main`方法开始：`public static void main(String[] args)`。一个静态的`main`方法是任何 Java 程序的必需部分，但对于 Java 库来说则不一定。

*   **代码块与语句**：在 Java 语法中，使用分隔符`{...}`来表示一个代码块和新的代码区域。每条代码语句必须以分号结尾。

*   **关键字/保留字**：这些是特殊单词，软件开发人员不能将其用作类、方法、变量等符号的名称。

语法是你将在实践中学习的东西。在你点击编译按钮之前，IDE 就能在很大程度上检测出任何语法错误。在继续之前，我们还想提一下 Java 编程语言中的保留字集。这被列为基本语法规则总结列表中的最后一点。

## 游戏编程

如果不花点时间谈谈游戏编程，那将是我的失职。毕竟，在探索 Java 编程语言的过程中，我们将在挑战中使用三款不同的游戏。正如我之前提到的，电子游戏是一种相当独特且具有挑战性的程序类型。它们通常需要实时响应、智能的电脑对手、复杂的输入、视频、音乐、音效以及多人/网络支持。这就要求电子游戏开发者掌握必要的 API，以便完成计算机几乎能做的所有事情。

关于游戏编程，一个重要的入门信息是主游戏循环。主游戏循环是游戏和游戏引擎的一个共同属性；它是一种几乎出现在每个游戏中的特殊结构。它是游戏编程的基础，因此，我们将在下一节中更详细地讨论主游戏循环。稍后，我们还会在本书中再次遇到这个主题，并且会讨论得更深入。

### 主游戏循环

主游戏循环是一种常见结构，几乎出现在每一款电子游戏中。使电子游戏具有交互性和动态性的代码在主游戏循环中运行，但被划分为不同的主要职责。循环本身是一个受控的无限循环，使游戏能够实时运行；它是游戏中所有对象被更新并随后绘制到屏幕上的地方^(⁷)。

游戏循环的基本职责可以分解为以下步骤：初始化、更新和绘制。初始化步骤可能发生在游戏生命周期的不同时刻，但必须在某个时刻发生。更新步骤是所有游戏逻辑运行的地方。在这里，代码处理诸如碰撞检测、命中、音效、敌方 AI 等事务。最后，既然游戏状态已经更新，就该将新的游戏状态绘制到屏幕上了。这就是绘制步骤的用武之地。同时请记住，根据具体游戏的不同，网络功能也可以添加到更新步骤中。

在联网多人游戏的情况下，主游戏循环将包含与网络对象的同步。这就是主游戏循环的结构，在进行游戏编程时，你应该牢记这一点。在下一节中，我们将讨论所包含游戏项目的一般结构。这将有助于我们开始浏览不同项目的类。

### 程序结构

本书附带的三款游戏（地牢陷阱、记忆匹配和乒乓克隆）都使用相同的游戏引擎。即 Java 版本的 MmgGameApi 2D 游戏引擎。整个引擎存在于附带的 `MmgGameApiJava.jar` 文件中。总的来说，项目的复杂度递增如下：

*   乒乓克隆：简单
*   记忆匹配：中等
*   地牢陷阱：专家级

每个项目都包含以下 Java 类的自定义版本以及专有的支持类：

*   静态主类：与游戏本身同名。每个包，包括每个挑战包，都有一个用于执行该版本游戏的静态主类。
*   `MainFrame.java`：`GamePanel` 类的父类，代表容纳游戏面板的主窗口框架。
*   `GamePanel.java`：负责绘制、处理输入、切换屏幕等更多功能的主类。
*   `ScreenGame.java`：运行游戏的屏幕类。
*   `ScreenMainMenu.java`：运行游戏菜单屏幕的屏幕类。

游戏类有点复杂，所以暂时不要尝试自行编辑它们。到本书结束时，你应该能够很好地处理它们，但在此之前我们还有不少内容要学习。在接下来的部分中，我们将更详细地描述本书附带的游戏。
