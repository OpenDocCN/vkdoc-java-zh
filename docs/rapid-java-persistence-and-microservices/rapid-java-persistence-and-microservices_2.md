# 1. 引言

欢迎阅读《快速掌握 Java 持久化与微服务》。在过去的几年里，我们看到新技术层出不穷。尤其是对于新手来说，成为一名全能型开发者，甚至决定先学习哪种编程语言，都变得愈发困难。程序员通常希望学习像 Java 这样的语言，以便能在任何领域工作并做任何事情，包括 Web 开发、大数据、移动开发、机器学习等。这并不容易，而一些使这一目标更加复杂的问题包括：

*   每种编程语言都有许多框架和技术。

*   竞争激烈，上市时间非常短暂。

*   对可扩展性和数据存储的需求日益增长且永无止境。

*   复杂的部署、发布管理和自动化影响着我们的决策。

本书通过多种框架介绍基于 Java 的后端开发，这应能使开发变得更加简单、快速和高效。本书面向那些不耐烦的开发者与架构师，他们不想阅读过多的理论、数百个概念和重复的文字。相反，本书是一个快速工具，通过直击要点的解释和具体的实际案例，帮助你快速重温概念。

在进入本书主体部分之前，本章将探讨 Java 的一些优势，解释本书提供的内容，涵盖 Java 当前面临的问题，并讨论本书涉及的各种实际问题及解决方案。

## 为什么选择 Java 和本书

那么，是什么让 Java 成为一个优秀的开发解决方案，为什么开发者应该选择这本书？

对于第一个问题，在我看来，答案包括 Java 的以下优势：

*   企业级业务应用开发。

*   长期可靠且保持稳定的解决方案。

*   卓越的可读性和可维护性。

*   非常强大的社区，为你提供强大的支持及问题解决方案。

*   语言中的语法快捷方式可能仅在短期内有所帮助。

鉴于 Java 的这些优势，以下是阅读本书的几个充分理由：

*   我们通过 Java 以简化形式涵盖了所有数据访问变体。这些变体包括通过 ORM、JDBC 和 NoSQL 访问关系型数据库。

*   作为架构师，你将希望快速重温所有关键和高级概念，以便做出更好的设计决策并解决开发问题。

*   一些开发者正在转向 Java 以外的语言进行开发，以及 Spring 以外的框架。他们认为 Java 变得过于庞大，而像 Spring Data JPA、Hibernate 等框架开发速度缓慢。由于学习难度大，上市时间增加。我们将通过快速概览 JPA 及各种必要解决方案的简短形式来简化这一过程。

*   互联网上充斥着许多误导性的示例，它们使用过时的方法将 Java 与其他语言进行比较。本书采用快速笔记式的格式，也将有助于指导你。

*   几年前，开发者仅凭 Java 知识在面试中脱颖而出就面临困难。现在让我们思考一下现状，并考虑所有开发者需要掌握的知识：
    *   Java 及其所有概念和 API。

    *   最新框架及其优缺点。

    *   SQL 和关系型数据库概念。

    *   NoSQL 概念及其生产环境问题。

    *   企业级和全文搜索相关解决方案，如 ElasticSearch 和 Solr。

    *   大数据和机器学习概念。

    *   响应式开发。

    *   从 API 到中间层及数据层的安全机制。

    *   无头计算或架构。

为什么不能通过一种语言为所有这些领域和层次提供解决方案？我们都需要快速的解决方案和带示例的代码。网络上有很多信息，但并非没有问题：

*   许多在线文章已经过时。它们可能会带你进行不必要的尝试和实验。

*   它们通常包含不必要的解释。我们需要直击要点的快速解决方案。

考虑到 Java 8 及其他语言框架（如 Django）的成功，编写最少的代码行数是每个人的需求。即使这些快捷方式看起来粗糙或带有主观色彩，但在当今世界，这有时正是我们所需要的。即使概念很好，我们也不想为了完成一件小事而浏览成千上万个概念。



## Java 生态系统当前面临的问题

在享受广泛且成熟的 Java 生态系统带来的好处的同时，也存在一些需要注意的挑战。以下列表概述了其中最重要的挑战：

*   Java 已变得十分庞大。从 Java 8 到 Java 11，引入了许多新概念和 API 工具，掌握它们都需要时间。

*   对于每一层和每个系统，都有众多框架可供选择。

*   除了 Java 之外，还有许多 JVM 语言选项，包括 Scala、Clojure、Groovy、Kotlin 和 Jython。

*   以下是当前 Java 持久化状态需要考虑的一些重要方面：
    *   对于新开发项目，有大量的选项可供选择。

    *   对于初级开发者，选项从 JDBC 开始。此外还有各种轻量级库，例如 Apache DBUtils、Sql2o、Yank、JDBI 和 Spring JDBC 模板。

    *   接下来是 ORM 框架，例如 Hibernate 和 EclipseLink，以及数据映射持久化框架，例如 MyBatis。

    *   然后是 Spring 支持的 J2EE 规范 API，例如 EJB 和 JPA。

    *   如今我们还有一些更智能的选项，例如 JOOQ、Querydsl 和 Spring Data 抽象层。

## 现实世界的问题与解决方案

在本书中，我们将从架构师的经验出发，探讨所有现实世界中的实际问题。我将尝试根据我的经验，指导您如何入门。本书将涵盖所有必要的概念，并展示复杂问题的解决方案。

以下是我在 14 年经验中遇到的企业开发技术栈主要变革的简要历史：

*   一切始于 C、C++、VB、消息系统（如 MQ 系列）和数据库（RDBMS）。消息传递可能是我见过的最古老的技术。它至今仍是扩展架构最流行的方法。

*   Java、消息传递、小程序等。Java 通过 J2EE 和 JMS 拯救了企业 Web 开发。

*   Java Web 服务（SOAP）和 MVC 框架。Struts、SOAP、Spring 等让事情变得更简单。

*   REST 服务。更多的 MVC 和 Web 框架，以及 SOA 的扩展。SOA 世界有两个方面——通过 Web 服务和通过 MOM（面向消息的中间件，例如 IBM Message Broker、Tibco Rendezvous 等）。

*   门户技术（特别是 Weblogic Portal）和内容管理系统（Documentum 和 Filenet）。

*   升级的开源消息系统（AMQP、ActiveMQ、RabbitMQ、ZeroMQ 等）和框架（Protobuf 等），以及更多的 Web 框架（Play、JSF 等）。

*   HTML5（Bootstrap、AngularJS、Ember.js 等）。这包括 Thymeleaf 和其他模板技术、CSS3、Bootstrap 和 JavaScript 世界的演进、NoSQL 数据库、函数式编程（Scala 等）以及云技术（如 AWS）。

*   NodeJS、Python 等。这包括多语言编程、通过 Docker、Chef 等实现的部署演进，以及通过大数据实现的分析革命。

*   响应式编程、HTTP2、WebSockets 等。

*   采用多语言系统的微服务和云自动化部署。目前技术和可能性太多，但包括 RESTful 微服务、无头架构/函数即服务、事件驱动/响应式服务，以及基于 WebSocket 的连接服务。

*   GraphQL 现在已经开始颠覆 RESTful 世界。然而，存在许多注意事项，社区也在寻找其他可能性。
